## 今天的 brainstorm 过程摘要

今天围绕的核心问题不是“要不要做特殊骰子”，而是：

- 特殊骰子机制怎样和《世界未解之谜周刊》的核心 fantasy 结合。
- 这些机制主要应用在什么位置。
- 在现有游戏框架下，骰子应该代表什么。

讨论过程大致经历了以下阶段：

1. 先发散 `Slice & Dice` 之外还能有哪些特殊骰子机制。
2. 再讨论这些机制怎样和《世界未解之谜周刊》的世界观与周循环结合。
3. 最终收敛到：当前最主要的应用位置，是记者进入具体事件节点后的探索成果判定。
4. 同时确认该判定既要支持非战斗事件，也要支持有对手骰子的战斗或对抗事件。
5. 最后明确了三类骰子：`角色骰`、`道具骰`、`其他骰`。

## 已确认的设计边界

这些是今天讨论里已经相对明确的部分：

- 特殊骰子机制当前主要服务于“记者去事件节点探索”的判定层。
- 该判定层不能只回答“成功还是失败”，还要回答“怎么成功”“付出了什么代价”“带回了什么类型的成果”。
- 同一套判定语法需要尽量兼容非战斗与战斗节点，而不是维护两套完全不同的底层规则。
- 三类骰子的职责必须分开，不能都在做同一件事。

## 三类骰子的建议职责

### 角色骰

角色骰代表记者本人，负责：

- 稳定主能力
- 职业差异
- 角色成长
- 少量个人招牌关键词

角色骰适合承载：

- 洞察
- 隐蔽
- 压制
- 取证
- 归档
- 即时触发类个人特长

### 道具骰

道具骰代表相机、录音机、护符、药剂、武器或交通工具，负责：

- 一次性爆发
- 结果转化
- 风险对冲
- 特殊玩法开关

道具骰适合承载：

- 一次性强效果
- 结果转换
- 蓄力
- 封印污染
- 复制或延迟结算

### 其他骰

其他骰不是玩家资产，而是节点本身的压力来源，负责：

- 场景压力
- 对手反击
- 异象污染
- 舆论噪音
- 升级或倒计时风险

其他骰适合承载：

- 暴露
- 伏击
- 污染
- 倒计时
- 援军或失控升级

## 今天最推荐的方向

在今天所有发散里，最推荐的一条不是纯“点数比大小”，而是：

## 标签对撞 + 成果分配

一句话定义：

先用骰子解决“有没有压住现场、有没有突破节点”，再用剩余成果决定“带回来的东西更适合公开发表，还是更适合内部归档，以及代价是什么”。

这条线被认为最适合《世界未解之谜周刊》，原因有四个：

1. 记者调查的结果天然不该只有二元成败。
2. 它能把探索直接接到“公开报道 / 内部档案”的双层真相结构。
3. 它能统一非战斗节点和战斗节点。
4. 它保住了《世界未解之谜周刊》的编辑部视角，而不是把系统变成纯战斗游戏。

## 推荐的底层语法

每个节点拆成两部分：

- `节点需求`
- `节点压力`

节点需求回答：

- 你要怎样才算把事情做成。

节点压力回答：

- 这个节点会怎样反扑你。

示意例：

- 节点需求：`突破 2`、`取证 1`、`隐蔽 1`
- 节点压力：`暴露 2`、`受伤 1`、`污染 1`

这种语法既能表示潜入、采访、追踪，也能表示驱魔、遭遇战或撤离。

## 推荐的正面成果标签

今天讨论里，较适合作为第一版核心标签的有：

- `Break`
  - 推进节点目标，决定是否突破
- `Proof`
  - 提供可公开的证据和稿件价值
- `Archive`
  - 提供内部档案价值，推动长期真相线
- `Cover`
  - 压低暴露与舆论热度
- `Control`
  - 压制敌人、异常或混乱
- `Escape`
  - 保护撤离、节约时间、避免事故升级

## 推荐的负面成果标签

对应的负面标签建议先收敛到：

- `Alarm`
  - 惊动现场、提高热度、引来监管
- `Wound`
  - 记者受伤、装备损耗
- `Taint`
  - 污染、狂性增长、异常附着
- `Delay`
  - 额外耗时、错过窗口
- `Noise`
  - 结果失真、素材杂讯太多、不适合直接发稿

## 推荐的节点结算流程

### Step 1：组骰池

- 记者提供角色骰
- 装备提供道具骰
- 节点自动加入其他骰

### Step 2：掷骰

- 所有骰子一起掷
- 少量即时关键词先结算

### Step 3：先处理节点压力

玩家优先用正面标签去对冲负面结果：

- `Cover` 主要对冲 `Alarm / Noise`
- `Control` 主要对冲 `Wound / Taint`
- `Escape` 主要对冲 `Delay`

### Step 4：再填节点需求

用剩余正面标签满足节点需求，决定是否突破节点。

### Step 5：结算剩余成果

如果突破成功，未被消耗的成果转成带回资产：

- `Proof` -> 本周公开线索价值
- `Archive` -> 内部档案价值
- `Cover` -> 降低后续风险
- `Control` -> 降低伤病或污染后果

### Step 6：输出结构化结果

今天讨论中更偏好的结果档位不是简单三档，而是结构化结果：

- 干净成功
- 脏成功
- 部分回收
- 现场失控

这样更符合记者调查与危险出版物的题材气质。

## 第一批值得尝试的特殊关键词

今天的 brainstorm 里，最值得优先尝试的关键词是：

- `Cantrip`
  - 掷出即提供小型即时收益
- `Sticky`
  - 锁定结果，换取更强定向价值
- `Single-use`
  - 一次性爆发道具面
- `Charge`
  - 当前不兑现，未来放大
- `Convert`
  - 在 `Proof / Archive / Cover / Control` 之间转换
- `Seal`
  - 对抗污染
- `Countdown`
  - 未解决时下一轮恶化
- `Corrupt`
  - 把好结果扭成带代价的结果

## 今天已经形成的判断

以下判断在今天的讨论里已经比较稳定：

- 特殊骰子机制不应只服务“判定更炫”，而应服务“结果结构更丰富”。
- 《世界未解之谜周刊》里，骰子最自然的作用不是输出伤害，而是生产调查成果和风险后果。
- 相比纯阶段推进型，当前阶段更推荐先做“标签对撞 + 成果分配”的基础版。
- 若以后要做更重型的节点，才考虑把阶段推进机制加到高风险节点或 Boss 事件上。

## 仍然没有定稿的问题

这些问题今天还没有细化，不应直接写进正式文档：

1. 每类节点的标准需求模板应该有几种。
2. `Proof` 与 `Archive` 的比例如何影响后续报刊和档案系统。
3. 重掷、锁定、即时结算在 UX 上如何表达才不混乱。
4. 对手骰在战斗节点里是否和异常骰完全共语法，还是需要局部特例。
5. 第一批记者的签名骰面应该怎么分配，才能立住人物差异。

## 建议的后续细化顺序

如果后续继续推进，建议按这个顺序走：

1. 先定 `节点需求 / 节点压力 / 结果资产` 的通用语法。
2. 再定 8-12 个第一批关键词。
3. 再做 3 个节点样例：
   - 非战斗采访/潜入节点
   - 高污染异常节点
   - 带对手骰的战斗或对抗节点
4. 最后再决定哪些关键词进入首轮原型。

针对今天这个需求，当前最合适的技能是：

## 结论

今天形成的最有价值的结果，不是某个单独关键词，而是一个方向判断：

《世界未解之谜周刊》的节点判定层，应该从“单纯成功率检定”升级成“标签对撞后再分配成果与代价的调查结果系统”。

这就是后续值得继续细化的核心。
