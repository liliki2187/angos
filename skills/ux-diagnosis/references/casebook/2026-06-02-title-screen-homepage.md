# 2026-06-02 · 标题页 A/B 首屏与角色可读性

## 案例 1：UI 遮挡污染主视觉评估

- **场景**：标题页 / 首页 A/B 实验中，左侧 LOGO、说明文案、CTA 和次级按钮过大，压住背景角色。
- **调用**：`ux_laoge` 桌面 16:9 只读诊断。
- **判断**：这不是普通美观问题，而是 UI 权重污染 A/B 主视觉评估。玩家第一眼记住的是大按钮，而不是“周刊改变现实”的主视觉承诺。
- **用户反馈**：用户要求 LOGO 和按钮缩小、上移，并减少背景放大。
- **可迁移规则**：当页面目标是验证主视觉吸引力时，UI 必须退到启动区；CTA 不能进入角色 / 主动作安全区。
- **不升级原因**：这是标题页实验的布局案例，暂时不写进 UX 核心规则；若多次出现“UI 污染视觉实验”的问题，再提炼成实验页通用规则。

## 案例 2：背景 `cover` 会制造错误裁切变量

- **场景**：A/B 图接近 16:9，但截图视口曾使用 1440×900，`cover center` 导致背景放大和裁切。
- **调用**：`ux_laoge` 桌面构图诊断。
- **判断**：若目标是比较 A/B 主视觉，背景缩放不能成为隐藏变量；否则测试结果会受视口比例影响。
- **用户反馈**：用户指出背景似乎被放大，并要求减少角色遮挡。
- **可迁移规则**：视觉实验页优先锁定桌面 16:9，并为每张主视觉明确 `scale + position`；不要无脑套通用 `cover center`。
- **不升级原因**：当前已被“只做桌面 16:9”项目硬规则部分覆盖，具体 `cover` 处理先留作案例。

## 案例 3：UI 上移后，遮罩必须随之退场

- **场景**：LOGO 和按钮已经上移、缩小后，旧的整条左侧暗幕仍然压暗角色。
- **调用**：父级 Codex 根据用户反馈继续修正，并按 UX 老哥“保护主视觉”的判断执行。
- **判断**：遮罩原本用于保护大 UI 阅读；UI 退到左上后，遮罩应变成局部品牌阅读保护，不再覆盖角色动作。
- **用户反馈**：用户两次指出角色被压暗，希望能清楚看到人。
- **可迁移规则**：当 UI 位置或权重下降后，旧遮罩、旧面板和旧说明也要同步降级；不能让遗留保护层继续压住新的视觉主语。
- **不升级原因**：这是具体标题页构图修正；如果后续多个界面都出现“旧保护层压住新主语”，再升级到 UI 执行规范。

## 案例 4：移动端范围属于项目级，不属于 UX 自身规则

- **场景**：标题页验收过程中，用户明确要求 Angus 当前完全不需要移动版。
- **调用**：父级 Codex 同步 `AGENTS.md` 和 `docs/onboarding/ui-interaction-guidelines.md`。
- **判断**：这是项目级顶层约束，不是 UX 老哥的诊断偏好。
- **用户反馈**：用户确认 `AGENTS.md` 是项目级规则，不是 subagent 自身规则。
- **可迁移规则**：范围边界、截图口径、平台目标这类约束优先沉淀到项目级文档；subagent 只读取并遵守，不把它当作自身案例偏好。
- **不升级原因**：该规则已经升级到项目级文档，本案例只说明分层原因。
