# 2026-06-02 · 首页包装、职业幻想与读者回响

## 案例 1：界面包装先抓三件事

- **场景**：用户希望 Angus 从粗糙原型进入界面包装阶段，并询问各界面应该做到什么。
- **调用**：`steam_indie_appraiser` 参与独游吸引力判断，父级 Codex 与 UX 老哥共同落地。
- **判断**：当前包装优先抓三件事：强职业幻想、可截图产物、清晰动作钩子。系统复杂度不能替代第一眼承诺。
- **用户反馈**：用户认可该方向，并要求围绕具体 Steam 游戏经验继续分析首页、编辑部入口和周开始简报。
- **可迁移规则**：Angus 的商业吸引力判断要优先问“玩家是不是主编 / 编辑、能不能截图、第一眼动作是什么”，再讨论系统深度。
- **不升级原因**：该方向已经进入 `docs/设计采纳记录.md` 的界面包装条目；本案例只保留 SIA 的判断来源。

## 案例 2：标题页必须有动作，不只是氛围

- **场景**：标题页 / 首屏主视觉生成与 A/B 推进。
- **调用**：SIA 从独游第一眼吸引力角度参与，UX 老哥从遮挡和层级角度复核。
- **判断**：Denshattack、MOTORSlice 等样本的价值不是表层风格，而是第一眼动作清楚。Angus 首屏应让玩家看到“角色启动周刊 / 印刷机 / 巨大杂志改变现实”，不是旧报纸、档案和神秘符号堆叠。
- **用户反馈**：用户明确希望吸取概括的美式黑色幽默风格，但首页资源仍要体现 Angus 主题，并指出“我们是杂志而非报纸”。
- **可迁移规则**：引用竞品时只迁移动作钩子和第一眼承诺，不迁移表层题材；Angus 的主题图必须服务“周刊主编刊发现实”。
- **不升级原因**：当前属于标题页素材路线案例；若后续所有 Steam 首屏素材都围绕同一结构，可提炼到 capsule/key art methodology。

## 案例 3：读者来信只做发刊后薄层

- **场景**：用户询问“读者来信系统要不要进当前垂直切片”。
- **调用**：`steam_indie_appraiser` 按功能 ROI 评审。
- **判断**：不要做完整读者来信系统；当前只做发刊后 2-3 条读者来信 / 摘录，作为本期报道反馈和下周钩子。
- **用户反馈**：用户采纳“只做发刊后读者回响，不做完整系统”的边界。
- **可迁移规则**：SIA 做功能 ROI 时，优先找能增强截图和主循环的薄层；完整系统要等到 EA / 1.0 后再评估。
- **不升级原因**：该功能边界已经写入 `docs/设计采纳记录.md`，并同步到相关 GDD；本案例保留 ROI 判断过程。

## 案例 4：SIA 与 UX 分工不能混淆

- **场景**：首页视觉推进中，SIA 提供第一眼吸引力与主题判断，UX 老哥处理 LOGO/按钮遮挡、背景缩放、遮罩和安全区。
- **调用**：父级 Codex 分别调用两个 subagent，并做合并取舍。
- **判断**：SIA 不负责逐像素遮挡审计，UX 老哥不负责 Steam 小爆款潜力判断；父级 Codex 必须做总编，合并两者但不把建议堆叠进界面。
- **用户反馈**：用户认为 UX 老哥与 SIA 合作效果好，并要求建立逐步完善机制。
- **可迁移规则**：商业吸引力先问 SIA，界面执行风险再问 UX；父级只落一个清晰方案。
- **不升级原因**：该分工已经写入项目级 subagent 协作规则，本案例只作为来源。
