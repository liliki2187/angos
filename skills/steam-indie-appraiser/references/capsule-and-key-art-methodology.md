# Steam 头图 / Capsule / 主视觉方法论

## 硬规则

1. Logo 是第一主角，图像服务 Logo，不抢 Logo。
2. 主体通常只要 1 个大物件 + 1 个情绪反差。
3. 小尺寸测试优先：缩到约 `120x45` 仍应读出游戏名轮廓和主物件。
4. 不把 UI 细节当头图。头图承诺 fantasy，截图解释玩法。
5. 不把完整剧情海报当头图。玩家首先需要类型和动作，不是案件梗概。
6. 除游戏名和官方副标题外，尽量避免长文字；画面内道具文字也会抢读图。
7. 色块要狠。少量主色比复古杂物堆更容易被 Steam 列表识别。
8. 主物件要能成为记忆符号：船、牌、证件、植物、报纸、档案、骰子、印章等。
9. 风格要比题材更先被看见。不要落入“AI 复古纸张 + 神秘眼睛 + 红章 + 蓝光”的通用套路。
10. 头图不是美术设定集。不要把所有系统元素塞进一张图。

## 常见有效模式

| 模式 | 例子 | 可迁移点 |
| --- | --- | --- |
| Logo + 单一职业物件 | `DREDGE` 的船、`Papers, Please` 的证件/徽记 | 先确立职业 fantasy |
| 玩法物件直接上封面 | `Balatro` 的牌、`Inscryption` 的牌桌 | 一眼知道主要交互媒介 |
| 日常物件被异常污染 | `Strange Horticulture`、`Book of Hours` | 日常越具体，异常越有力 |
| 高对比符号化 | `WORLD OF HORROR`、`Return of the Obra Dinn` | 牺牲写实细节，换缩略图记忆 |
| 角色 / 物件 + 强情绪承诺 | `Slay the Princess`、`Mouthwashing` | 适合叙事冲击，但需避免误导 Angus 类型 |

## Angus 头图判断

Angus 头图必须让玩家第一眼读出：

1. 这是关于“周刊 / 报纸 / 编辑出版”的游戏。
2. 这不是普通报社经营，而是未解之谜 / 异常现实。
3. 玩家不是旁观案件，而是在选择版面和解释权。

失败信号：

- `M330` 或单一案件标题比游戏名更强。
- 画面像悬疑电影海报、网文封面或视觉小说主视觉。
- 大量报纸小字、印章、便签和符号互相抢注意力。
- 缩小后只剩旧纸和蓝光，读不出“周刊玩法”。

## 推荐视觉探索顺序

1. **黑白缩略图**：先画 12 张只含大形的 120x45 草图，不上细节。
2. **Logo 方向**：单独试 5 个字标方向，确认游戏名是否能在小尺寸成立。
3. **主物件测试**：报纸版面、组版手、印刷机、档案盒、头版槽位分别做图标化试验。
4. **竞品旁排**：把草图和竞品 capsule 放同一列表，看谁先被看见。
5. **再做大图**：只有缩略图过关后，才生成或绘制完整主视觉。

## Angus 可试方向

- **组版动作符号化**：一只手把异常照片放进头版槽，势力压力用图形符号压向版面。
- **报纸怪物化**：周刊版面本身翻卷、发光、露出异常，而非画完整办公室。
- **极简报纸裂缝**：黑底 + 白报纸 + 一个冷色异常 + 大 Logo。
