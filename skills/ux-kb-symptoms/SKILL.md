# KB-Symptoms · 症状词完整库（30 个）

> **使用频率**：长尾按需（D 类）
> **加载触发**：主 spec §2 列的 15 个常用症状都不能精确描述当前问题时
> **来源**：Magic UX 采访 Round 1 C 节

---

## 使用方法

1. **先查主 spec §2 列出的 15 个常用症状**——如果能精确匹配你的问题，直接用
2. **如果常用 15 个都不够准确**——查本库 30 个完整列表
3. **如果 30 个都不够**——用主 spec §2 末尾的"造词模板"新造一个，参考 KB-templates

## 引用规范

使用本库的症状时，必须在诊断标题加标注：`（症状参考：KB-S-XX）`

## 30 个症状词完整列表

| # | 症状名 | 适用场景 | 视觉/交互特征 |
|---|--------|---------|--------------|
| KB-S-1 | **决策瘫痪** | 选项过多且无推荐/对比辅助 | 同质选项卡并排>5个，无高亮/推荐标 |
| KB-S-2 | **价值黑箱** | 核心价值（价格/收益/稀有度）不可见或弱化 | 关键数字<12px、灰色、被图片挤压 |
| KB-S-3 | **信任危机** | 界面内信息自相矛盾 | 两处文案给出不同数值/时间/条件 |
| KB-S-4 | **认知断裂** | 决策依赖信息分散在不相邻区域 | 眼动需跨越>150px 或翻页才能同时获取 |
| KB-S-5 | **焦点竞争** | 多个元素同时争夺第一注意力 | ≥3 个元素使用饱和色/动效/大字 |
| KB-S-6 | **状态黑箱** | 操作后状态变化不可感知 | 点击后无任何视觉/听觉反馈 |
| KB-S-7 | **视觉打架** | 信息层级混乱导致扫描无序 | 同一区域混合 3+ 种字号且无间距节奏 |
| KB-S-8 | **操作黑洞** | 操作执行后无法确认是否生效 | 按钮点击后保持原状，无 loading/成功态 |
| KB-S-9 | **时间焦虑陷阱** | 倒计时/限时信息制造非理性紧迫感 | 多个倒计时同屏、红色闪烁、无法确认截止时间 |
| KB-S-10 | **信息洪流** | 单屏信息密度超过工作记忆容量 | 同屏>12 个独立信息单元无分组 |
| KB-S-11 | **路径迷雾** | 用户不知道下一步该做什么 | 无进度指示、无引导文案、多个入口无优先级 |
| KB-S-12 | **误操作陷阱** | 危险操作无防护且靠近高频操作 | 删除/提交按钮紧邻日常按钮，无确认弹窗 |
| KB-S-13 | **空间霸占** | 低优先级内容占据核心视觉区域 | 装饰图>60% 面积，功能区被挤到边角 |
| KB-S-14 | **幽灵可操作** | 可交互元素看起来像静态文本 | 无 hover 态、无边框、无光标变化 |
| KB-S-15 | **虚假层级** | 视觉层级与信息重要性不匹配 | 标题比正文小、次要信息比主要信息颜色更深 |
| KB-S-16 | **锚点缺失** | 无参照物导致无法评估当前状态好坏 | 进度条无目标标记、数值无"满值"对照 |
| KB-S-17 | **语境蒸发** | 进入子页面后丢失来源上下文 | 弹窗/新页面无标题回指、无面包屑 |
| KB-S-18 | **选择悖论** | 选项数量过多导致决策质量下降 | >7 个平级选项无筛选/排序/推荐 |
| KB-S-19 | **密度失衡** | 界面某区域极密某区域极疏 | 左侧 15 个元素挤压，右侧大片留白 |
| KB-S-20 | **反馈延迟** | 操作后反馈不及时导致重复操作 | 点击后>300ms 无响应，用户连续点击 |
| KB-S-21 | **心算陷阱** | 强迫用户做本应由系统完成的计算 | 需要手动加减/比对两组数据才能得出结论 |
| KB-S-22 | **状态耳语** | 状态变化有但反馈过于微弱 | 变化仅体现在某处小字/图标颜色微调 |
| KB-S-23 | **时空错位** | 时间信息与空间位置暗示的顺序矛盾 | 时间轴从右到左、新内容出现在底部但旧的在顶部 |
| KB-S-24 | **身份模糊** | 多个同类对象缺乏视觉辨识度 | 5 张卡片除文字外完全相同样式 |
| KB-S-25 | **期望落空** | 视觉暗示的可操作性与实际不符 | 看起来可点但不可点，或看起来不可点但能点 |
| KB-S-26 | **上下文剥夺** | 做决策时看不到决策所需的历史/全局信息 | 详情页无法对比、确认弹窗看不到原始数据 |
| KB-S-27 | **冷启动荒原** | 首次/空状态无任何引导 | 空列表无文案无图无操作引导 |
| KB-S-28 | **终态缺席** | 不知道任务完成后什么样 | 无成功态预览、无目标状态示意 |
| KB-S-29 | **回退恐惧** | 不确定操作是否可撤销 | 无"撤销"入口、无"可重来"暗示 |
| KB-S-30 | **层级坍塌** | 所有信息在同一视觉平面无主次 | 标题/正文/辅助信息同字号同颜色同间距 |

## 主 spec 已包含的（无需查 KB）

主 spec §2 已列出常用 15 个：决策瘫痪 / 价值黑箱 / 信任危机 / 认知断裂 / 焦点竞争 / 状态黑箱 / 视觉打架 / 操作黑洞 / 信息洪流 / 路径迷雾 / 误操作陷阱 / 虚假层级 / 心算陷阱 / 身份模糊 / 冷启动荒原

## KB 独有的"冷门症状"（15 个）

- KB-S-9 时间焦虑陷阱（限时营销专用）
- KB-S-13 空间霸占（活动主页常用）
- KB-S-14 幽灵可操作（可发现性问题）
- KB-S-16 锚点缺失（数据可视化常用）
- KB-S-17 语境蒸发（多级导航常用）
- KB-S-18 选择悖论（与决策瘫痪有区别——后者强调操作，前者强调质量下降）
- KB-S-19 密度失衡（视觉节奏问题）
- KB-S-20 反馈延迟（与操作黑洞有区别——有反馈但慢）
- KB-S-22 状态耳语（与状态黑箱有区别——有反馈但弱）
- KB-S-23 时空错位（时间序列界面常用）
- KB-S-25 期望落空（与幽灵可操作互补——反方向）
- KB-S-26 上下文剥夺（详情页/弹窗常用）
- KB-S-28 终态缺席（流程类界面常用）
- KB-S-29 回退恐惧（编辑/创作类常用）
- KB-S-30 层级坍塌（与虚假层级有区别——后者错位，前者完全无层级）
