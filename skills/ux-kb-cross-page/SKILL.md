# KB-Cross-Page · 跨界面 / 产品级诊断库

> **使用频率**：任务条件触发（B 类，~15% 任务）  
> **加载触发**：用户提供 >1 个界面（同一产品的多个界面）时**必须**加载  
> **来源**：Round 5 Q 节

---

## 使用方法

单界面诊断用主 spec §1-§13 即可。**跨界面/产品级诊断**时必须加载本 KB：
1. 先对每个界面做单独诊断（用主 spec）
2. 再对照本 KB 的 10 项跨界面检查
3. 最后用 Q3 产品级报告模板输出综合结果

---

## Q1. 跨界面额外检查项（10 条）

| # | 检查项 | 检查方法 | 典型问题 |
|---|--------|---------|---------|
| KB-X-1 | **术语全局一致性** | 列出核心概念（资源名/操作名/状态名），检查每个界面是否用同一个词 | A 界面叫"编队"B 界面叫"组队"C 界面叫"配置" |
| KB-X-2 | **视觉规范一致性** | 同类元素（按钮/卡片/标签）在不同界面的样式是否统一 | A 界面 CTA 是圆角橙色，B 界面 CTA 是直角蓝色 |
| KB-X-3 | **状态传递完整性** | 从 A 界面到 B 界面时，A 的操作结果是否在 B 中可见/可确认 | 在编队界面选了角色 → 进入战斗 → 看不到自己选了谁 |
| KB-X-4 | **导航可回溯性** | 从任何子界面是否能回到上一级？路径是否可预测？ | 进入详情弹窗后无返回按钮/关闭按钮位置不一致 |
| KB-X-5 | **数据同步一致性** | 同一数据（资源量/等级/进度）在不同界面是否显示相同值 | 主城显示金币 10000，商城显示金币 9800 |
| KB-X-6 | **操作范式一致性** | 同类操作在不同界面是否用同一交互方式 | 装备界面用拖拽，技能界面用点击选择——同为"放入槽位" |
| KB-X-7 | **信息层级一致性** | 核心信息在不同界面的视觉优先级是否一致 | 角色 HP 在战斗中是最大字号，在队伍界面却是最小灰字 |
| KB-X-8 | **流程衔接完整性** | A→B 界面的过渡是否有上下文保持（标题/面包屑/进度指示） | 从活动列表点入子活动后，完全不知道自己从哪来的 |
| KB-X-9 | **错误恢复路径** | 如果在 B 界面操作失败，是否能回到 A 界面的有效状态 | 支付失败后没有"返回商品"入口，只能关闭重开 |
| KB-X-10 | **新手流程连贯性** | 新手首次路径是否跨界面无断裂（每步都有引导衔接到下一步） | 创建角色后进入主城，无任何指引"接下来做什么" |

## Q2. 发现 A 界面合理但与 B 冲突的方法

### 方法：跨界面语义域对齐表

```
步骤 1：列出所有界面共享的语义域
       （资源名、操作按钮样式、状态颜色编码、导航结构、术语）

步骤 2：为每个语义域建立对齐表

示例：
| 语义域：主要操作按钮 | 界面A（地图） | 界面B（编队） | 界面C（战斗） | 一致？ |
|--------------------|-------------|-------------|-------------|--------|
| 颜色 | 橙色 | 橙色 | 绿色 | ❌ C 不一致 |
| 形状 | 圆角 12px | 圆角 12px | 圆角 12px | ✓ |
| 位置 | 底部居中 | 底部右侧 | 底部右侧 | ⚠️ A 与 BC 不同 |
| 字号 | 16px | 14px | 16px | ⚠️ B 偏小 |

步骤 3：标记不一致项 → 评估是否有合理原因
       - 如果有（如战斗界面绿色=开始，不同于常规橙色 CTA）→ 记录但不算问题
       - 如果没有（纯粹设计不统一）→ 标记为跨界面 P2

步骤 4：重点检查"状态传递"
       模拟用户从 A→B→C 的完整流程：
       - 在 A 做的选择，到 B 时是否可见？
       - 在 B 的操作结果，到 C 时是否保持？
       - 如果任何一步"状态消失" → 标记为跨界面 P1
```

### 发现冲突的心理模型

把自己想象成**第一次玩这个游戏的玩家**，按实际路径走一遍：

```
主城 → 点击"编队" → 配置角色 → 返回 → 点击"出击" → 进入战斗
```

每次界面切换时问自己：
- 我在上一个界面做的事，这个界面"记得"吗？
- 我在上一个界面学到的规则/颜色编码，在这里还适用吗？
- 如果我想回去改刚才的配置，怎么回去？

**任何一个"不确定"的瞬间 → 就是跨界面问题。**

## 跨界面诊断的"跨 P0"判定

**跨 P0**：单看任何一个界面都"合理"，但放在产品流程中导致用户卡死/误操作。

| 典型跨 P0 | 表现 |
|---------|------|
| 关键状态消失 | 在 A 界面选了 X，到 B 界面看不到选择结果 |
| 跨界面术语不一致 | A 叫"钻石" / B 叫"水晶"——玩家以为是两种货币 |
| 流程断头路 | 进入子界面后无法回到主流程 |
| 数据不同步 | 同一资源在不同界面数字不同 |

跨 P0 问题在单界面诊断中**永远发现不了**，必须用跨界面对齐表才能挖出来。

## 实战案例：世界未解之谜周刊（6 界面）

如果是这种多界面产品（全球地图/区域节点/编队/判定/合成/组版），跨界面检查重点：
1. **术语一致性**：探索/调查/搜索 — 三个动词描述同一操作？
2. **颜色编码**：属性色（探索绿/生存红/胆识紫）在所有界面是否完全一致？
3. **状态传递**：在编队界面选的角色 → 在判定界面是否高亮显示？
4. **流程衔接**：取材 → 编队 → 判定 → 合成 → 组版 这条主路径是否有断点？
5. **新手引导**：第一次玩家从全球地图到最后组版，能否一路理解每一步该做什么？
