# 探索与节点派遣

> **状态**：草案
> **作者**：Codex + 仓库综合整理
> **最后更新**：2026-05-08
> **对应支柱**：调查必须导向发刊

## 概览

这个系统覆盖周循环前半段的探索部分：区域可用性、节点选择、员工分配、派遣校验，以及线索输出准备。它的任务，是把有限时间和员工能力转化为可进入内容生产链的素材。

## 玩家幻想

玩家应感受到自己正在指挥一支脆弱但有能力的调查团队，在投入报社本周有限资源之前，先阅读风险、稀缺性和潜在收益。

## 详细设计

### 核心规则

1. 世界按区域组织，每个区域包含常驻、临时或隐藏节点。
2. 区域通过与宏观属性、标记或既有证据相关的显式规则解锁。
3. 每次派遣可选择 1-3 名员工，其属性会聚合后与节点需求比较。
4. 只有当玩家剩余天数足够，并满足节点要求的属性阈值时，该节点才有效。
5. 普通任务成功会生成线索或推进任务；失败可能只产生弱线索、压力效果或关闭一次性机会。黑骰任务另按四档结果处理。
6. 当玩家主动停止，或本周预算已不足以支持有效行动时，探索结束。
7. 本系统只运行在 `weekly-run-loop.explore` 父阶段内；是否切到 `editorial` 由周循环根据 `remaining_days`、玩家输入和 `legal_dispatch_count` 判定。

### 任务类型与首区试玩落地

首轮 HTML 原型验证中，北美禁区带承担任务类型教学。任务卡与派遣配置页必须同时展示任务类型名称与行为说明，确保玩家在点击前后都能理解差异。

| 入口 / 标识 | 试玩承载 | 行为 |
|------------|----------|------|
| 白色调查 | `51 区外围公路` | 可反复追踪；失败不封死题材，后续派遣会继续累积同一条调查线。 |
| 红色截稿 / 临时情报 | `突发：雷达异常光点` | 一次性情报窗口；成功、失败或超过截止日期后都关闭并从地图消失。 |
| 深度调查连续任务 | `罗斯威尔档案残页` 起点 | 每阶段完成后生成下一阶段任务，事件逐层推进，直到最终完结；地图只显示当前最新任务入口，用叠层表现背后有链。 |
| 危险标识 | 普通任务、红色截稿或深度链中的高难节点 | 不是独立任务类型，只表示难度较高、失败后果更重，可能带来 debuff，但不使用黑骰特殊机制。 |
| 黑骰任务 | 深度调查终局、少数事件或主线临时任务 | 使用黑骰特殊判定机制与四档结果，通常不是普通地图高危任务。 |

首轮不要求全地图普及这些类型；核心目标是让玩家在首个可玩区域明确感知“任务不同，派遣决策也不同”。

### 深度调查任务链 UI

深度调查任务链必须让玩家感知“这是连续调查”，但地图不再完整摊开整条链。地图层只承担当前行动入口；链条结构通过同一个入口的叠层与展开动效表达。

- 地图上只显示当前最新任务环。完成第 1 环后，地图标识替换为第 2 环；旧环不继续留在地图上占位。
- 默认态是规整对齐的任务页叠层，提示“多个任务合一”；不使用斜散、扑克牌式堆叠。
- 鼠标悬停只做轻量预展开，让几张卡略微错开，不完整暴露下一环信息。
- 点击入口后，同一组叠层卡片平滑展开成线性链，显示当前环与下一条 `?` 线索；未到达的后续环不得提前完整展示。
- 侧栏任务卡仍展示链条进度、上一环结果、当前环风险和成功后是否解锁下一环。
- 结算弹窗必须明确说明下一环是否出现，以及旧环如何归档为已完成历史。

### 地图标签避让

区域地图上的任务点、线索点与可展开任务入口必须经过碰撞避让，而不是只依赖固定坐标。深度链、黑骰、高危等高优先级入口优先保留原位；普通任务与线索标签在不离开区域视觉范围的前提下让位。验收时应覆盖默认态、hover 态、深度链展开态，以及宽 / 中 / 窄视口，确认标签不互相压盖。

### 状态与切换

| 状态 | 进入条件 | 退出条件 | 行为 |
|------|---------|---------|------|
| `region_browse` | 进入探索阶段 | 选中区域 | 浏览区域列表与解锁提示 |
| `node_select` | 已选中区域 | 员工与节点准备完毕 | 检查节点、过滤项与可用性 |
| `dispatch_ready` | 存在合法的节点与员工组合 | 按下执行 | 展示概率与任务摘要 |
| `dispatch_resolved` | 事件检定返回结果 | 玩家选择下一步 | 应用线索与状态变化 |

### 与其他系统的交互

- **周循环与状态** 提供 `explore` 父阶段、`remaining_days`、`active_tasks`、`opportunity_ids`、当前周与宏观快照。
- **事件检定结算** 消耗聚合后的员工值、节点难度与对手值，再返回结果层级。
- **宏观属性与现实偏移** 会影响隐藏节点和区域解锁规则。
- **线索与内容素材库存** 会消费本系统产出的线索对象。

## 公式

### 员工属性聚合

```
team_total[attr] = sum(staff_i[attr] for each selected staff member)
```

| 变量 | 类型 | 范围 | 来源 | 说明 |
|------|------|------|------|------|
| `staff_i[attr]` | int | 0-10+ | 员工数据 | 单个员工在某属性上的值 |
| `team_total[attr]` | int | 0-30+ | 计算值 | 面向某需求的团队聚合值 |

**预期输出范围**：0 到 30+  
**边界情况**：未选择任何员工时，派遣无效。

### 派遣风险预览

派遣配置页的整体风险预览用于帮助玩家判断“当前队伍能否覆盖任务需求”。风险口径应优先看需求覆盖度、缺口与关键骰面数量；不能因为加入一个低贡献角色导致整体风险反向升高。低贡献角色应单独提示为“无相关面 / 贡献低 / 占用人数槽”，让玩家理解这是效率问题，而不是任务本身变得更危险。

### 节点可用性判定

```
dispatch_enabled =
  (remaining_days >= node.days)
  and staff_count in [1, 3]
  and all(team_total[attr] >= node.need[attr] for attr in node.need)
  and node_visibility == true
```

| 变量 | 类型 | 范围 | 来源 | 说明 |
|------|------|------|------|------|
| `remaining_days` | int | 0-7 | 周状态 | 可用时间预算 |
| `staff_count` | int | 0-3 | 选择状态 | 已分配员工数 |
| `node.need[attr]` | int | 0-10+ | 节点数据 | 每项属性需求 |
| `node_visibility` | bool | true/false | 解锁逻辑 | 玩家是否能看到该节点 |

## 边界情况

| 场景 | 预期行为 | 理由 |
|------|----------|------|
| 未选择员工 | 节点不可执行 | 防止非法派遣状态 |
| 临时节点已被处理 | 本周内禁用该节点 | 保留时间压力 |
| 运行中途满足隐藏节点解锁条件 | 立即揭示，而不重建整个循环 | 让宏观压力显得是活的 |
| 需求只差一点 | 节点仍不可用，但 UI 必须解释原因 | 保持可读性 |

## 依赖

| 系统 | 方向 | 依赖性质 |
|------|------|----------|
| 周循环与状态 | 本系统依赖它 | 读取阶段、天数、标记与周状态 |
| 事件检定结算 | 本系统依赖它 | 使用它结算派遣结果 |
| 线索与内容素材库存 | 它依赖本系统 | 消费线索输出 |
| 宏观属性与现实偏移 | 双向依赖 | 决定区域/节点解锁，也可能被结果修改 |

## 调参旋钮

| 参数 | 当前值 | 安全范围 | 提高后的效果 | 降低后的效果 |
|------|--------|----------|--------------|--------------|
| MVP 区域数 | 2 | 2-4 | 内容更多样，但制作压力更大 | 引导更紧凑 |
| 单次派遣员工上限 | 3 | 2-4 | 组合深度更高 | 更难满足需求 |
| 节点天数消耗 | 1-3 | 1-4 | 周压力更高 | 单周可派遣次数更多 |
| 节点需求阈值 | 按节点配置 | 低到中 | 需要更强配队 | 更容易进入，张力下降 |

## 视觉 / 音频需求

| 事件 | 视觉反馈 | 音频反馈 | 优先级 |
|------|----------|----------|--------|
| 区域选择 | 明显的选中状态与提示面板刷新 | 轻量 UI 确认音 | 中 |
| 节点可用性变化 | 清晰的禁用原因文本 | 可无 | 高 |
| 执行派遣 | 任务面板强调与结果揭示 | 强确认与结算提示音 | 高 |

## UI 需求

| 信息 | 显示位置 | 更新频率 | 条件 |
|------|----------|----------|------|
| 区域提示与解锁状态 | 区域面板 | 选择或状态变化时 | 探索阶段 |
| 节点消耗、风险与原因文本 | 节点列表 | 刷新时 | 探索阶段 |
| 已选员工摘要 | 员工面板 | 每次选择变化 | 探索阶段 |
| 概率预览 | 任务面板 | 员工或节点变化时 | 派遣可评估时 |
| 任务类型说明 | 节点卡与派遣配置页 | 节点变化时 | 所有可派遣任务 |

## 验收标准

- [ ] 玩家能使用 1-3 名员工完成一次合法派遣。
- [ ] 所有不可用节点都必须明确说明原因。
- [ ] 探索阶段产出的线索数据可被下游系统消费。
- [ ] 区域解锁规则与隐藏节点可见性会响应宏观状态变化。
- [ ] 线索很少的一周依然合法、依然可发刊。
