# 编辑排版与发行策略

> **状态**：草案
> **作者**：Codex + 策划稿整理
> **最后更新**：2026-04-10
> **对应支柱**：发刊必须是一种立场选择

## 概览

这个系统负责把候选稿件组织成一期真正的周刊。它决定什么做头版、什么做陪衬、什么被删掉，以及本周周刊到底想以什么姿态面对读者和势力。这里处理的是“如何排”和“为什么这样排”，并显式消化来自原型的 `公开取向`、`同题疲劳` 与 `取向冲突惩罚`。

## 玩家幻想

玩家应感受到自己在做真正的主编工作。不是简单把稿件都塞进去，而是要为每个版位承担机会成本，并接受排版本身就是一种发行策略。

## 详细设计

### 核心规则

1. 每期周刊都必须从候选稿件中选出一组可刊登内容，并按版位结构完成排版。
2. 版位是有角色差异的结构单元，不只是数量槽位。至少要区分头版、常规内容版和功能性版位。
3. 排版系统必须支持“删稿”与“弃稿”。不是所有本周产出的稿件都应该上版。
4. 稿件放入不同版位后，会影响其传播权重、势力任务满足度和结算预期。
5. 空白版位允许存在，但必须有明确可见的代价。
6. 同一题材在一周内连续刊登时，后续稿件必须受到 `同题疲劳` 影响，导致其有效基础值低于稿内原始值。
7. `抢先快讯` 不写入公开取向历史；`深度报道 / 个人专栏 / 爆炸性新闻` 必须携带 `公开取向`，并在与既往同题公开取向冲突时触发需求惩罚预览。
8. 当前正式方向只锁“排版取舍会影响发行结果”；更细的连携、专题和缓冲规则保留在系统层继续打磨，不在本文件里一次性锁死。
9. 本系统是 `weekly-run-loop.editorial` 父阶段的后半段；它读取 `article_candidates`，冻结 `published_issue`，再把结果送入 `summary`。

### 状态与切换

以下状态均发生在 `周循环与状态.editorial` 内部，并位于内容生产之后。

| 状态 | 进入条件 | 退出条件 | 行为 |
|------|---------|---------|------|
| `candidate_review` | 候选稿件生成完毕 | 开始组版 | 浏览、筛选与舍弃稿件 |
| `layout_edit` | 进入排版 | 满足最小刊印条件 | 安排头版、内页和功能位 |
| `pre_publish_check` | 玩家准备发刊 | 通过校验或返回编辑 | 检查空位、任务、风险与预期 |
| `publish_commit` | 确认发刊 | 进入结算 | 固化本期版面结构 |

### 与其他系统的交互

- **内容生产链与稿件生成** 提供候选稿件池。
- **发刊结算与受众反馈** 消费最终版面结构、稿件属性、有效基础值和取向冲突结果。
- **四大势力任务、干预与声望** 会校验本期是否满足某些刊登要求。
- **同题公开取向历史** 由长期状态提供，供本阶段做取向冲突预览；本阶段只读不写。
- **周循环与状态** 负责在本周结束前保存 `slot_assignment`、`published_issue` 与 `settlement_preview`。

## 公式

### 版位加权

```
issue_value = sum(article_value * slot_weight * fit_bonus for each published article)
```

| 变量 | 类型 | 范围 | 来源 | 说明 |
|------|------|------|------|------|
| `article_value` | int | 0+ | 稿件数据 | 单篇稿件的基础传播价值 |
| `slot_weight` | float | 0.2-2.0 | 版位配置 | 版位的传播权重 |
| `fit_bonus` | float | 0.5-1.5 | 规则计算 | 稿件与版位匹配度修正 |

### 同题疲劳

```
effective_article_value = intrinsic_base * topic_fatigue_multiplier
```

说明：

- 同题稿件在同一周越晚公开，`topic_fatigue_multiplier` 越低
- 当前原型基线：
  - 第 2 篇及以后同题稿件统一乘 `0.72`
  - 若该稿件是 `深度报道 / 个人专栏 / 爆炸性新闻`，且不是同题首篇，再额外乘 `0.9`
- 排版与结算都应同时展示 `intrinsic_base` 与 `effective_article_value`

### 取向冲突惩罚

```
stance_clash_multiplier = max(0.72, 1 - 0.09 * clash_count)
```

说明：

- 仅 `深度报道 / 个人专栏 / 爆炸性新闻` 参与该判定
- `抢先快讯` 不计入公开取向追踪
- 该值应在发刊前以预览形式显式展示

### 发刊有效性

```
publish_valid = required_slots_filled and blocked_task_not_violated
```

| 变量 | 类型 | 范围 | 来源 | 说明 |
|------|------|------|------|------|
| `required_slots_filled` | bool | true/false | 版面状态 | 是否满足最小刊印要求 |
| `blocked_task_not_violated` | bool | true/false | 势力任务 | 是否触犯硬性禁令 |

### 成刊快照字段契约

`published_issue` 至少应包含：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `week` | int | 是 | 所属周次 |
| `placed_articles` | array[dictionary] | 是 | 每个版位上的稿件快照 |
| `slot_assignment` | dictionary | 是 | 版位到稿件 ID 的映射 |
| `stats_preview` | dictionary | 是 | 发刊前冻结的结算预览 |

其中 `placed_articles` 的每条记录至少应包含：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `slot_id` | string | 是 | 版位 ID |
| `article_id` | int/string | 是 | 稿件 ID |
| `recipe_type` | enum | 是 | 稿件类型 |
| `public_stance` | enum/null | 是 | 公开取向 |
| `primary_topic_key` | string/null | 是 | 同题追踪主键 |
| `intrinsic_base_value` | int | 是 | 稿内原始基础值 |
| `effective_base_value` | int/float | 是 | 吃过同题疲劳后的有效基础值 |

## 边界情况

| 场景 | 预期行为 | 理由 |
|------|----------|------|
| 有稿件但头版空缺 | 允许尝试发刊，但后果必须显著 | 头版是主编意志的核心表达 |
| 一周稿件很少 | 仍允许凑出低质量周刊 | 保持周闭环 |
| 势力任务与头版策略冲突 | 不自动替玩家解冲突，由玩家承担取舍 | 这是玩法张力 |
| 多篇稿件都适合头版 | 必须择一，其余下沉或弃用 | 强化主编判断 |
| 同题稿件公开取向互相矛盾 | 允许刊登，但必须在发刊前看到需求惩罚预览 | 让排版承担立场代价 |

## 依赖

| 系统 | 方向 | 依赖性质 |
|------|------|----------|
| 内容生产链与稿件生成 | 本系统依赖它 | 提供可刊登稿件 |
| 发刊结算与受众反馈 | 它依赖本系统 | 消费最终版面结果 |
| 四大势力任务、干预与声望 | 双向依赖 | 校验任务要求并输出任务完成度 |
| 周循环与状态 | 双向依赖 | 管理本周最终版面数据 |

## 调参旋钮

| 参数 | 当前值 | 安全范围 | 提高后的效果 | 降低后的效果 |
|------|--------|----------|--------------|--------------|
| 最小刊印要求 | 中 | 低到中 | 周刊更难凑齐 | 玩家更容易保底 |
| 头版权重 | 高 | 中到高 | 头版决策更重要 | 版位差异变平 |
| 同题疲劳强度 | 中 | 低到高 | 连续追同题成本更高 | 系列追踪更稳定 |
| 取向冲突惩罚 | 中 | 低到高 | 立场摇摆更痛 | 玩家更敢混搭 |
| 空白版位惩罚 | 中 | 低到高 | 发刊完整性更重要 | 玩家更能硬凑 |

## 验收标准

- [ ] 玩家可以明确区分“候选稿件”和“已刊登稿件”。
- [ ] 排版阶段能体现头版和普通版位的不同价值。
- [ ] 势力任务会在排版校验中真正产生影响。
- [ ] 发刊前能够看到 `同题疲劳` 与 `公开取向` 的预览反馈。
- [ ] 发刊前能看到足够清晰的风险和预期反馈。
