# 330 公交黑骰关卡玩法设计草案

> **状态**：第一版实验页已落地，正式关卡仍待试玩验证  
> **创建时间**：2026-05-19  
> **用途**：定义 330 公交黑骰第一关的局内玩法规则，重点验证“鬼混入人群”的博弈是否成立。  
> **上层来源**：`core-experience.md`、`gameplay-design-principles.md`  
> **适用范围**：当前 HTML 原型中的 330 公交黑骰关卡；不作为所有黑骰任务的通用规则。
> **实验页**：`design/prototypes/html/labs/black-dice-330-bus-lab.html`

---

## 1. 设计目标

330 公交黑骰关卡的目标不是把普通判定变难，而是让玩家在已经能达标时仍然犹豫：这次成功是否干净，自己是否相信眼前的骰子。

本关只使用三个核心机制：

1. **锁定**：保住当前骰面，也代表玩家选择相信它。
2. **重投**：除锁定骰之外，所有未锁定骰一起重投。
3. **提示**：重投后，只提示刚刚重投的未锁定骰组是否出现鬼迹。

玩家的核心决策应是：

- 为了保住点数，要锁多少骰？
- 为了获得提示，要让多少骰参与重投？
- 当前点数已经够了，但提示不干净时，是否接受脏成功？
- 是否愿意解锁高点，让它进入重投组接受检查？

---

## 2. 设计边界

### 必须保留

- 鬼只附身在玩家角色骰上。
- 不引入乘客骰、查票按钮或其他新骰子类型。
- 不把黑骰做成单纯占用骰位、固定扣点或额外数值税。
- 不在普通任务中泛化这套规则。

### 第一版不做

- 不显示每颗骰子的“已验 / 未验”状态。
- 不在决策阶段直接给单颗骰打鬼面标记。
- 不显示鬼的精确数量、概率或所在角色。
- 不提示锁定骰是否有鬼。
- 不做“鬼随机转移”或“鬼主动躲避”。

---

## 3. 核心规则

### 3.1 初始投骰

1. 玩家进入 330 公交黑骰判定。
2. 系统生成本次玩家角色骰结果。
3. 鬼附身在其中一颗玩家角色骰的某个真实骰面之上。
4. 初始投骰后不直接给出鬼迹提示。
5. 玩家只能根据当前点数和任务压力决定锁定与重投。

鬼不应伪装成异常高点。更合适的是附在“看起来合理、但可能刚好有用”的小点数或任务相关骰面上。

### 3.2 锁定

锁定的含义是：玩家选择相信这颗骰，保留它的当前结果。

- 锁定骰不会参与下一次重投。
- 锁定骰不会被下一次提示检查。
- 锁定不等于系统认证安全。
- 玩家可以在下一次重投前解锁骰子；一旦解锁，它会随所有未锁定骰一起重投，失去当前点数。

### 3.3 重投

重投不是逐颗指定，而是批量行为：

1. 玩家决定哪些骰保持锁定。
2. 点击重投后，所有未锁定骰一起重投。
3. 重投会改变这些骰子的当前点数。
4. 重投后触发一次提示。

如果所有骰都已锁定，则不能重投；玩家需要提交结果或先解锁至少一颗骰。

### 3.4 提示

提示只回答一个问题：

> 刚刚被重投的未锁定骰组里，当前是否出现鬼迹？

提示范围只覆盖本次被重投的骰组，不覆盖锁定骰。

提示必须双向存在：

| 情况 | 提示含义 |
| --- | --- |
| 未锁定重投组里没有鬼迹 | 本次重投组当前未见鬼迹 |
| 未锁定重投组里有鬼迹 | 本次重投组当前至少有鬼迹 |

提示不指出具体是哪颗骰，不显示数量，也不额外标注骰子状态。若未锁定重投组只剩一颗骰，组提示自然等价于确认该骰；这是玩家通过锁定其他骰、承担点数风险换来的信息，不需要额外 UI 标记。

### 3.5 提交与结果

玩家提交时，按当前判定系统的有效点规则结算。330 黑骰只额外检查是否带入鬼。

当前第一版实验页按真实角色骰判定口径处理：关卡必须显示需求属性与目标点，只有命中本关需求属性的骰面计入有效点；任务外骰面即使有数值，也不应被当作达标点数。2026-05-19 调试版实验页暂时关闭重投次数限制，并将需求降为 5 点，用于先观察锁定、重投组提示和污染结算的策略手感；正式关卡仍需后续单独确认重投次数与目标值。

重投组提示在第一版实验页中采用确定口径：如果刚刚重投的未锁定骰组里包含被鬼附身的角色骰，提示一定显示有鬼迹；如果不包含，则提示没有鬼迹。提示仍然不点名单颗骰，也不显示“已验 / 未验”。提交结算时，只有锁定骰会进入本次结果；若被鬼附身的锁定骰当前贡献了本关有效点，则视为“用了鬼判定”，触发脏成功或脏失败。未锁定骰即使当前有鬼，也只是候选 / 风险来源，不会被提交自动带入。

| 结果 | 条件 | 玩法含义 |
| --- | --- | --- |
| 干净成功 | 达标，且未带入鬼 | 最佳结果，可进入正常成功或大成功口径 |
| 脏成功 | 达标，但带入鬼 | 表面达成，取消优秀结果，并触发角色临时污染 |
| 安全失败 | 未达标，且未带入鬼 | 没完成目标，但没有把鬼带回来 |
| 脏失败 | 未达标，且带入鬼 | 失败并触发更重污染或后续压力 |

若带入鬼，至少应造成：

- 有效点大幅降低，或直接取消优秀结果。
- 对应角色获得临时污染，影响后续该角色属性或骰面表现。

污染是局内后果，不直接等同于“反噬”。真正反噬仍应主要发生在报道发布后的世界回响中。

## 4. 提示文案口径

提示必须简短、现场化，并明确作用范围是“刚刚重投的骰子”。不要写成全局安全判断。

### 可用方向

| 情况 | 示例文案 |
| --- | --- |
| 未见鬼迹 | 刚刚重投的骰子里，没有多出来的呼吸。 |
| 未见鬼迹 | 刚刚重投的座位安静下来了。 |
| 出现鬼迹 | 刚刚重投的骰子里，有东西跟了上来。 |
| 出现鬼迹 | 刚刚重投的座位，多出了一道影子。 |

### 避免方向

- “这几颗已验证安全。”
- “锁定区没有鬼。”
- “鬼在第 3 颗骰子。”
- “当前全场安全。”
- “鬼概率 33%。”
- “已验 / 未验”常驻状态标。

提示应服务玩家刚刚做出的选择，而不是替玩家整理完整信息表。

---

## 5. 策略深度检验

### 5.1 信息代价

提示只给未锁定重投组，因此信息不是免费的。

- 锁得越多，点数越稳，但提示覆盖越少。
- 锁得越少，提示覆盖越多，但点数波动越大。
- 想检查一颗高点骰，必须解锁它，让它参与重投并失去当前点数。

这让锁定从“保存高点”变成“我愿意相信它”。

### 5.2 典型局面

假设目标为 6，初始结果如下：

| 骰子 | 当前点数 |
| --- | ---: |
| A | 2 |
| B | 2 |
| C | 1 |
| D | 1 |
| E | 0 |

总点数刚好 6。隐藏真相：C 被鬼附身。

#### 路线一：保住成功，少量试探

玩家锁定 A、B、C、D，只重投 E。

- 优点：保住 6 点。
- 提示只覆盖 E。
- 若提示未见鬼迹，玩家仍不知道 A、B、C、D 是否干净。
- 如果直接提交，会带入 C，形成脏成功或污染。

这条路让玩家明白：保点数不等于排除鬼。

#### 路线二：放弃低点，换取较大范围提示

玩家锁定 A、B，重投 C、D、E。

- 如果提示出现鬼迹：玩家知道 C、D、E 当前组有问题，但不知是哪颗。
- 如果重投后 C、D、E 点数很高，玩家会面对“高分但可疑”的局面。
- 玩家可以继续重投这组三颗，也可以锁住其中某颗冒险提交。

这条路制造核心博弈：高点数和干净信息不能同时免费获得。

#### 路线三：全部重投，换最大范围信息

玩家不锁任何骰，全部重投。

- 提示覆盖最大。
- 但原本刚好够的点数完全丢失。
- 如果提示未见鬼迹但点数不够，玩家得到的是安全失败方向。
- 如果提示出现鬼迹且点数很高，玩家仍要决定是否接受脏风险。

这条路适合玩家非常害怕污染时使用。

### 5.3 预期最优策略

合理玩家不会有单一固定最优解，而会按当前点数、剩余重投次数和污染代价调整：

1. 点数刚好够时，倾向锁一部分高点，重投低点或可疑区。
2. 若提示未见鬼迹，玩家会把本次重投结果视为相对可信，但不会获得 UI 标记。
3. 若提示出现鬼迹，玩家需要在继续重投、放弃点数、或接受脏风险之间取舍。
4. 若后续获得点数余量，玩家可能解锁原本保留的高点去换取提示。
5. 若污染代价很重，玩家可能宁愿安全失败，也不接受脏成功。

本关好玩的关键是：玩家能过，但不一定愿意这样过。

## 6. 反无脑策略设计

| 潜在无脑策略 | 风险 | 约束方式 |
| --- | --- | --- |
| 锁够分数，只重投废骰 | 玩家几乎不排鬼却稳定提交 | 锁定组不被提示覆盖；脏成功代价必须明显 |
| 每次都全部重投 | 玩家无脑追求最大提示范围 | 重投次数有限，且点数会大幅波动 |
| 看到提示有鬼就永远不提交 | 节奏拖慢或变成保守唯一解 | 重投次数有限；安全失败仍可能失去重要机会 |
| 完全忽略提示直接拼点 | 黑骰变成普通高难任务 | 鬼污染必须影响后续角色，且取消优秀结果 |
| UI 替玩家标出已验信息 | 记忆与判断被 UI 接管 | 不显示“已验 / 未验”；只保留即时提示 |

## 7. 调参旋钮

| 参数 | 第一版建议 | 作用 |
| --- | --- | --- |
| 玩家骰数量 | 4 到 5 颗 | 保证锁定与重投分组有空间 |
| 鬼数量 | 1 个 | 第一关只教学核心博弈 |
| 重投次数 | 正式关卡待定；当前实验页暂不限制 | 实验页先观察策略，正式关卡再限制节奏与防止无限验 |
| 目标点数 | 实验页暂为 5 点；正式关卡让常见初始局面处在刚好够或多 1 点 | 制造“要不要花余量查鬼”的抉择 |
| 鬼附身位置 | 低点、任务相关面、关键补点面 | 让鬼像混入，而不是像显眼奖励 |
| 脏成功惩罚 | 取消优秀结果 + 临时污染 | 防止无脑保点数提交 |
| 安全失败收益 | 可保留少量线索或无污染撤出 | 让“干净但失败”不是完全无意义 |

调参时优先看玩家是否出现真实犹豫，而不是只看成功率高低。

---

## 8. UI 与表现要求

本草案只定义玩法，不定义最终 UI 造型。后续落地时应遵守 `docs/onboarding/ui-interaction-guidelines.md`。

### 必须表达

- 本关需要哪些属性与几点；只有命中需求属性的骰面计入有效点。
- 哪些骰子当前被锁定。
- 点击重投会影响所有未锁定骰。
- 重投后出现一条提示，说明刚刚重投的骰子里是否有鬼迹。
- 提交后明确展示干净成功、脏成功、安全失败或脏失败。
- 若发生污染，明确落到对应角色。

### 不应表达

- 不给每颗骰显示“已验 / 未验”标签。
- 不把锁定骰标成安全。
- 不在决策阶段直接点名鬼所在骰。
- 不用大段说明替代即时提示。
- 不在 UI 上堆过多规则注释。

### 提示位置

提示应出现在骰子判定现场，靠近重投结果，而不是远离骰子的全局 toast。提示是玩家理解本次重投价值的核心反馈，必须和刚刚变化的骰子建立空间关系。

## 9. 验收标准

- [ ] 玩家能只通过锁定、重投、提示完成一次 330 黑骰判定。
- [ ] 判定界面显示本关需求属性与目标点，且有效点计算与真实判定规则一致。
- [ ] 重投确实只影响未锁定骰，不存在逐颗重投按钮。
- [ ] 提示只针对刚刚重投的未锁定组，不提示锁定组。
- [ ] UI 不显示“已验 / 未验”状态标。
- [ ] 玩家在“保点数”和“获得提示”之间会产生实际取舍。
- [ ] 脏成功与干净成功有明显差异，脏成功不能获得优秀结果。
- [ ] 污染能回到后续角色状态，让玩家记住带鬼的代价。
- [ ] 第一版不需要乘客骰、查票按钮或额外小系统。

## 10. 待确认问题

1. **未锁定组只有一颗骰时，组提示自然等价于确认该骰。** 这是否符合你对第一关信息强度的预期？
2. **脏成功的最低惩罚**是否定为“取消优秀结果 + 对应角色临时污染”，还是还要额外扣本次有效点？
3. **安全失败**是否允许带回少量线索，还是完全失败但无污染？
4. **污染持续时间**第一版建议是“下一次该角色派遣”或“本周内一次”，需要后续在实现前确认。
