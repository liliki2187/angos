# 发刊结算与受众反馈

> **状态**：草案
> **作者**：Codex + 策划稿整理
> **最后更新**：2026-04-06
> **对应支柱**：传播必须推动现实偏移

## 概览

这个系统把已经排好的周刊转成世界后果。它负责销量、营收、受众反馈、势力反应、宏观属性变化和下一周钩子的结算。它只消费“这一期最终发了什么”，不回头改写稿件结构。

## 玩家幻想

玩家应感受到自己真的把一期危险周刊送进了世界。发刊不是按一下结算按钮，而是看着本周的选择如何同时改变读者、势力、周刊身份和现实。

## 详细设计

### 核心规则

1. 结算输入是最终版面结构、刊登稿件属性、本周任务结果和当前长期状态。
2. 结算至少输出销量、营收、势力反应、宏观属性变化和下一周状态钩子。
3. 受众反馈必须可读。MVP 可先聚焦少量核心读者群，而不追求完整市场模拟。
4. 发刊结算不负责重新生成稿件，只对最终刊印结果做后果计算。
5. 势力任务完成与否必须在这里产生明确奖励或报复。
6. 现实偏移反馈应在结算后显式展示，而不是埋在隐藏变量里。

### 状态与切换

| 状态 | 进入条件 | 退出条件 | 行为 |
|------|---------|---------|------|
| `issue_evaluate` | 发刊确认 | 传播值与结算项计算完毕 | 读取最终周刊并计算后果 |
| `audience_resolve` | 基础结算完成 | 受众与势力反馈写回 | 处理销量、订阅与反应 |
| `world_feedback` | 反馈写回完成 | 玩家确认下一周 | 展示宏观属性与现实偏移变化 |

### 与其他系统的交互

- **编辑排版与发行策略** 提供最终版面和刊登稿件。
- **四大势力任务、干预与声望** 提供任务奖励与惩罚模板。
- **宏观属性与现实偏移** 消费稿件属性和任务结果，更新世界状态。
- **周循环与状态** 接收订阅、资源和下周准备数据。

## 公式

### 发刊覆盖值

```
issue_reach = base_reach + reputation_scale + issue_value + audience_modifier
```

| 变量 | 类型 | 范围 | 来源 | 说明 |
|------|------|------|------|------|
| `base_reach` | int | 0+ | 周刊状态 | 报纸的基础覆盖 |
| `reputation_scale` | int | 0+ | 声望属性 | 声望带来的天然扩散能力 |
| `issue_value` | int | 0+ | 排版系统 | 本期周刊综合传播价值 |
| `audience_modifier` | int | 负到正 | 受众计算 | 受众匹配带来的修正 |

### 利润

```
profit = sales_income + sponsor_income - print_cost - operation_cost - penalty_cost
```

| 变量 | 类型 | 范围 | 来源 | 说明 |
|------|------|------|------|------|
| `sales_income` | float | 0+ | 销量结果 | 售出收入 |
| `sponsor_income` | float | 0+ | 财团/广告 | 额外商业收入 |
| `print_cost` | float | 0+ | 发刊配置 | 印刷成本 |
| `operation_cost` | float | 0+ | 周状态 | 编辑部运营成本 |
| `penalty_cost` | float | 0+ | 任务/违规 | 罚款、撤资等额外损失 |

## 边界情况

| 场景 | 预期行为 | 理由 |
|------|----------|------|
| 周刊勉强成刊但质量很差 | 允许发刊，并产生可学习的坏结果 | 支持失败叙事 |
| 同时命中多个势力任务 | 奖惩可叠加，但反馈必须分项展示 | 避免黑箱 |
| 零利润或亏损 | 继续进入下一周，但状态更差 | 保持长线张力 |
| 发刊造成重大现实偏移 | 必须在结算面板中显式展示 | 突出项目独特点 |

## 依赖

| 系统 | 方向 | 依赖性质 |
|------|------|----------|
| 编辑排版与发行策略 | 本系统依赖它 | 提供本期周刊 |
| 四大势力任务、干预与声望 | 双向依赖 | 读取任务后果并写回关系变化 |
| 宏观属性与现实偏移 | 双向依赖 | 消费结算结果并输出世界变化 |
| 周循环与状态 | 双向依赖 | 写回下一周基础状态 |

## 调参旋钮

| 参数 | 当前值 | 安全范围 | 提高后的效果 | 降低后的效果 |
|------|--------|----------|--------------|--------------|
| `base_reach` | 中 | 低到中高 | 周刊更稳定 | 早期更脆弱 |
| 声望扩散倍率 | 中 | 低到高 | 声望更重要 | 稿件质量更重要 |
| 惩罚成本强度 | 中 | 低到高 | 违规更痛 | 玩家更敢冒险 |

## 验收标准

- [ ] 发刊后会一次性返回销量、利润、势力后果和宏观属性变化。
- [ ] 结算只消费本期周刊结构，不反向改写稿件。
- [ ] 至少有一种现实偏移结果会被清楚展示给玩家。
- [ ] 势力任务成功和失败的后果可被明确区分。
