# 玩法设计原则

> **状态**：玩法规则设计指导草案  
> **创建时间**：2026-05-14  
> **上层来源**：[`core-experience.md`](./core-experience.md)  
> **用途**：把核心体验翻译成玩法设计准绳，尤其用于任务、骰子、黑骰、深度链、路线与反噬设计。本文不是具体数值规格；具体实现仍写入对应系统 GDD。

---

## 1. 玩法先产生决策，再承载叙事

规则首先要让玩家产生有意义的选择。不要把叙事拆成大量小骰面效果或小文本修正，导致机制本身没有玩法记忆点。

**判断题**：

- 玩家是否需要在两个都合理的选择之间取舍？
- 机制是否改变了玩家操作，而不只是改变结算文案？
- 去掉文案后，这个规则是否仍然好玩？

## 2. 普通任务负责日常循环，不强塞反噬

普通任务的职责是建立周刊经营的日常手感：看任务、派人、投骰、拿线索、进入内容生产。它们不需要局内反噬，也不需要每次都出现异常规则。

反噬应主要出现在：

- 深度链后段；
- 黑骰任务；
- 路线推进后的周开局事件；
- 发刊后反馈；
- 旧报道引出的新任务或地图变化。

## 3. 发现、接近、反噬要分层

玩法设计应区分三层：

| 层级 | 目标 | 适用内容 |
| --- | --- | --- |
| 发现未知 | 让玩家看见“不对劲”的线索 | 普通任务、早期线索、低风险探索 |
| 接近未知 | 让玩家投入资源继续深入 | 深度链、关键派遣、高难任务 |
| 反噬显形 | 让过去解释回到玩家面前 | 黑骰、中后期事件、现实偏移 |

不要把三层同时塞进每个任务。早期任务如果只完成“发现未知”，就是合格的。

## 4. 真相是解释权，不是单一答案

游戏不是传统探案。一个现象可以同时支持科学、神秘、阴谋、猎奇或民俗解释。玩法应允许玩家阶段性相信某个解释，并在后续通过新线索或世界反馈重新打开它。

**设计要求**：

- 重要谜题至少提供一个可暂时成立的解释。
- 深度链或后续任务应能留下新的疑点。
- 不要把所有答案都写成“唯一真相揭晓”。
- 也不要让玩家永远没有阶段性收获。

## 5. 路线完全由五维宏观属性表达

路线不是硬分支按钮，也不是独立等级条。科学、神秘、阴谋、猎奇、民俗等气质，应由 `公信 / 诡名 / 声望 / 守序 / 狂性` 的组合长期浮现。

**禁止倾向**：

- 新增“科学线等级”“神秘线等级”等与五维属性平行的路线条。
- 让玩家在开局一次性选定不可逆路线。
- 用文案声称路线变化，却不让五维属性、任务池或反馈变化。

## 6. 反噬是发布后的回响，不等于负面惩罚

反噬通常来自某个报道结果发布后，玩家选择的解释通过世界反馈回到玩家面前。它可以是负面影响，但更常见、更有味道的承载是：

- 新任务出现；
- 周开局新事件；
- 读者反馈、广告反馈、势力反馈；
- 随机信息或角色对白；
- 旧线索回响；
- 地图或区域状态变化；
- UI / 骰子 / 版面出现轻微异常表现。

数值惩罚只能作为其中一种手段。反噬的重点是让玩家感觉“我的解释正在回过头影响世界”。

黑骰局内结果不直接等同于反噬。黑骰在局内造成的是干扰、负面后果、任务失败 / 大失败或后续压力；只有当某个报道结果发布后，相关解释通过读者、广告、势力、新任务、开局事件、地图变化或现实偏移回到玩家面前时，才进入严格意义上的反噬。

## 7. 骰子多数时候是判定工具

普通骰子主要负责判定。它不需要承载过多叙事。玩家应能清楚看懂：哪些骰面有效、哪些被计入、需求是否达成、当前结果是什么。

骰子玩法可以有重投、锁定、计入、放弃、道具补骰等操作，但这些操作必须可读、可预测，并且服务当前任务目标。

角色骰判定 UI 中，`有效点 / 目标值 / 达标率` 是本次判定结果口径；任务属性条是贡献来源 / 需求拆解。两者不能做成同级并列目标。判定弹窗应先显示“本次是否过线”的结果摘要，再用次级结构解释哪些属性贡献了有效点。

## 8. 黑骰是异常介入规则

黑骰不是普通高难任务的数值加强版，也不是固定坏骰。黑骰应围绕关卡主题形成大颗粒玩法身份，让玩家感到“这个谜题正在改变本次判定”。

一般黑骰关卡的局内目标仍是凑够属性通关；玩家需要避免黑骰干扰，或围绕黑骰进行博弈和抉择。少数黑骰关卡可以拥有不同胜利机制，但不作为默认。

当前前期 demo 暂不展开黑骰时间分层；先聚焦前期黑骰的局内可玩性。下面的阶段只作为未来中后期设计参考：

| 阶段 | 目标 | 设计边界 |
| --- | --- | --- |
| 早期黑骰 | 异样接触 | 轻微干扰，后续小回响，不打满反噬 |
| 中期黑骰 | 规则开始异常 | 重投、锁定、计入等既有动作产生新意义 |
| 后期黑骰 | 报道回响关联 | 黑骰任务与已发布报道、新任务和世界反馈形成联动，但反噬仍以发布后的回响为准 |

## 9. 优先使用已有玩家动作

设计特殊机制时，先问能否用现有动作表达：

- 派遣；
- 投骰；
- 重投；
- 锁定；
- 计入；
- 放弃；
- 发刊；
- 压稿 / 选择报道方向；
- 查看反馈。

只有当现有动作无法表达核心玩法时，才考虑新增按钮或新系统。比如 330 公交车黑骰当前不优先新增“查票”按钮，而应尝试用重投、锁定和计入表达“鬼混入人群”。

## 10. 黑骰不能只是占用资源

如果黑骰只是占一个骰子、替换一个骰子或减少加成，而玩家显然不会选择它，它就只是资源税，无法表达“异常混入”。

黑骰应至少制造一种决策张力：

- 看起来有用，但可能危险；
- 重投可能暴露异常，也可能损失好结果；
- 锁定代表信任，但信错会留下后果；
- 避开异常更安全，但可能错过真正发现；
- 成功可能带来更深真相，也可能带来后续回响。

### 330 末班车当前规则方向

330 公交车黑骰重点是“鬼混入人群”。

- 黑骰最好一开始不可见，否则很难体现混入和试探。
- 鬼只附身在玩家角色骰上，暂不引入乘客骰或其他骰子。
- 鬼初始不可见，表现为附着在真实骰面之上；第一版决策阶段不直接给单颗骰子打鬼面标记，而通过本次未锁定重投组提示暴露鬼迹。若未锁定重投组只剩一颗骰，组提示自然等价于确认该骰。
- 鬼骰不伪装成高点数，因为玩家知道自己的角色骰面，突然出现异常高面反而会暴露“这是系统塞进来的东西”，削弱混入感。
- 重投承担试探工作，而不只是换结果；330 公交黑骰中，重投默认是除已锁定骰子之外，所有未锁定骰子一起重投，而不是逐颗指定重投。多次重投会提高撞见鬼面的概率，但也可能损失已经足够通关的好点数。
- 锁定代表“我相信这颗骰是真的”。
- 提示只针对刚刚被重投的未锁定骰组，回答这组当前结果是否出现鬼迹；不提示锁定组，也不在骰子上额外标注“已验 / 未验”。
- 属性达标但计入鬼骰时，一定应有反面效果，不应给优秀效果。
- 若最终计入鬼骰，应大幅降低点数，并造成对应角色的临时污染，影响后续该角色属性。
- 若出现新任务，应视为后续压力或展开，不作为局内正面奖励。
- 未来可以设计“目标就是找出某颗骰子”的特殊黑骰任务，但这不是默认胜利机制。

## 11. 内容后果应回到周刊循环

探索阶段得到的结果，应尽量回到周刊循环中发挥作用：

- 成为线索、现象、情报或报道角度；
- 影响本期内容选择；
- 影响五维宏观属性；
- 引出读者、广告、势力或地图反馈；
- 成为下一周任务或深度链的一环。

即使是黑骰和反噬，也应尽量通过周刊本职回到玩家面前，而不是把游戏切成另一种类型。

## 12. UI 与表现要让差异发生在决策现场

玩法差异不能只存在于后台或结算后文字。玩家应在做决策时看见差异：

- 任务入口提示特殊约束；
- 派遣页显示本次骰池和关键风险；
- 判定弹窗显示骰子阶段、锁定 / 重投 / 计入结果；骰子区必须像投骰主舞台，而不是被说明面板挤到页面下沿。
- 反噬回响通过读者反馈、新任务、地图或周开局事件出现；
- 黑骰的异常表现要发生在玩家理解规则的现场。

## 13. 玩法设计落地流程

新玩法进入实现前，按顺序确认：

1. 它服务核心体验的哪一层：发现、接近还是反噬？
2. 它是否仍然保持周刊经营者身份？
3. 它使用哪些既有玩家动作？
4. 它给玩家什么阶段性确定感？
5. 它留下什么后续裂口或回响？
6. 它如何影响五维宏观属性或任务池？
7. 它需要在哪个 HTML 原型中最小验证？
8. 如果有可见改动，如何截图验证？

若这些问题答不清楚，不应直接进入代码实现。
