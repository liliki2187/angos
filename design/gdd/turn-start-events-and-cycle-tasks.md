# 回合起始事件与周期任务

> **状态**：草案
> **作者**：Codex + 策划稿整理
> **最后更新**：2026-04-06
> **对应支柱**：四大势力必须改变你的发刊决定

## 概览

这个系统定义每周 Day 1 发生什么。它负责生成开场简报、抽取回合起始事件、发放本周周期任务，并把这些压力锁成一周内真实存在的约束。如果没有它，周循环就只是在重复派遣和发刊，而不是一份会被外部世界持续施压的危险刊物。

## 玩家幻想

玩家应感受到每一周开始时，世界都会先对这家周刊表态。你不是在真空里自由排班，而是在已有政治、学术、神秘和资本压力下做本周决定。

## 详细设计

### 核心规则

1. 每周开始时，系统先生成一条回合起始事件，再生成本周可见的周期任务集合。
2. 回合起始事件是短平快的周级输入，主要承担氛围、风险提示、资源扰动和轻量分支选择。
3. 周期任务是贯穿当周的硬约束或软引导，会直接影响探索、内容生产或发刊选择。
4. 周任务来源于四大势力与世界状态，候选集合受宏观属性、势力关系、区域进度和上周发刊结果共同影响。
5. 单周展示的任务数量必须受控。MVP 目标是让玩家每周看懂压力，而不是被任务墙淹没。
6. 未完成任务不会立即中断周循环，而是在发刊或周结算时转化为明确后果。

### 状态与切换

| 状态 | 进入条件 | 退出条件 | 行为 |
|------|---------|---------|------|
| `briefing_generate` | 新周开始 | 候选事件与任务生成完毕 | 计算回合事件、任务与提示 |
| `briefing_review` | 生成完成 | 玩家确认进入探索 | 展示本周世界态势与约束 |
| `task_active` | 玩家进入探索 | 本周发刊完成 | 任务作为周内持续约束存在 |
| `task_resolve` | 发刊完成 | 后果写回长期状态 | 结算完成、失败与延续影响 |

### 与其他系统的交互

- **周循环与状态** 提供周次切换入口，并保存本周活动事件与任务。
- **四大势力任务、干预与声望** 提供势力候选任务模板与关系阈值。
- **宏观属性与现实偏移** 提供事件与任务的解锁条件，并在结算后消费其结果。
- **全球地图、区域与节点可用性** 会读取本周任务，决定是否添加临时节点或封锁既有区域。

## 公式

### 本周任务数量

```
weekly_task_count = clamp(base_task_count + pressure_bonus, 1, 3)
```

| 变量 | 类型 | 范围 | 来源 | 说明 |
|------|------|------|------|------|
| `base_task_count` | int | 1-2 | 周配置 | 本周基础任务数量 |
| `pressure_bonus` | int | 0-1 | 状态计算 | 高压力周额外增加的任务数 |
| `weekly_task_count` | int | 1-3 | 计算值 | 本周实际展示任务数量 |

### 起始事件候选权重

```
event_weight = base_weight * macro_gate * reputation_gate * recency_penalty
```

| 变量 | 类型 | 范围 | 来源 | 说明 |
|------|------|------|------|------|
| `base_weight` | float | 0+ | 事件配置 | 事件基础出现权重 |
| `macro_gate` | float | 0-3 | 宏观属性 | 属性命中时提升候选概率 |
| `reputation_gate` | float | 0-3 | 势力关系 | 势力相关事件的附加权重 |
| `recency_penalty` | float | 0-1 | 运行时状态 | 避免相同事件连续重复 |

## 边界情况

| 场景 | 预期行为 | 理由 |
|------|----------|------|
| 没有势力满足任务条件 | 仍生成基础世界事件，任务可为空或退化为中性任务 | 保持周结构完整 |
| 四大势力都强烈施压 | 只展示 1-3 条最重要的任务，其余转成背景压力说明 | 避免信息过载 |
| 两条任务直接冲突 | 允许同时存在，并由发刊阶段承担取舍 | 这是项目张力来源之一 |
| 上周已经遭遇重罚 | 本周可降低硬任务权重，避免连续死局 | 保留恢复空间 |

## 依赖

| 系统 | 方向 | 依赖性质 |
|------|------|----------|
| 周循环与状态 | 本系统依赖它 | 提供周开始与周结束的生命周期 |
| 四大势力任务、干预与声望 | 双向依赖 | 生成任务并写回结果 |
| 宏观属性与现实偏移 | 双向依赖 | 读取门槛并写入世界反馈 |
| 全球地图、区域与节点可用性 | 它依赖本系统 | 消费任务生成的临时探索机会 |

## 调参旋钮

| 参数 | 当前值 | 安全范围 | 提高后的效果 | 降低后的效果 |
|------|--------|----------|--------------|--------------|
| `weekly_task_count` 上限 | 3 | 2-4 | 周压力更强 | 更易读但外部干预更弱 |
| 势力任务触发阈值 | 按关系档位 | 低到中 | 势力更频繁入场 | 更晚体现势力存在 |
| 起始事件强度 | 轻中度 | 轻到重 | 每周更不稳定 | 周起始更平 |

## 验收标准

- [ ] 每周开始都能生成一条可读的起始事件或简报输入。
- [ ] 四大势力任务会在同一套周起始流程中进入玩法。
- [ ] 冲突任务不会让周循环中断，而是转化为明确取舍。
- [ ] 任务结果会在发刊或周结算时写回长期状态。
