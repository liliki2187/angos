# HTML 实验页

本目录保存仍可直接在浏览器打开、但不属于正式玩法真源的实验页。

## 使用原则

- 这里的页面用于验证局部交互、规则假设或信息结构。
- 它们可以服务于 `design/gdd/` 的讨论，但不能替代正式 GDD。
- 如果实验规则被采纳，应先回写到正式设计文档，再决定是否继续保留实验页。

## 当前页面

- `black-dice-330-bus-lab.html`
  - 330 公交黑骰策略实验页，用于验证 5 名角色满编下，仅用锁定、批量重投和重投组提示，是否能形成“保点数 vs 换信息”的博弈；这是玩法实验，尚未套用到完整全链试玩。
- `dice-animation-lab.html`
  - 角色骰原地急旋实验页，保留判定弹窗原有信息布局，只把骰子判定区替换为 Canvas 不透明实体骰动效；文字和骰点以离屏面贴图映射到骰面，不再使用屏幕坐标标签。当前各方案全部沿用原方案一的快速节奏、短摇动和急停曲线，只比较原版纵轴、横轴变体、斜轴单向变体，以及先纵轴再横轴的换轴变体，尚未直接套用到完整全链试玩，也不依赖本地服务器或外部 3D 库。
- `dice-judgement-card-lab.html`
  - 角色骰判定选择阶段的交互实验页，只验证骰面卡片入口、3/5 名参判角色固定槽位、需求实时联动和提交前结果预览；不包含一键推荐、黑骰扩展或投骰动效，尚未套用到完整全链试玩。
- `dispatch-configuration-p0p2-lab.html`
  - 派遣配置候选排序实验页。2026-05-21 用户审阅后仅保留 `任务贡献`、`达标提升`、`未派遣优先` 排序标签，其它 P0/P1/P2 实验性布局已撤回；当前页面用于验证排序标签放在现有派遣页壳层内的效果，尚未套用到完整全链试玩。
- `dispatch-configuration-ux-fix-lab.html`
  - 派遣配置 UX 修正实验页，用于验证需求覆盖父子层级、候选角色 hover 后的达标率预览、禁用按钮原因、已选槽位移出、临时线人雇佣语义和真实显示模式切换；这是局部实验页，尚未套用到完整全链试玩。

- `synth-workbench-lab.html`
