# Godot 4.4-4.6 迁移关注点

最后核验：2026-04-09

> 说明：官方没有把 4.3 -> 4.6.2 的所有差异集中写成一份适合本项目直接消费的短表。这里按 Angus 目前最相关的 UI / GDScript / 资源工作流提炼出升级关注点，原始判断以 Godot 官方发布页与迁移指引为准。

## 4.4

| 项 | 官方信号 | 对 Angus 的影响 | 当前动作 |
|------|------|------|------|
| 升级前要读 migration guide | 4.4 发布页明确要求现有项目升级前阅读迁移指南 | 说明 4.3 -> 4.4 不是“无脑替换编辑器” | 锁版本时已把 4.4 视为需要核验的迁移点 |
| Universal UID support | 4.4 强化了 UID 工作流，并提供升级工具 | 资源移动、脚本 preload 与引用稳定性会受影响；首次用新版本打开或导出项目时，可能批量生成 `.uid` 文件 | 后续资源目录稳定后，优先采用 UID 友好的工作流，并在引擎迁移提交中接受合理的 `.uid` 文件生成 |
| Typed dictionaries | 4.4 正式引入 typed dictionaries | 对周状态和内容配置建模有价值 | 后续优先把需要强类型的映射结构迁到 typed dictionaries |

## 4.5

| 项 | 官方信号 | 对 Angus 的影响 | 当前动作 |
|------|------|------|------|
| 升级前要读 migration guide | 4.5 发布页继续明确要求阅读迁移指南 | 4.4 -> 4.5 仍然不是零风险小步升级 | 已纳入版本参考文档 |
| `duplicate_deep()` | 4.5 发布页明确指出旧 `duplicate(true)` 对数组、字典、资源的深拷贝语义并不总是可靠 | 周状态复制、模板数据复制、临时预览复制会更容易踩坑 | 当前项目优先使用显式重建；需要深拷贝时优先评估 `duplicate_deep()` |
| UID preload 工作流 | 4.5 开始支持在脚本中以 UID 方式更稳地预载资源 | 未来整理美术 / UI / 资源目录时有价值 | 当前先保留路径引用，等资源目录稳定后再迁 |

## 4.6

| 项 | 官方信号 | 对 Angus 的影响 | 当前动作 |
|------|------|------|------|
| `pivot_offset_ratio` | 4.6 为 `Control` 提供按比例定义 pivot 的方式 | UI 旋转 / 缩放不必再手写像素 pivot 修正 | 后续 UI 动效优先采用 |
| Clickable files in Output | 4.6 输出面板可直接点到报错脚本 | 提高本地调试效率 | 锁定 4.6.2 后可直接纳入标准工作流 |
| Required object params/results | 4.6 对原生 API 的 nullable 约束更明确 | 主要影响 GDExtension / 原生绑定，不是当前 GDScript 主阻塞 | 目前记为中长期注意项 |

## 当前结论

- 对 Angus 当前 sprint 而言，**真正重要的不是 4.6 新功能本身，而是版本统一**。
- 从 4.3 升到 4.6.2 后，最直接能立刻受益的是：
  - 更稳定的项目内版本锁定
  - 更好的输出面板调试体验
  - 更清晰的 Control pivot 工作流
- 升级后**仍然不能**使用 `Array[Array[String]]` 这类嵌套 typed collections。
