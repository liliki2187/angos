# 深度调查任务链 UI 与详细设计

> **状态**：设计草案  
> **日期**：2026-05-06  
> **用途**：定义“深度调查任务链”在地图、任务卡、结算与后续报道合成中的可视化与交互规则。  
> **关联采纳**：`docs/设计采纳记录.md` 中 A3、B1、B2、B3、B4。

---

## 1. 设计目标

深度调查不是“同一个任务反复刷”，而是一条在地图上逐步展开的连续任务链。玩家应该能清楚看到：

- 这是一条连续任务，不是普通临时点；
- 当前走到第几环；
- 哪些环已经完成；
- 下一环从哪里出现；
- 越往后越高危，最终可能进入黑骰任务；
- 玩家可以在关键节点收手，把已有素材带去报道合成。

---

## 2. 基本结构

一条深度调查链由多个任务环组成：

```text
起点任务
→ 深入任务 1
→ 高危任务
→ 黑骰终局任务（可选）
→ 链条完结
```

每一环都是一个独立任务，只做一次判定。  
探索阶段只负责产出素材、解锁下一环和记录风险；报道方向在探索结束后的专门界面处理。

---

## 3. 地图 UI

### 3.1 地图节点状态

深度调查链的节点至少有四种状态：

| 状态 | 地图表现 | 含义 |
|------|----------|------|
| 未发现 | 不显示或显示为模糊问号 | 玩家还不知道后续 |
| 当前可执行 | 高亮、有阶段编号 | 当前任务环 |
| 已完成 | 灰色 / 空心 / 打勾 | 已完成任务环，保留在地图上 |
| 已完结 | 整条链加完成标记 | 任务链已结束 |

### 3.2 连续关系表现

同一任务链应在地图上形成可读路径：

- 后续阶段尽量生成在上一阶段附近；
- 任务点之间可以用细线、虚线、脚印、剪报线、红绳或地图钉连接；
- 当前环突出显示，已完成环弱化但不删除；
- 未解锁环可以是虚线问号，不提前暴露完整名称。

### 3.3 示例

```text
[✓ 乘客口述] ── [✓ 废弃总站] ── [当前 无名站牌] ── [? 最后录像]
```

地图上玩家应一眼看出：这不是四个孤立任务，而是一条调查线。

---

## 4. 任务名称规则

任务名要有系列感，避免每环像完全无关的新任务。

推荐格式：

```text
事件名 · 阶段主题
```

例如 330 公交车：

```text
330 末班车 · 乘客口述
330 末班车 · 废弃总站
330 末班车 · 无名站牌
330 末班车 · 最后三秒录像
```

例如球状闪电：

```text
球状闪电 · 厨房火球
球状闪电 · 气象站记录
球状闪电 · 废弃变电所
球状闪电 · 复现实验
```

---

## 5. 侧栏任务卡

深度调查任务卡必须展示：

- 任务链名称；
- 当前阶段进度，例如 `第 2/4 环`；
- 当前任务类型：普通 / 高危 / 黑骰终局；
- 上一环简短结果；
- 当前环主要需求属性；
- 成功后是否解锁下一环；
- 是否可收手；
- 当前风险提示。

示例：

```text
330 末班车
第 3/4 环 · 高危任务

上一环：废弃总站发现“撕掉的调度页”。
当前目标：调查地图上不存在的站牌。
成功后：可能解锁最终环“最后三秒录像”。
风险：失败可能给队员临时 debuff。
```

---

## 6. 任务配置页

配置页应突出三类信息：

### 6.1 连续任务身份

在概率 / 风险预览前显示：

```text
深度调查任务链 · 第 3/4 环
本任务完成后会推进调查链，旧任务环将保留为灰色记录。
```

### 6.2 本环属性需求

按具体属性显示，不只显示功能域：

```text
需要：洞察3 + 胆识2
相关角色骰面：洞察 / 胆识 / 诡思
```

### 6.3 后续提示

不剧透真相，但说明结构：

```text
成功后：解锁下一环
失败后：可能保留弱线索，但调查链风险上升
```

---

## 7. 结算 UI

每环结算需要同时反馈“本环结果”和“链条状态变化”。

### 7.1 成功

```text
本环成功：发现“无名站牌”的旧线路编号。
调查链推进：地图上出现下一环“最后三秒录像”。
```

地图反馈：

- 当前环变灰 / 打勾；
- 下一环出现并高亮；
- 连线延伸。

### 7.2 失败

普通失败可有几种处理：

- 仍推进，但下一环风险增加；
- 不推进，保留本环可重试；
- 给弱线索，并提示“可收手进入编辑部”；
- 红色/限时链环失败则关闭该分支。

具体规则按任务设计，但 UI 必须清楚写出后果。

### 7.3 链条完结

最终环完成后：

```text
深度调查完结：330 末班车调查链已归档。
你获得了可用于报道合成的素材组。
```

地图反馈：

- 整条链显示完成；
- 当前任务不再可派遣；
- 可在档案 / 任务记录中回看。

## 8. 高危任务与黑骰任务在链中的位置

### 8.1 高危任务

高危任务通常位于深度链后段，也可以直接出现在地图上。

特点：

- 难度较高；
- 失败后果更重；
- 可能带 debuff；
- **没有黑骰特殊机制**；
- 仍是简单判定。

### 8.2 黑骰任务

黑骰任务通常是深度调查最终环，也可由特殊事件、主线或临时任务触发。

特点：

- 使用关卡专属黑骰机制；
- 结果为大成功 / 成功 / 失败 / 大失败；
- 大失败会给较重当局后续 debuff；
- 任务卡必须提前说明黑骰规则。

---

## 9. “再追一次？”的位置

“再追一次？”不是普通任务常驻按钮，而是深度调查链中的临时事件选择。

常见触发：

```text
倒数第二环完成后，玩家已获得足够素材，但仍有关键疑点。
```

玩家选择：

- **收手**：带着当前素材进入报道合成；
- **再追一次**：消耗资源，进入黑骰终局任务。

UI 示例：

```text
临时事件：录像里还有三秒

你已经可以写稿了，但技术员说空白段里也许藏着东西。

[收手，把当前材料带回编辑部]
[再追一次？消耗 1 行动点，进入黑骰任务]
```

## 10. 数据字段建议

```ts
{
  chainId: "bus_330",
  chainTitle: "330 末班车",
  chainStage: 3,
  chainStageTotal: 4,
  chainState: "active", // hidden | active | completed | failed | archived
  previousStageId: "bus_330_depot",
  nextStageId: "bus_330_tape",
  isHighRisk: true,
  isBlackDiceTask: false,
  completedStageIds: ["bus_330_witness", "bus_330_depot"],
  mapLinkStyle: "thread",
}
```

字段原则：

- `isHighRisk` 与 `isBlackDiceTask` 分开；
- 不要用 `riskTier: high` 暗含黑骰；
- 任务链状态和任务本身状态分开记录。

---

## 11. 验收标准

- 玩家在地图上能看出深度调查是一条连续任务链。
- 已完成任务环不会完全消失，而是灰化保留。
- 当前任务环有明显高亮和阶段编号。
- 任务卡能说明上一环结果、当前目标和下一环解锁关系。
- 高危任务和黑骰任务在 UI 文案和机制上不混用。
- “再追一次？”明确表示进入黑骰任务，而不是普通补考。
- 链条完结后玩家能回看这条调查线已经完成。
