# 头像裁图与接入通用规范

> 适用范围：本仓库所有“角色头像卡”类场景（探索配队、悬浮卡、队伍列表等）。  
> 目标：让任何一次换图都能稳定产出“头部完整、构图一致、无拉伸、可直接接入”的结果，避免反复人工校对。

---

## 1. 设计目标与优先级

每次裁图必须按以下优先级决策（从高到低）：

1. **头部完整**：头顶不可贴边，不可裁掉发髻/帽檐/耳朵等关键头部信息。  
2. **脸部居中**：以“脸部视觉中心”为中线，不以整图几何中心为准。  
3. **禁止拉伸**：只允许等比裁切 + 等比缩放。  
4. **卡内信息饱满**：避免明显无效留白（尤其底部大空白）。  
5. **主体优先**：身体可牺牲，背景、边缘装饰、文字可牺牲。  

---

## 2. 输出与显示规格（统一）

### 2.1 输出文件

- 格式：`PNG`
- 尺寸：`384 x 512`（固定 3:4）
- 路径：`Assets/avatars/`
- 命名：`s{序号}_{英文或拼音标识}.png`

### 2.2 前端显示（头像卡）

- 容器比例：`3 / 4`
- 显示策略：`object-fit: cover; object-position: center top;`
- 禁止：
  - `object-fit: fill`
  - 非等比缩放或二次拉伸

---

## 3. 构图硬约束（必须满足）

### 3.1 头顶留白

- 头部最高点到图像上边缘保留约 `4% ~ 8%` 画高留白。  
- 不允许“头贴上沿”。

### 3.2 脸部中线

- 脸部中心 x 与头像中心 x 的偏差建议不超过 `±4%` 画宽。  
- 对偏心构图素材，优先移动裁切框，而不是缩小主体。

### 3.3 主体占比

- 头像以“头 + 上半身”为主。  
- 若素材是全身图，允许裁掉腿脚换取头部与上半身信息密度。

### 3.4 文本与背景

- 源图边缘文字可保留，但不得压过脸部或抢主体注意力。  
- 若文字与主体冲突，优先保主体构图，必要时裁掉文字区域。

### 3.5 特殊角色例外（项目规则）

- `薛定谔`：必须同时保留“完整猫 + 人头/真人上半身”，且猫尽量接近中线。

---

## 4. 执行流程（每次替换头像都走）

1. 在 `Assets/pic/` 收集源图。  
2. 先确定每张图的“脸部中心”和“头顶留白”。  
3. 按 3:4 裁切框导出到 `Assets/avatars/`（384x512 PNG）。  
4. 在游戏内目标界面实机查看（不是只看导出图）。  
5. 按验收清单逐条打勾。  
6. 通过后再提交。  

---

## 5. 验收清单（发布前必须全绿）

- [ ] 全部头像为 `384x512`，比例一致  
- [ ] 无拉伸、无变形  
- [ ] 头顶有留白，不贴边  
- [ ] 脸部视觉居中（允许小偏差）  
- [ ] 卡内无明显底部大块空白  
- [ ] 主体清晰，背景/边缘元素不喧宾夺主  
- [ ] 特例角色（如 `薛定谔`）满足专门规则  
- [ ] 在真实 UI 中看起来一致（不是仅在图片查看器里一致）

---

## 6. 常见失败模式与修复

### 失败模式 A：人物看起来“总是偏右/偏左”

- 原因：按整图中心对齐，而非脸部中心。  
- 修复：以眼睛连线中点/鼻梁中心为基准重新平移裁切框。

### 失败模式 B：头顶被切

- 原因：裁切框 y 过低或后续二次 cover 裁掉上边缘。  
- 修复：上移裁切框，保留 4%~8% 头顶留白；确认显示为 `center top`。

### 失败模式 C：下方出现大空白

- 原因：`contain` 导致整图塞入容器且主体偏小。  
- 修复：改为 `cover` 并提高主体占比；必要时重新裁切上半身构图。

### 失败模式 D：看起来有拉伸

- 原因：输出尺寸和裁切比例不一致，或显示端用了 fill。  
- 修复：裁切阶段严格 3:4，显示端使用 `cover`。

### 失败模式 E：多角色拼图串格

- 原因：源图是 2x2/多姿态图，裁切框跨入邻格。  
- 修复：先锁定目标姿态格边界，再做 3:4 裁切。

## 7. 参数模板（建议长期维护）

建议在后续新增一个参数表（如 `docs/avatar-crop-params.json` 或本文件附录），记录：

- `source_file`
- `target_file`
- `crop_x`, `crop_y`, `crop_w`, `crop_h`
- `notes`（如“头顶留白优先”“允许保留文字”“猫需居中”等）

这样可实现：

- 换图时快速复用
- 版本迭代可追溯
- 减少主观反复微调

---

## 8. 变更约束建议

- 若只换头像素材：优先只改 `Assets/avatars/*`，避免误改玩法逻辑。  
- 若调整显示样式：必须同步回归已有头像，确保不会引入“二次裁切”问题。  
- 大批量替换后，建议至少在以下界面回归一次：  
  - 配队卡列表  
  - 悬浮人物卡（如有）  
  - 任务结果页人物缩略图（如有）

---

**维护建议**：每次确认一轮可用头像后，把参数与示例截图一并补到文档，形成仓库内可复用的“头像资产工艺标准”。
