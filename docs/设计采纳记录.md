## 一、已采纳 · 通用机制

### A1. 任务分白色调查 / 红色截稿
- **来源**：2026-04-25 骰子机制讨论 → 借鉴自 Disco Elysium。
- **机制**：所有任务在数据上多一个枚举字段 `checkType`：
  - **白色调查**：可被多次派遣，结果会**累积揭示**信息（与现有 `topicKey` 周内揭示机制配合）。适合都市怪谈、连载、可反复挖的题材。
  - **红色截稿**：仅触发一次的判定，**失败即永久关闭**（不再出现在线索池），氛围"周五拍板就决定输赢"。适合临时事件触发的限定任务。
- **应用范围**：通用，影响所有线索/任务的设计。
- **用户原话备注**：「这个设计很好，某些特殊任务类型，特别是一些临时出现的任务或事件触发的任务，失败就不会再出现了。」
- **已修订（2026-04-28）**：白色调查与红色截稿可直接落地到游戏内；首轮试玩中，红色截稿优先作为**红色临时情报任务**，可新增临时关卡，或与已有合适临时关卡（如北美禁区带的限时异常任务）合并。
- **已修订（2026-05-06）**：红色截稿的关闭条件明确为：**成功、失败或超过截止日期后都会消失 / 关闭**。失败关闭不是唯一关闭条件。

### A2. 周内"临时人卡"机制（雇佣兵 / 临时线人 / 雇佣记者）
- **来源**：2026-04-25 骰子机制讨论 → 借鉴自 Slay the Spire / Inscryption 的局内构筑。
- **机制**：周内可获取**临时人物卡**：临时线人卡、雇佣记者卡等。仅在本周有效，周末清算（沿用 `cleanupWeekScopedEffects`）。功能上类似"临时骰面贴纸"——派遣时与正式员工一起加入判定。
- **取得渠道**：事件 / 任务奖励 / 主动雇佣（消耗资源）。
- **应用范围**：通用，作为周内策略的灵活补充手段。

### A3. 线索是神秘素材，最终解释由编辑发挥
- **来源**：2026-04-28 黑色幽默小报基调讨论。
- **机制**：调查发现的线索依然应该是神秘、有趣、可报道的素材，但不一定揭示真实原因和最终结果。玩家带回的是证词、照片、雷达记录、档案矛盾、异常残留等“素材”，最终如何解释、包装、掩盖或夸张，交由编辑与报道系统决定。
- **势力取舍**：不同势力会希望玩家把同一素材导向不同解释，例如政府希望把 UFO 真相报道成普通飞行事故；小报读者、神秘学社群或商业流量则可能希望报道更煞有介事。
- **高危例外**：部分深度任务、黑骰任务或额外代价任务可能真的触及让人掉 SAN / 狂性上升的神秘真相，但这应是高风险、高代价、少量出现的内容，不替代日常素材采集的灰色解释空间。
- **已修订（2026-04-29）**：同一事件的报道方向不应平铺展示，而应跟随探索递进逐层展开：初期线索开放初级报道方向；深入发现会升级、反转或新增方向；黑骰 / 高危终局才开放让人掉 SAN 的危险真相方向。
- **应用范围**：通用，影响探索线索、合成编辑、势力任务、深度调查与黑色幽默叙事基调。

### A4. 角色专精 + 骰面数值的判定成长方向
- **来源**：2026-04-30 通用骰子与角色成长讨论。
- **机制**：采用“角色专精 + 骰面数值”作为后续角色骰方向。角色不再主要依靠苏丹牌式多属性数值直接增加骰子数量，而是通过专精（如科学、神秘、采访、现场、潜入、编辑等）决定其专属骰面能产出什么调查价值；升级后可以提升某些骰面的数值，或增加 / 改变骰面。
- **角色规模取舍**：相比苏丹牌，本项目不需要大量角色来推进剧情。苏丹牌依靠多角色群像与人物事件推进，本项目更依赖特殊事件、现象、深度任务和报道解释推进，因此可以接受角色数量更少，但每个角色骰面更有辨识度。
- **骰面插槽边界**：骰面插槽可以作为特殊道具、卡牌或事件结果加入，但不作为所有角色的普遍升级选项。常规成长以专精骰面数值提升和骰面替换为主，插槽用于制造少量强记忆点和临时构筑感。
- **已修订（2026-05-06）**：不修改原有 10 项个人属性，不把 `探索 / 生存 / 洞察 / 想象 / 理性 / 笔力 / 胆识 / 诡思 / 人脉 / 造势` 直接改名为新专精。后续以功能域组织这些属性：外勤域（探索、生存、胆识）与认知域（洞察、理性、诡思）为探索判定常见属性；内容域（笔力、想象、造势）服务写作、报道与局前特殊事件，部分探索任务也可使用；关系域（人脉）服务势力、采访和线索追查，部分线索任务与其关联度较高。
- **已修订（2026-05-06 · 角色骰规则细化）**：主干方向改为角色骰新路线，旧属性加总路线作为分支/历史方案保留。首版骰面暂为单一属性数值，特殊角色后续可有复合属性面。任务需求按具体属性写，特殊关卡可有属性替代折算。普通任务只看是否达标；黑骰任务使用大成功/成功/失败/大失败，大失败给较重的当局后续 debuff。普通任务最多 3 人，高危任务最多 5 人；临时人物卡算完整角色骰并纳入人数限制；道具骰、黑骰、支援骰计入总骰数但道具骰不纳入人数限制。每次角色升级必须改骰面，沿固定或半固定分支成长路线选择，可能是数值提升、增加骰面或替换骰面；骰面数值上限暂不设，允许 4 点以上。特殊插槽只来自道具/卡牌/事件，不作为普遍升级项。普通任务没有风险面；风险面主要由黑骰失败 debuff 赋予个别角色，个别角色强力技能可自带风险平衡。普通判定需要选骰，但应有便捷按钮跳过全部选骰；个别特殊面、高危任务或 debuff 可禁止重投。道具卡均为一次性；临时人卡仅本周有效，成本较高或来自周前特殊事件的正确选择及代价。概率预览只需低/中/高风险；角色卡只显示属性专精，详情页显示完整 6 面，升级页显示新旧 6 面变化，任务配置页突出本任务相关面。
- **已修订（2026-05-08 · 高危降为危险标识）**：高危不再作为独立任务类型或独立人数规则；非黑骰任务统一最多 3 人。`riskTier == high` 仅作为“危险标识 / 高难度标记”，用于提示需求更高、失败后果更重或可能带来 debuff。黑骰任务仍是独立特殊入口，可使用更高人数上限与四档结果。
- **已修订（2026-05-11 · 风险预览口径）**：任务配置页整体风险应按任务需求覆盖度与缺口判断，不再用“相关骰面 / 总骰面”比例直接判定。加入低贡献角色不应让整体风险反向升高；低贡献角色通过“无相关面 / 占用人数槽 / 贡献低”提示给玩家。
- **已修订（2026-05-13 · 达标率口径修正）**：角色骰首版中，多属性任务需求表示“哪些骰面是本任务相关有效点”，不再要求每个属性在单次掷骰中逐项硬达标。普通任务达标改为：已计入的相关骰面有效点达到任务目标值；当前 HTML 原型目标值为 `max(1, 需求属性数, ceil(需求总值 / 3))`。风险预览必须基于同一结算公式计算实际达标率，不能用整套骰面池的潜在覆盖冒充“低风险”。
- **已修订（2026-05-14 · 首周可玩难度修正）**：进一步修正 2026-05-13 的口径：只改风险显示不够，若适配 3 人队仍只有约 15% 达标率，本质是难度不可玩。角色骰目标值改为 `max(需求总值 >= 4 ? 2 : 1, ceil(需求总值 / 3))`，不再把需求属性数量作为硬下限。多属性需求应扩大相关骰面范围，而不是天然压低成功率；首周红色截稿“突发：雷达异常光点”从 `探索5 / 生存3` 下调到 `探索2 / 生存1 / 胆识1`，后续高危链条中明显超调的“废弃雷达站夜访”从 `探索4 / 洞察4 / 生存2` 下调到 `探索3 / 洞察3 / 生存2`。首周红色截稿 / 高危机会需要保持可搏，适配队伍不应落入几乎不可能成功的区间。
- **已修订（2026-05-14 · 任务档位难度分层）**：普通 / 常驻任务应是最低压力层，不应以 3 人满编作为最低可行门槛。普通任务以 1 到 2 名适配角色可玩为基准；3 人应带来稳妥或提效。线索追查和深度链前段以 2 人可玩、3 人明显变稳为目标；红色截稿、高危和黑骰可以要求更多资源，但额外难度应来自时限、失败后果、黑骰反噬或资源占用，而不是把基础成功率压到玩家不想点击。该原则已落地到当前 HTML 原型任务数值：51 区外围公路、神农架、复活节岛等普通任务下调到 1 到 2 人可玩；罗斯威尔、330、球状闪电前段链条下调为 2 人可推进；红色截稿保留中风险并通过道具拉高胜率。
- **应用范围**：通用，影响角色成长、派遣判定、黑骰交互、临时人卡 / 道具设计和后续 UI 表达。

### A5. 角色骰与任务入口的感知层落地
- **来源**：2026-05-08 三个界面表现层讨论。
- **机制**：前三个界面的后续落地应从“标识和说明”升级为“玩家操作真的不同”。任务配置页以“本次骰池预览”为核心，让玩家感知自己正在构建骰池；角色骰详情要采用更像骰面展开的表现方式，参考 `Slice & Dice` 中骰子六面展开的认知模型，让玩家能把角色能力直接联想到“骰子”；任务入口按真正改变操作的类型组织，降低类型标签堆叠带来的认知负担。
- **已修订（2026-05-08 · 首轮落地口径）**：首区任务入口按白色调查、红色截稿、深度链、黑骰入口组织；危险只作为任务卡与地图点上的感叹号 / 高难标记，不再和这些入口并列为任务类型。
- **已修订（2026-05-09 · 深度链地图入口交互）**：深度连续任务在地图上不再完整展示所有链路节点，也不保留灰色旧环；地图只显示当前最新任务入口。入口采用规整叠层任务页：默认态提示“多个任务合一”，hover 只轻微预展开，点击后同一组卡片展开为线性链，只显示当前环与下一条 `?` 线索。
- **已修订（2026-05-11 · 地图标签避让）**：区域地图中的任务点、线索点与可展开深度链入口需要渲染后避让机制。深度链、黑骰、高危入口优先保留原位；普通任务和线索标签让位，避免动态任务或长标题互相压盖。复查后补充：该机制必须按默认态、hover 态、点击展开态，以及宽 / 中 / 窄视口验收，避免只在单一桌面截图中看似成立。
- **已修订（2026-05-13 · 竖屏手机截图口径）**：当前阶段暂时不考虑竖屏手机端使用；常规原型 / UI 回归后续不额外制作竖屏手机或极窄长页截图，以节省时间和对话负载。只有用户明确要求，或任务本身就是移动端 / 窄屏适配问题时，再做对应截图。
- **已修订（2026-05-14 · 派遣角色骰视觉语法）**：派遣配置页里只要展示完整 6 面角色骰，候选角色卡、已选角色槽和角色详情都应优先使用骰子立方体摊开的长尾十字架 / dice net 结构。横向 chip 行只能作为极小摘要、日志或一次性道具简表，不能替代角色骰的核心展示。压缩候选卡高度时也不得牺牲“看起来像骰子”的核心感知。当前 HTML 原型已将新版派遣配置的候选角色卡和已选角色槽改回 dice net；一次性道具仍保留横向 chip 摘要。
- **应用范围**：通用，影响首区任务总览、普通任务配置页、角色详情弹窗、后续黑骰与道具骰表现。
- **状态**：已采纳为 UI / 感知层落地方向；深度链地图入口已形成初步满意方案并落地到 HTML 原型。

### A6. 全球地图选择页主操作与选中区域详情
- **来源**：2026-05-13 全球地图选择 UI 实验确认。
- **机制**：全球地图先选区域，再通过主按钮“进入选中区域”。右侧从平铺所有区域改为选中区域详情；地图点与右侧区域小卡 hover / click 联动；详情用固定槽位展示可见目标、限时机会、线索、深度链；锁定区域仍可选中预览，但显示进度 / 缺口并禁止进入；“下一天”在全球地图降级为次级操作。
- **已修订（2026-05-14 · 地图点角标安全区）**：全球地图点上的进度百分比、事件数量、状态徽章等不能以未预留空间的右上角绝对定位遮挡区域名称。角标必须拥有独立布局槽位或明确安全区，并在截图验收中检查标题、角标和按钮文字的矩形不重叠。该原则同样适用于区域地图、任务卡、头像组和任何高密度按钮 / 卡片角标。
- **已修订（2026-05-14 · 地区选择与任务预览分离）**：全球地图页的真实功能是选择取材地区并进入地区页，因此右侧不能铺成可点击任务列表，也不能复用区域任务卡语法。右侧应由地区选择器、进入按钮、地区概况数值和少量只读情报预览组成；具体任务选择只在进入地区后发生。只读预览默认 2 到 3 条，不放任务操作按钮，不使用 `interactive` 交互类。
- **已修订（2026-05-14 · 主进入按钮右下角强化）**：全球地图页的主操作是进入选中地区，按钮必须作为流程终点放在右下角，并使用足够大的按钮与箭头图标。右上角不承担核心进入动作；`下一天` 等次级动作在视觉重量上必须低于主进入按钮。
- **已修订（2026-05-14 · 全球地图不推进时间）**：全球地图是地区选择入口，不显示 `下一天`。时间推进放在当前地区任务派遣 / 区域任务上下文；推进到第二天或完成到期结算后，默认回到当前地区继续派遣。本阶段暂不支持 7 天内换地区，区域页不提供返回全球换区入口；后续若支持换区，应作为明确系统设计成本与限制。
- **应用范围**：通用，影响全球地图、区域选择、探索入口与 UI 感知层。
- **状态**：已采纳，并已落地到 HTML 完整全链试玩。

### A7. 派遣配置页按“本次骰池”重构
- **来源**：2026-05-13 派遣配置 UI 重构讨论，用户确认直接在本体内推进并保留快速回退口子。
- **机制**：派遣配置页应围绕“本次骰池”组织，而不是平铺角色属性表。顶部固定展示任务摘要、耗时、剩余、截止与需求；中部展示已选角色槽、道具骰槽、需求覆盖、缺口与风险；下方候选角色按本任务贡献排序，hover / click 需要能即时反馈加入后会补哪些需求。临时资源与一次性道具作为补骰池手段展示，不替代角色选择。配置页主操作是“开始调查 / 开始线索调查”，全局“下一天”不应在该页作为浮动按钮抢焦点。
- **已修订（2026-05-13 · 任务摘要信息层级）**：派遣配置页顶部任务信息不能把不同层级的信息做成并列模块。任务耗时不应单独占一个 UI 模块，应并入任务标题 / 行动按钮 / 轻量 meta；探索周剩余时间是全局资源，不是任务属性，不应放在任务相关数值组里；截止日期是红色截稿 / 突发任务的特殊限制，应作为特殊状态提示或警戒条处理，而不是和常规耗时、需求并列。任务摘要区应只保留玩家做派遣决策真正需要的任务身份、类型、关键需求与特殊约束。
- **已修订（2026-05-13 · 截稿剩余时间视觉主语）**：红色截稿在派遣页不应同时显示“红色截稿”标签和红色说明条。剩余时间是该特殊约束的核心数值，应使用单一高权重倒计时块呈现，例如 `截稿关闭 / 剩余 3天`；成功 / 失败后关闭、失败伤员等后果说明降级为一行文字。
- **应用范围**：通用，影响派遣配置、角色骰预览、临时资源、风险反馈和后续 Godot UI 落地。
- **状态**：已采纳为 UI 落地方向，并已落地为 HTML 完整全链试玩默认派遣配置；旧版配置页保留为 `dispatchold=1` 回退 / 对照入口。

### A8. 角色骰判定弹窗的阶段化表现与计入反馈
- **来源**：2026-05-13 角色骰判定 UI 反馈；用户确认当前方向“没有大问题”，要求继续下阶段落地。
- **机制**：角色骰判定弹窗应恢复明确的摇骰表现，并按“基础骰摇骰 → 黑骰介入 → 停骰选择 → 结果收束”的阶段表达。选择阶段需要用固定槽位显示已计入骰面、需求进度、黑骰压力、任务外骰面与当前骰面影响，按钮文案实时提示达标 / 未达标，减少玩家靠说明文字猜规则。
- **应用范围**：通用，影响角色骰判定、黑骰任务、后续道具骰 / 支援骰表现和 Godot UI 落地。

### A9. 区域地图与任务列表的已派遣状态表达

- **机制**：已派遣状态要表达“任务正在被谁占用”。区域地图任务点右上角直接显示派遣角色小头像组；右侧任务卡把“已派遣”放在任务名称右侧作为当前状态，不放入耗时 / 危险 / 对手骰等任务属性标签行。

### A10. 核心乐趣以“发现未知 / 未解之谜”为主，周刊是组织形式
- **来源**：2026-05-14 核心乐趣与黑骰设计讨论。
- **机制**：项目核心魅力不是“报刊经营”或“报道刊登选择”本身，而是发现未知、接近未解之谜、与异常现象打交道。周刊是把分散的未解之谜主题聚拢起来的组织形式；故事解释、报道选择与刊登更偏向增加可玩性、复玩性和形式感的手段，不应反客为主成为核心乐趣。
- **应用范围**：通用，影响骰子检定、黑骰、探索任务、深度链、内容生产链和后续 GDD / 原型设计的优先级判断。
- **状态**：已采纳为核心乐趣定位修正；后续玩法设计应优先服务“发现未知”的体验，再考虑周刊组织与刊登层的表达。

### A11. 核心乐趣分层：发现、接近、反噬，由周刊串联
- **来源**：2026-05-14 核心乐趣分层讨论。
- **机制**：核心体验可按三层组织：第一层是发现型，第二层是接近型，第三层是反噬型；周刊组织型负责把三者串起来。反噬需要控制使用强度，正常任务不必加入反噬；多数反噬应落在局外，尤其是不同报道方向产生的后果。少数复杂任务，特别是黑骰任务，可以在局内体现一部分异常或反噬乐趣。
- **应用范围**：通用，影响探索任务、深度链、黑骰任务、报道方向反馈、周刊结算与长期现实偏移的设计边界。
- **状态**：已采纳为核心乐趣结构与反噬强度控制原则；具体真相结构、路线代价和周刊反噬强度仍待后续细化。

### A12. 游戏不是探案，而是解释现实并让认知反过来塑造现实
- **来源**：2026-05-14 核心真相结构讨论。
- **机制**：项目不应按传统探案游戏处理“唯一真相”。更合适的定位是“解释现实的游戏”：同一异常现象可以被解释为普通光学现象、UFO、政府阴谋、神秘征兆或其他叙事；玩家选择的解释会影响公众认知，而公众认知可能通过类似皮格马利翁效应 / 自证预言的方式让事物逐渐真的出现。科学解释路线也不应等同于安全或绝对真相，持续深入后同样可能遇到无法解释的边界，并导向不可知性与 SAN / 认知代价。
- **应用范围**：通用，影响未解之谜结构、报道角度、路线差异、现实偏移、科学 / 神秘路线设计和长期反噬逻辑。
- **状态**：已采纳为核心玩法定位；后续设计应避免把谜题全部写成“找到唯一答案”的探案结构。

### A13. 玩家认知曲线：前期清楚操作目标，后期逐步意识到自己在形塑事实
- **来源**：2026-05-14 核心体验曲线讨论。
- **机制**：游戏前期应让玩家清楚自己在做什么：可以以为自己是在刻意编造耸人听闻的故事赚钱，也可以以为自己是在探寻真相、寻找事情最真实的样子。随着不同路线推进，玩家逐步发现自己的报道、解释和持续相信正在形塑事实；到后期，客观认知会被路线经历逐渐动摇甚至崩溃。
- **应用范围**：通用，影响新手体验、任务文案、报道方向、路线推进、现实偏移、SAN / 认知代价和中后期反噬节奏。
- **状态**：已采纳为玩家体验曲线原则；前期不应让玩家过早陷入“我完全不知道系统想表达什么”的混沌感。

### A14. 确定感节奏：阶段性确认与重新扑朔迷离交织
- **来源**：2026-05-14 玩家确定感与未解之谜体验讨论。
- **机制**：玩家体验不应走向“全部确定”或“全部不确定”。合理节奏是：玩家阶段性以为自己探寻到真相，随后发现无法解释的异常；继续深入后再次恍然大悟，获得新的确定感；再过一段时间，通过关联故事、路线联动或现实偏移重新变得扑朔迷离。确定感负责给玩家掌握感和推进动力，不确定感负责保留未解之谜的吸引力。
- **应用范围**：通用，影响深度链结构、任务后续解锁、报道方向、跨故事联动、现实偏移反馈和核心悬念节奏。
- **状态**：已采纳为核心悬念节奏原则；设计时应避免全程糊成谜语，也避免过早把谜题彻底钉死。

### A15. 周刊经营始终是日常本职，现实改写是后期异常挑战
- **来源**：2026-05-14 周刊玩法与现实改写边界讨论。
- **机制**：周刊始终应作为“解释真相”的经营类玩法存在，玩家的日常本职仍是调查、组织解释、经营周刊和完成每周内容。现实被改变 / 真相被形塑更像经营类游戏后期遇到的外部挑战与异常压力，而不是把核心类型切换成“现实改写游戏”。类似饭店经营后期遭遇黑社会：玩家仍主要在经营饭店，正因为日常本职与异常挑战形成错位，神秘感才更有吸引力。
- **应用范围**：通用，影响内容生产链、周刊结算、现实偏移、反噬强度、后期事件和类型边界控制。
- **状态**：已采纳为玩法边界原则；后续设计应避免让现实改写层过早或过强地吞没周刊经营日常。

### A16. 路线选择是软性长期倾向，由五宏观属性承载
- **来源**：2026-05-14 科学 / 神秘等路线差异讨论。
- **机制**：科学、神秘、阴谋、猎奇、民俗等解释路线不应首先做成硬分支按钮或一次性阵营选择，而应作为软性的长期倾向。玩家通过持续调查、报道与经营选择，推动 `公信 / 诡名 / 声望 / 守序 / 狂性` 五项宏观属性形成不同组合；玩家可以大致知道自己在往哪个方向走，但不必被过早锁死。
- **已修订（2026-05-14 · 路线完全由五维属性表达）**：前期玩家可以同时从“做耸人听闻的小报赚钱”和“认真探寻未解之谜真相”两种动机进入；后续通过路线引导和反馈让玩家意识到五维属性会决定报刊的报道路线。路线表达应完全由 `公信 / 诡名 / 声望 / 守序 / 狂性` 承担，避免额外新增“科学线等级”“神秘线等级”等独立路线条。
- **应用范围**：通用，影响报道角度、路线气质、宏观属性、现实偏移、区域/事件解锁和长期结局分支。
- **状态**：已采纳为路线结构原则；具体路线气质候选先进入机制库备查，不作为正式规则，见 M3。

### A17. 反噬边界：前期只露草灰蛇线，深层任务逐步显形
- **来源**：2026-05-14 反噬边界讨论。
- **机制**：反噬不应在游戏前期作为明牌主玩法出现，前期最多让玩家体会到一点草灰蛇线和轻微异样。黑骰任务本身要求更深的探索层级，因此其最终真相可以带有一定反噬影响力；但前期黑骰任务与后期黑骰任务的反噬强度、影响范围和可感知程度应明显不同，不能一开始就把终局级异常压力打满。
- **已修订（2026-05-14 · 反噬不等于负面惩罚）**：反噬不能简单理解为玩法中的负面影响。它可以体现为剧情变化、新任务出现、旧线索回响、世界状态偏移、角色认知变化或纯表现层异样；重点是让玩家产生“自己的解释正在回过头影响世界”的感受，而不是每次都扣资源、扣属性或加惩罚。
- **已修订（2026-05-14 · 反噬常见承载）**：前期草灰蛇线可以只是读者反馈、一日过去后的随机对白 / 信息，也可以是新任务或一定程度的地图变化。黑骰或其他任务的反噬影响，最常见的承载应是新任务、周开局新事件、读者反馈、广告反馈、势力反馈（任务、奖惩、关系变化）和类似随机信息的文本。
- **已修订（2026-05-14 · 黑骰局内结果不直接等于反噬）**：黑骰局内结果不应直接称为反噬；黑骰造成的是干扰、负面后果、任务失败 / 大失败或后续压力。真正的反噬通常来自某个报道结果发布后，玩家选择的解释通过读者、广告、势力、开局事件、新任务、地图变化或现实偏移回到玩家面前。
- **应用范围**：通用，影响普通任务、深度链、黑骰任务、现实偏移、周刊后果和中后期事件节奏。
- **状态**：已采纳为反噬强度控制原则；具体强度分层与触发门槛仍待细化。

### A18. 黑骰按探索阶段分层表达反噬
- **来源**：2026-05-14 黑骰反噬层级讨论。
- **机制**：黑骰可以先按阶段分层：早期黑骰体现“异样接触”，中期黑骰体现“规则开始异常”，后期黑骰体现“现实反噬介入”。不同阶段的黑骰不应只改变数值强度，还应改变玩家感受到的异常程度、后续任务回响和世界反馈范围。
- **应用范围**：黑骰任务、深度链终局、特殊判定 UI、后续任务生成、周开局事件与现实偏移反馈。
- **状态**：已采纳为黑骰阶段化设计方向；具体 330 公交车黑骰规则仍待玩法设计文档继续细化。
- **已修订（2026-05-14 · 前期 demo 暂不分层）**：当前仍在做前期 demo 体验，黑骰时间分层暂不进入具体设计；先聚焦前期黑骰的局内可玩性和 330 公交车“鬼混入人群”的主题玩法。后续如进入中后期设计，再重新展开阶段化黑骰。

## 二、已采纳 · 特殊任务/特殊角色类型

### B1. 「再追一次？」按钮 + 解锁更糟黑骰
- **来源**：2026-04-25 骰子机制讨论 → 借鉴自 Call of Cthulhu 的 Push Roll + Forbidden Lands 的灾厄累积。
- **机制**：派遣结算后**特定角色或特定任务类型**会出现「再追一次？」按钮：
  - 消耗 1 行动点；
  - 第二次判定**解锁更糟的黑骰内容**（更易触发反噬 / 灾厄面）；
  - 多次累积可触发"灾厄事件链"（负面事件序列）。
- **应用范围**：限角色特性 / 限任务类型，**非全员通用**。
- **合并备注**：与原来的 Forbidden Lands "灾厄面累积"思路合并为同一机制。
- **已修订（2026-04-28）**：「再追一次？」按钮与灾厄链首轮不做成常驻通用按钮，而是在**高难度关卡最后**作为惊喜 bonus 环节出现，类似一个临时事件；可与高危任务五档和深度调查终局重合。
- **已修订（2026-05-06）**：「再追一次？」更适合作为深度调查终局前后是否进入**黑骰任务**的临时事件选择。玩家可以收手，也可以消耗资源进入该关卡定义的黑骰判定；不再理解为普通任务里临时加一个小修正。

### B2. 「深度调查 + 真相风险」高难任务类型
- **来源**：2026-04-25 骰子机制讨论 → 借鉴自 Quacks of Quedlinburg 的推运气。
- **机制**：某些**高难任务类型**允许玩家**主动追加调查次数**：
  - 越多次调查 → 越接近真相；
  - 但每次都增加风险，可能触发：
    - **SAN 爆炸** / 狂性大幅上升；
    - 普通解释暴露（"煞有介事"档位下降）；
    - **灾厄累积**（负面修正链 / 与宏观势力挂钩的事件权重上升；**不**使用独立「混沌袋」抽象，见 M1 撤回说明）。
  - 玩家自行决定**何时停手**，体现走近科学式张力。
- **应用范围**：高难任务的标志性玩法，不要泛用化。
- **合并备注**：与 B1 同源（都属"再追一次"族），但 B1 是"按钮选项"层、B2 是"任务类型"层，分别成立。
- **已修订（2026-04-28）**：深度调查与真相风险应做成**连续任务类型**：每次任务完成后不消失，而是变成下一个任务，直到任务最终完结；玩家感知到的是事件逐层推进，而不是同一任务简单重复。
- **已修订（2026-05-06）**：深度调查任务链必须在地图位置、任务名称和 UI 上可视化连续关系：后续阶段应与前阶段形成路径；已完成任务环保留为灰色 / 空心 / 打勾状态；当前环高亮并显示阶段进度；结算后明确提示下一环出现位置。

### B3. 五档判定结果（部分高危任务）
- **来源**：2026-04-25 骰子机制讨论 → 借鉴自 Dungeon World / Witcher RPG。
- **机制**：高危任务的结果不止成功/失败二元，分五档：
  - **大成功**
  - **成功但反噬**（成功 + 公信/声望/狂性等代价）
  - **部分成功**（拿到一半收益）
  - **失败但有线索**（失败但留下下次可用信息或新 `topicKey`）
  - **大失败**
- 每档配独立 deadpan 文案。
- **应用范围**：高危任务专用，不替换现有普通任务的成功/失败逻辑。
- **已修订（2026-04-28）**：高危任务五档结果可以和 B1、B2 重合，作为高危任务的共同特点：连续深度调查的后段/终局可启用五档，并在合适结果后接「再追一次？」bonus 与灾厄链。
- **已修订（2026-05-06）**：高危任务与黑骰任务分离。高危任务通常直接出现在地图上，难度较高、失败后果较重、可能带来 debuff，但机制简单且**没有黑骰**。黑骰任务才使用特殊判定机制，并通常作为深度调查最终环、主线/事件触发的临时任务出现。旧“高危五档”表述保留为历史讨论，后续主干改为普通任务达标判定、黑骰任务四档。

### B4. 黑骰关卡：对手骰与特殊技能面
- **来源**：2026-04-29 黑骰关卡机制讨论。
- **机制**：特殊关卡下会出现对手骰，即黑色骰子。黑骰除了会抵扣己方投出的对应点数 / 成功外，还会有特殊技能骰面，用于削弱己方骰子、改变结算顺序或制造高危反噬。
- **时机原则**：默认黑骰在己方骰子投出前先投出，让玩家提前看到本次威胁并获得策略空间；少数特殊黑骰面可以打破该原则，例如在己方骰面确定后触发重投或改写。
- **应用范围**：特殊高危关卡、深度调查终局、再追一次 bonus；不作为普通任务常驻规则。
- **已修订（2026-04-29）**：黑骰应围绕关卡大叙事形成主题机制包，而不是小叙事面清单。330 公交车的黑骰核心应体现“普通人中混入了鬼”：黑骰会把负面骰强制加入或替换进玩家骰池。球状闪电若采用“似乎有生命 / 地球神经元”方向，黑骰应体现异常对观测与信号的反馈，而不是单纯过载扣点。
- **已修订（2026-05-06）**：黑骰任务结果使用 **大成功 / 成功 / 失败 / 大失败** 四档；大失败赋予较重当局后续 debuff。黑骰具体功能不在通用系统层面一次性定完，而在具体关卡设计中定义。
- **已修订（2026-05-14 · 前期 demo 黑骰边界）**：一般黑骰关卡的局内目标仍是凑够属性通关，玩家需要避免黑骰干扰，或围绕黑骰进行博弈和抉择；少数黑骰关卡可以设计不同胜利机制，但不作为默认。前期 demo 暂不设计黑骰时间分层。
- **已修订（2026-05-14 · 330 末班车方向）**：330 公交车黑骰重点是“鬼混入人群”。该关卡里黑骰最好一开始不可见，否则无法体现混入和试探。重投承担试探工作；锁定代表“我相信这颗骰是真的”。如果属性达标但计入了鬼骰，一定应有反面效果，不应给优秀效果；若产生新任务，也应视为后续压力或展开，而不是作为局内正面奖励。后续可以另做“目标就是找出某颗骰子”的特殊任务，但这不是默认黑骰胜利机制。

## 三、后期进阶选项

### C1. 派遣姿态：保守 / 标准 / 激进
- **来源**：2026-04-25 骰子机制讨论 → 借鉴自 Blades in the Dark 的 Position × Effect。
- **机制**：派遣前玩家选择本次姿态：
  - **保守**：成功档位低，但**反噬少**（公信稳）；
  - **标准**：基线；
  - **激进**：成功档位高，但**官方辟谣 / 反转事件触发概率提升**。
- **接入位置**：与现有 r1–r4 报道类型平行的"调查姿态"开关。
- **应用范围**：后期进阶，不在第一阶段实现。

### C2. 取材骰面 → 素材牌（与角色升级 / 临时构筑结合）
- **来源**：2026-04-25 骰子机制讨论 → 借鉴自 Sagrada / Cities of Splendor。
- **机制**：取材投出的骰面直接转换为**素材牌**进入合成台槽位，与现有现象/认知/情报体系无缝串联。
- **延伸**：作为**角色升级**方向 —— 角色升级或装备道具后，可解锁/添加新的骰面。
- **与 A2（临时人卡）结合**：临时人卡也可被视作一种"为本周构筑骰面池"的手段。
- **应用范围**：后期进阶，需先验证基础骰子层。

### C3. 员工特性 Hook（差异化判定规则）
- **来源**：2026-04-25 骰子机制讨论 → 借鉴自 Dicey Dungeons。
- **机制**：员工差异化不只是骰面分布，而是**读骰方式**也不同（按面值最大成功 / 按面图案触发额外 / 必须两颗同色才成功 等）。
- **状态**：方向上认同，**等待进一步具体方案**。
- **下一步**：拿出 3–5 个具体员工的"独特用骰规则"清单，再决定要不要落地。

## 五、机制库（备查，未采纳）

### M1. Arkham Horror LCG · Chaos Bag（混沌袋）
- **来源**：2026-04-25 骰子机制讨论。
- **当前状态（历史）**：用户曾表示「没看懂」，曾重新解释后入机制库备查。
- **简要原理**：判定时不投骰，从一个布袋摸 token；token 上是数字修正或特殊符号；袋的内容会随玩家行为/剧情变化；玩家可随时查看袋里"现在有什么"，但摸到什么仍是随机。
- **已撤回（2026-04-27）**：**不采用 Chaos Bag。** 用户理由：该机制本质是**跨轮积累的随机修正池**，与项目中已有 **四大势力（宏观维度：如公信、诡名、声望、守序/狂性等）的长期积累与事件释放**功能**重合**；避免两套「世界对你的态度」并行。  
  **替代承接**：环境修正、累积性风险与「世界回声」继续走 **`macro` + `weekEvent` + `staffEffects`**（及与势力挂钩的派遣/结算修正），不在工程上维护独立 token 袋数据结构。

### M2. 回收噪音线索 / 清理假料
- **来源**：早期全链 HTML Demo 的全球地图按钮；2026-05-13 讨论后从当前主干移出。
- **当前状态**：机制库备查，未采纳为当前正式流程。
- **简要原理**：玩家可主动清理博眼球假料或低质量线索，以换取宏观属性修正。旧 Demo 示例数值为 `诡名 -2，声望 +1`。
- **移出原因**：当前全球地图没有显示“噪音线索”对象，按钮却要求玩家理解自己在处理线索库存；实际效果只是宏观属性调节，和“选区域 -> 进入区域”的主流程语义不一致，容易干扰主操作。
- **后续若重启**：必须先有可见的噪音 / 假料对象、库存或事件来源，再决定回收成本、次数限制、收益与风险；不能只作为无条件宏观属性兑换按钮出现。

### M3. 解释路线气质候选（路线点子备查）
- **来源**：2026-05-14 核心乐趣、真相结构与科学 / 神秘路线差异讨论。
- **当前状态**：机制库备查，未采纳为正式路线规则。
- **使用前提**：路线应是软性长期倾向，由 `公信 / 诡名 / 声望 / 守序 / 狂性` 五项宏观属性组合承载，而不是硬分支按钮；正式结构原则见 A16。
- **候选气质**：
  - **科学解释**：确定感来自数据、复核、仪器、模型和误差排除；后期可通过模型过准、观测污染结果、科学边界走向不可知来崩塌。倾向公信、守序较高，狂性可在深层科学异常中上升。
  - **神秘解释**：确定感来自符号、仪式、梦境、旧文本和跨来源共鸣；后期可通过“解释越有效，越像被某种东西回应或认领”来崩塌。倾向诡名、狂性较高。
  - **阴谋解释**：确定感来自文件、线人、删改痕迹、资金流和失踪名单；后期可通过证据链本身被设计、玩家成为另一层解释的传播器来崩塌。声望、诡名或公信可并存，守序容易承压。
  - **猎奇 / 小报解释**：确定感来自标题、读者反馈、销量、传播热度和夸张包装；后期可通过原本为了流量编出的故事开始自证成真来崩塌。倾向声望、诡名较高，公信可能受损。
  - **民俗 / 地方记忆解释**：确定感来自旧闻、地方传说、家族记忆、口述传统和旧地图；后期可通过过去经由传说回流并改写现在来崩塌。倾向诡名、声望较高，守序和狂性视题材变化。
- **设计提醒**：各路线可以都通向不可知，但不可知的味道不能相同；路线差异应体现在确定感来源、崩塌方式、代价类型和宏观属性组合上。科学路线不等于安全路线，神秘路线也需要自己的证据感和内部秩序；猎奇路线不只是低质量路线，它适合承载“故事反过来制造现实”的魅力点。
