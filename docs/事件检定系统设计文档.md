# 事件检定系统（复刻《苏丹的游戏》）——功能/技术设计文档（最新版）

> 版本：v1.0（原型已实现）  
> 目标：交付程序落地实现（与网页原型一致）

## 1. 文档目的与范围

- **目的**：提供一份可直接交付程序落地的设计文档，复刻《苏丹的游戏》的事件掷骰判定机制，并支持：
  - **纯概率数值模式**（只显示成功率与结算结果）
  - **骰子表现模式**（显示投骰结果，并有“对手对抗”二段式抵消表现）
- **范围**：事件触发时的检定计算、对抗扣减、多属性检定（综合/分别）、UI 展示与交互、数据结构与接口。
- **不在范围**：具体剧情文本、道具系统、成长系统、存档、联网。

## 2. 核心概念（与《苏丹的游戏》一致）

### 2.1 单次骰子（伯努利试验）

- 每颗骰子只有两种结果：**命中/未命中**（成功/失败）。
- 单骰成功率为 \(p\)，由难度决定，可叠加事件修正。

### 2.2 一次检定（二项分布）

- 令 \(n\) 为骰子数（通常由属性值决定），则命中数 \(X\) 服从：
  - \(X \sim Binomial(n, p)\)
- 判定条件：若 \(X \ge k\)，则检定成功。

### 2.3 难度映射到单骰成功率

- 简单：\(p = 0.60\)
- 普通：\(p = 0.50\)
- 困难：\(p = 1/3 \approx 0.3333\)

> 注：该映射来自《苏丹的游戏》公开攻略/Wiki 的通用表述。

## 3. 支持的检定类型（Check Types）

系统支持三类检定（同一事件可配置其一）：

### 3.1 单属性检定（single）

- **输入**：属性 A 值 `attrA`，阈值 `k`
- **骰子数**：`n = attrA`
- **对抗扣减**：`n_eff = max(0, n - enemyAttr)`
- **判定**：\(X \sim Binomial(n_{eff}, p)\)，成功条件 \(X \ge k\)

### 3.2 综合多属性检定（combined）

- **输入**：属性 A、B 值 `attrA/attrB`，阈值 `k`
- **骰子数**：`n = attrA + attrB`
- **对抗扣减**：`n_eff = max(0, n - enemyAttr)`
- **判定**：\(X \sim Binomial(n_{eff}, p)\)，成功条件 \(X \ge k\)

> 解释：综合检定把多属性合并为“同一堆骰子”，结算一次。

### 3.3 分别多属性检定（split）

- **输入**：属性 A、B 值 `attrA/attrB`，阈值 `kA/kB`
- **对抗扣减**：将对手属性按占比分摊扣减（见 4.3）
- **判定**：
  - \(X_A \sim Binomial(nA_{eff}, p)\)，成功条件 \(X_A \ge kA\)
  - \(X_B \sim Binomial(nB_{eff}, p)\)，成功条件 \(X_B \ge kB\)
- **结果组合（本原型采用）**：
  - 若 A、B 都成功 → **大成功**
  - 若仅一项成功 → **小成功**
  - 若都失败 → **失败**

> 备注：组合规则建议配置化（见 8.2）。

## 4. 对抗检定（Opponent / Contest）

### 4.1 标准对抗语义（与苏丹 Wiki 表述一致）

- **总骰数扣减**：`n_eff = max(0, n_self - enemyAttr)`

### 4.2 展示层的“对手存在感”（二段式表现）

为增强对抗存在感，骰子模式采用两阶段可视化：

1. **玩家回合**：先投出原始 `n_self` 颗骰子并展示结果（成功/失败）。
2. **对手回合**：对手“抵消” `enemyAttr` 颗骰子（将其标为作废 `—`），最后只统计未作废骰子的命中数作为最终结果。

**一致性要求（必须）**：

- 不能引入“优先抵消成功面”或“把成功翻成失败”等行为，否则会改变概率分布，不再等价于《苏丹》的“减骰子数”对抗。

**实现方式（保证等价性）**：

- 玩家先得到 `n_self` 个 i.i.d 的伯努利结果（成功概率 \(p\)）。
- 对手从这 `n_self` 个结果中**随机无放回抽取** `enemyAttr` 个索引，将其作废。
- 最终以未作废结果统计命中数。

> 在 i.i.d 伯努利前提下，上述“投完再随机作废”与“直接投 \(n_{eff}=n_{self}-enemyAttr\) 颗”在分布上等价。

### 4.3 split 模式下的对抗分摊（可解释 & 总量一致）

- 先计算：
  - `nTotal = attrA + attrB`
  - `enemyA = floor(enemyAttr * attrA / nTotal)`
  - `enemyB = enemyAttr - enemyA`
- 有效骰子数：
  - `nA_eff = max(0, attrA - enemyA)`
  - `nB_eff = max(0, attrB - enemyB)`
- 展示时对手分别抵消 A 组 `enemyA` 颗、B 组 `enemyB` 颗（同样随机抽取作废）。

## 5. 理论成功率计算（纯数值模式与 UI 提示用）

### 5.1 单/综合检定

理论成功率：

\[
P_{success} = P(X \ge k), \quad X \sim Binomial(n_{eff}, p)
\]

计算：

\[
P(X \ge k) = \sum_{i=k}^{n_{eff}} \binom{n_{eff}}{i} p^i (1-p)^{n_{eff}-i}
\]

### 5.2 split 检定（原型展示“至少一项通过”概率）

子检定成功率：

- \(P_A = P(X_A \ge kA)\)，\(X_A \sim Binomial(nA_{eff}, p)\)
- \(P_B = P(X_B \ge kB)\)，\(X_B \sim Binomial(nB_{eff}, p)\)

“至少一项通过”概率：

\[
P_{\ge 1} = 1 - (1-P_A)(1-P_B)
\]

可选：若 UI 需要展示三档分布：

- \(P_{both} = P_A P_B\)
- \(P_{one} = P_A(1-P_B) + (1-P_A)P_B\)
- \(P_{fail} = (1-P_A)(1-P_B)\)

## 6. 单骰成功率 p 的来源与修正（Δp）

- `p_base`：由难度选择得出（0.6/0.5/1/3）。
- `Δp`：事件级/临时修正（可正可负），用于微调某次检定的整体难度。
- 最终：
  - `p = clamp(p_base + Δp, 0, 1)`

> 说明：Δp 的设计目的是让策划能在不改动难度档位的情况下，对单个事件做精细化调参。

## 7. 运行时结算（随机流程）

### 7.1 纯数值模式（Numeric）

- UI 展示：理论成功率（百分比）+ 风险标签（可选）+ 结算结果（成功/失败/大成功等）。
- 结算逻辑：按对应检定类型做一次随机（伯努利试验 n 次并计数）。

### 7.2 骰子模式（Dice）

结算逻辑与纯数值一致，但增加可视化：

- 先展示玩家的原始投骰结果（`n_self` 颗）
- 再展示对手抵消（`enemyAttr` 颗作废）
- 以剩余骰子统计命中数并给出最终结果

动画要求：

- 抵消应“逐个发生”（建议每 80–110ms 1 颗），形成对手行动感
- 作废骰子显示为灰色 `—` 且带明显“划线”样式

## 8. 结果类型与可配置扩展

### 8.1 单/综合的结果扩展（可选）

- 默认：成功 / 失败（`X ≥ k`）
- 可扩展：按命中数分档（小成功/大成功），配置 `successBands`：
  - 例如：`k1 <= X < k2` 为小成功，`X >= k2` 为大成功。

### 8.2 split 的组合逻辑（建议配置化）

支持以下常见组合模式（可作为枚举）：

- `ALL`：必须全部子检定成功才成功
- `ANY`：任意一项成功即成功
- `BANDS`：按（双成功/单成功/全失败）映射到（大成功/小成功/失败）（原型采用）

### 9.1 难度表

```json

  "difficulty": {
    "easy":   { "p": 0.60 },
    "normal": { "p": 0.50 },
    "hard":   { "p": 0.3333333333 }
  }
}
```

### 9.2 事件检定配置（示例）

```json
{
  "checkType": "single",
  "attributeA": "wisdom",
  "k": 3,
  "deltaP": 0.0,
  "hasOpponent": true
```


## 10. UI/交互规范（落地要求）

### 10.1 必须提供的 UI 元素

- 检定类型选择：single / combined / split
- 输入（或由角色状态自动填充）：
  - `attrA`, `attrB`
  - `enemyAttr`（允许为 0）
  - `k` 或 `kA/kB`
  - `difficulty`（决定 p）
  - `Δp`（可选，若不用则固定 0）
- 模式切换按钮：
  - **骰子模式 / 纯数值模式** 一键切换
- 展示：
  - 理论成功率（至少显示 1 位小数或整数）
  - 本次结算结果（成功/失败/大成功/小成功）
  - 骰子模式下必须显示：投骰 + 对手抵消

### 10.2 推荐展示信息（便于玩家理解）

- 显示有效骰子数：
  - `n_eff = n_self - enemyAttr`（或 split 的 A/B 有效骰子）
- split 模式下显示：
  - A 组：命中数 vs kA
  - B 组：命中数 vs kB

一次“事件检定”执行后，建议返回如下结构给 UI：

```json

  "p": 0.5,
  "probabilityShown": 0.623,

    "enemyA": 1,
    "enemyB": 2
```

## 12. 一致性与注意事项（必须遵守）

- **对抗表现不能改变概率分布**：
  - 不允许“优先抵消成功面”、“把成功翻成失败”等行为；
  - 必须采用“随机作废骰子”以保持与 `n_eff = n_self - enemyAttr` 等价。
- `p` 必须 clamp 到 [0,1]。
- `k` 必须 clamp 到 [0, n_eff]（split 则 clamp 到各自有效骰子上限或原始上限）。
- 若 `n_eff = 0`：
  - \(X = 0\) 恒成立，检定必失败（除非 k=0）。
- split 的对抗分摊需保证：
  - `enemyA + enemyB = enemyAttr`
  - `nA_eff + nB_eff = (attrA + attrB) - enemyAttr`（在 attrA/attrB 足够大时）
