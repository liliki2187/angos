## 二、设计张力（难题 A）：两条都想保留

| 诉求 | 含义 |
|------|------|
| **过程感** | 玩家经历：发现线索 →（加工）→ 形成 **认知** / 编辑部观点，而非线索直接变稿。 |
| **线索进报道** | 现象、情报等必须在 **报道合成** 里真实参与，与组版、结算同一套叙事。 |

**草拟报道**曾被当作「线索先进报道」的一种低门槛出口；去掉后，张力落在：**流程顺序、消耗规则、是否要有可见的中间类型**。

## 三、设计张力（难题 B）：认知相对报道的顺序与「寿命」

**假设**：是否规定 **探索牌 → 先合成认知牌 → 再用认知牌合成报道**。

| 模式 | 结果 |
|------|------|
| **认知一次性 + 定向**（如：科学路径研究球状闪电） | 同题材难以再拿「玄学向」认知 → **有取舍、体量可控**，但玩家可能觉得**少玩半条线**。 |
| **认知可复用 / 无上限分支** | 每条线索理论上对应大量认知实例 → **组合爆炸、UI 与内容体量难控**。 |

核心：**中间层要有仪式感，又要在数据结构上可枚举**——需明确认知是 **实体牌** 还是 **槽位/状态**，以及 **同题材上限**。

## 四、讨论中形成的方向（待拍板，非定稿）

以下为对话里梳理的 **可选骨架**，可组合：

1. **方向 A**：认知做成 **题材槽位 / 立场**（科学·秘闻·流行互斥），减少「无限张认知牌」。
2. **方向 B**：保留实体认知牌，但 **同题材强上限 + 覆盖/互斥**，控制分叉。
3. **方向 C**：**草拟**仅作叙事轻入口（现象+情报），与认知养成解耦；或不用草拟，由快讯承担「线索先进报道」。
4. **方向 D**：**软化顺序**——无认知也可发快讯；有认知则深度/专栏等 **加成或解锁**，教程可推荐先认知但不硬卡。

**倾向性讨论摘要**：较看好 **B + D** 的混合（实体认知 + 上限；快讯不强制认知）；草拟可选简化为「快稿」或继续不用。

## 五、建议下一步定案用的三个问题（产品自答）

1. 同题材一周内，**更换研究取向**是否应有**明确成本**？  
2. 认知更应被理解成 **装备式卡牌** 还是 **本周编辑部决议（状态）**？  
3. **没有认知时**，深度报道是否必须能发？（能 → 节奏软；不能 → 强顺序）

答完即可收窄到「一页纸规则」，再回写正式 GDD / 实现。
