# 报刊结算与出版玩法设计文档（网页原型完整版）

> **对应实现文件**：项目根目录 `index.html`（单页 HTML + 内联 CSS/JS）  
> **文档性质**：与当前代码一一对应，便于策划复现、程序移植与平衡调参。  
> **刻意省略的系统**：员工士气、债务、法律风险等（原型未实现）。

---

## 1. 玩法总览

### 1.1 玩家循环

1. **故事库**：通过「随机刷新故事库」生成若干条候选报道；可选数量 8 / 10 / 12。
2. **组版**：将报道**拖拽**到当期版面的 6 个版位之一；可覆盖已有报道；可单版位「移除」或「清空版面」。
3. **预览**：拖拽悬停在版位上时，仅在该版位框内显示**预计利润 / 预计销量 / 关键原因**（可展开「详情」）；版面内已上稿且参与连携的版位会**绿色高亮**（连携预览）。
4. **结算本期**：点击「结算本期」执行一期完整经济结算，更新**订阅数**与**编辑定位值**，并在右侧面板展示净利润与分项收入、成本、乘数拆解。

### 1.2 界面分区

| 区域 | 功能 |
|------|------|
| **1) 本期故事库** | 随机刷新、条数选择、可拖拽报道卡片 |
| **2) 当期版面** | 6 个版位、拖拽投放、移除、清空、结算按钮 |
| **3) 结算面板** | 左侧为**实时统计**（未点结算也会随版面变化）；下方为**本期结算结果**（点击结算后覆盖） |
| **Toast（右上角）** | 覆盖已有报道时提示「已替换：旧标题 -> 新标题」，含**撤销**按钮 |

## 2. 报道（故事）数据结构

每条报道由程序随机生成，字段如下：

| 字段 | 说明 |
|------|------|
| `id` | 序号（与故事库条数对应） |
| `title` | 标题（见 2.2 生成规则） |
| `tags` | 1～2 个题材标签（见 2.1） |
| `quality` | 质量档：`Bronze` / `Silver` / `Gold` |
| `baseValue` | 基础数值：Bronze=150，Silver=300，Gold=450 |
| `negatives` | 负面标签字符串数组，可能为空 |

### 2.1 题材标签（8 类，英文 key → 中文展示）

代码常量 `TAGS`：

| key | 中文（界面） |
|-----|----------------|
| Politics | 时政 |
| Military | 军事 |
| Economy | 经济 |
| Sport | 体育 |
| Gossip | 花边新闻 |
| Pets | 猫狗 |
| Humor | 幽默 |
| Shopping | 购物 |

- 首标签：`tag1` 从 `TAGS` 均匀随机。
- 第二标签：概率 **55%** 再从「与 tag1 不同」的标签中随机；否则仅单标签。

### 2.2 标题生成（代入感）

模板依赖：

- **地点** `LOCATIONS`：曼哈顿、布鲁克林、皇后区、布朗克斯、华尔街、港口区、市政厅、唐人街  
- **机构** `ORGS`：市议会、警署、港务局、联邦调查局、工商总会、市政厅、消防总队、联邦法院  
- **议题** `SUBJECTS[主标签]`：按主标签从列表随机  
- **动作** `ACTIONS`：引发争议、进入调查、通过审议、宣布升级、出现反转、再起波澜、确认落地、紧急叫停  

规则：

- **单标签**：`{地点}{机构}{议题}{动作}`
- **双标签**：`{地点}{机构}就{议题}{动作}，{第二标签中文}线索同步浮现`

### 2.3 质量与负面标签

- **质量档**：在 Bronze / Silver / Gold 三者中均匀随机。
- **负面条数**：概率 **28%** 生成 1～2 条负面；否则 0 条。
- **负面类型**（英文 id，界面展示为 `负面:xxx/yyy`）：

`sloppy_writing`, `thin_source`, `late_edit`, `bias_overreach`, `fabrication`

## 3. 版面与版位

### 3.1 版位列表（固定 6 槽）

| id | 名称 | weight | 界面说明文案 | 版位乘数显示 |
|----|------|--------|--------------|--------------|
| front-main | 头版头条 | 1.0 | 最高曝光，约 3.0x | `1 + 2*weight` = **3.0x** |
| front-side | 头版次条 | 0.7 | 高曝光，约 2.4x | **2.4x** |
| feature-1 | 重点专题 A | 0.45 | 中高曝光，约 1.9x | **1.9x** |
| feature-2 | 重点专题 B | 0.4 | 中高曝光，约 1.8x | **1.8x** |
| inner-1 | 内页 A | 0.2 | 标准曝光，约 1.4x | **1.4x** |
| inner-2 | 内页 B | 0.2 | 标准曝光，约 1.4x | **1.4x** |

> 版位「加成」在 UI 上显示为 `(1 + 2 * slot.weight)`，与下方 `mLayout` 中的 `frontWeight` 计算配合使用。

### 3.2 组版规则

- 每个版位最多 **1** 篇报道。
- **拖拽**到某版位即放置；若已有报道则**覆盖**（见 7 节 Toast 与撤销）。
- 「移除」清空该版位；「清空版面」清空全部版位并重置右侧结算区为「尚未结算」提示。

## 4. 三轴题材分类（结算用）

用于**占比、偏科、均衡**，替代简单的「严肃/娱乐」二分。

| 轴 | 英文概念 | 包含标签 |
|----|-----------|----------|
| **P 公共事务** | Public Affairs | Politics, Military, Economy |
| **M 大众关注** | Mass Appeal | Sport, Shopping |
| **L 轻内容** | Lifestyle & Light | Gossip, Pets, Humor |

统计方式：对**本期已上版**所有报道的**所有标签**逐条计数（双标签报道贡献 2 条标签计数）。

定义（`totalTagCount = allTags.length`，至少为 1 避免除零）：

- `publicRatio = publicCount / totalTagCount`
- `massRatio = massCount / totalTagCount`
- `lightRatio = lightCount / totalTagCount`
- `dominantRatio = max(publicRatio, massRatio, lightRatio)`
- `dominantType`（代码内部）：`public` / `mass` / `light`（与三轴最大者对应）

界面中文（`analyzeTagEffects`）：

- 最大者为 public → **公共事务**
- 最大者为 mass → **大众关注**
- 最大者为 light → **轻内容**

## 5. 编辑定位 `editorialProfile`

### 5.1 取值与含义

- 范围：**[-100, 100]**，初值 **0**。
- 实时统计面板中的**文字标签**（仅展示，与阈值挂钩）：
  - `editorialProfile <= -35` → **信息型**
  - `editorialProfile >= 35` → **热度型**
  - 否则 → **平衡型**

> 负向偏「信息型」、正向偏「热度型」；具体数值参与归一化 `profileNormalized = (editorialProfile + 100) / 200`。

### 5.2 何时更新

仅在点击 **「结算本期」** 时更新（`settle()` 内），根据**本期**三轴占比：

```text
profileDelta = (lightRatio - publicRatio) * 24 - massRatio * 4
editorialProfile = clamp(editorialProfile + profileDelta, -100, 100)
```

含义简述：

- **轻内容占比相对公共事务越高**，定位越向**热度型**推（正值）。
- **公共事务相对轻内容越高**，定位越向**信息型**推（负值）。
- **大众关注**会**削弱**正向偏移（`- massRatio * 4`），作缓冲。

## 6. 核心结算公式（与代码一致）

以下变量均来自**本期已上版**报道集合。

### 6.1 中间统计量

- `placedCount`：已填版位数  
- `emptySlots = 6 - placedCount`  
- `totalBaseValue`：各篇 `baseValue` 之和  
- `allTags`：所有报道标签展开列表（含双标签）  
- `uniqueTags`：`allTags` 去重后的**种类数**  
- `negPenaltyPoints`：所有报道 `negatives.length` 之和  
- `tagCountMap`：每个标签 key 的出现次数  
- `comboRaw`：  
  `sum_over_tags( max(0, count-1)^0.8 )`  
- **连携（link）**：某标签出现次数在 **[2, 3]**（含）时，该标签计入 `linkedTags`，`linkCount = linkedTags.length`  
- 三轴占比：`publicRatio`, `massRatio`, `lightRatio`, `dominantRatio`, `dominantType`（见第 4 节）  
- `profileNormalized = (editorialProfile + 100) / 200`  
- `frontWeight`：  
  `placed 中各 slot.weight 之和 / 2.95`  
  （2.95 为设计用归一化常数，与 6 个版位最大 weight 之和相关）

### 6.2 需求乘数（连乘）

以下全部乘在 `baseDemand` 上。

| 符号 | 计算（JavaScript 等价） | 上限/下限 |
|------|-------------------------|-----------|
| **mQuality** | `min(1.85, 1 + 0.0008 * totalBaseValue)` | 上限 1.85 |
| **comboBase** | `min(1.35, 1 + 0.06 * ln(1 + comboRaw))` | 上限 1.35 |
| **comboProfileBoost** | `1 + 0.2 * max(0, profileNormalized - 0.5)` | — |
| **mCombo** | `min(1.45, comboBase * comboProfileBoost)` | 上限 1.45 |
| **mDiversityBase** | `min(1.3, 1 + 0.05 * ln(1 + uniqueTags))` | 上限 1.3 |
| **diversityStyleBoost** | `1 + 0.25 * max(0, 0.5 - profileNormalized) * publicRatio` | — |
| **mDiversity** | `min(1.45, mDiversityBase * diversityStyleBoost)` | 上限 1.45 |
| **mLayout** | `1 + 1.2 * frontWeight` | — |
| **mEmpty** | `max(0.7, 1 - 0.06 * emptySlots)` | 下限 0.7 |
| **mPenalty** | `max(0.6, 1 - 0.025 * negPenaltyPoints)` | 下限 0.6 |
| **mLink** | `min(1.25, 1 + 0.05 * linkCount)` | 上限 1.25 |
| **mBias**（偏科） | 若 `dominantRatio > 0.7`：`max(0.75, 1 - 0.25 * (dominantRatio - 0.7) / 0.3)`；否则 `1` | 下限 0.75 |
| **mBalance**（均衡奖励） | 若 `publicRatio ∈ [0.25, 0.55]` **且** `lightRatio ∈ [0.2, 0.5]` → `1.06`，否则 `1` | — |

**需求：**

```text
demand = baseDemand * mQuality * mCombo * mDiversity * mLayout * mEmpty * mPenalty * mLink * mBalance * mBias
```

默认常量：`baseDemand = 3200`。

### 6.3 销量与缺货率

```text
sold = min(round(demand), printCapacity)
stockoutRate = demand <= 0 ? 0 : max(0, (demand - sold) / demand)
```

默认：`printCapacity = 4200`。

### 6.4 收入

**卖报：**

```text
circulationRevenue = sold * coverPrice
```

默认：`coverPrice = 0.3`。

**订阅（简化模型）：**

```text
newSubRate = 0.004 + 0.0008 * uniqueTags + 0.0006 * sqrt(max(1, totalBaseValue / 150))
churnRate = 0.003 + 0.0012 * negPenaltyPoints + 0.002 * stockoutRate
newSubs = sold * newSubRate
lostSubs = subscribers * churnRate
subDelta = round(newSubs - lostSubs)
nextSubs = max(0, subscribers + subDelta)
subscriptionRevenue = nextSubs * subUnitValue
```

默认：`subscribers` 初值 1200，`subUnitValue = 0.18`。  
仅在 `calculate(true)`（结算）时写回 `state.subscribers = nextSubs`。

**广告：**

```text
adProfileBonus = 1 + 0.15 * (1 - abs(editorialProfile) / 100)
adMatch = min(1, (adMatchBase + uniqueTags * 0.05 - negPenaltyPoints * 0.03) * adProfileBonus)
adUnit = 180 + 320 * adMatch
adsRevenue = adSlots * adUnit
```

默认：`adMatchBase = 0.6`，`adSlots = 2`。

**总收入：**

```text
grossRevenue = circulationRevenue + subscriptionRevenue + adsRevenue
```

### 6.5 成本与利润

```text
paperPacks = ceil(printCapacity / 1000)
paperCost = paperPacks * paperPackCost
totalCost = paperCost + salaryCost + opsCost
profit = grossRevenue - totalCost
```

默认：`paperPackCost = 210`，`salaryCost = 980`，`opsCost = 260`。

> 纸张成本按**印刷上限**折算包数，与是否「卖完」无关（原型简化）。

## 7. 拖拽、预览与覆盖

### 7.1 拖拽

- 报道卡片 `draggable = true`。
- `dragstart`：记录 `draggingStoryId`，`dataTransfer` 存 story id，`dragging` 样式。
- `dragend`：清空 `draggingStoryId`，调用 `clearDropPreview()`。

### 7.2 悬停版位（`dragover`）

- 必须存在 `draggingStoryId`；`preventDefault` 以允许 drop。
- 用 **假设将当前拖拽稿放入该版位** 调用 `calculateWithPlacement(slotId, storyId)` 得到 `projected`，与**进入拖拽前**快照的 `baseline = calculate(false)` 对比。
- **预览区**（`renderImpactRows`）：
  - **默认三行**：预计利润变化、预计销量变化、关键原因（字符串，由多条提示用 `；` 拼接）
  - **详情**（`<details>`）：连携题材数变化、多样性乘数变化、偏科惩罚乘数变化、负面惩罚点变化
- **关键原因**可能包含（按条件追加）：
  - 有连携：`连携奖励: {中文标签、顿号分隔}`
  - `dominantRatio > 0.7`：`偏科减益: {公共事务|大众关注|轻内容}占比{百分比}%`
  - `editorialProfile < -10`：`信息型定位: 多样性收益增强`
  - `editorialProfile > 10`：`热度型定位: 连携收益增强`
  - 若该版位已有稿：`将替换: {原标题}`
- **版位边框**：`profitDelta > 0` → `good-preview`（绿），`< 0` → `bad-preview`（红），否则无。
- **连携高亮**：仅对**版面内已放置**、且标签命中 `projected.linkedTags` 的版位加 `synergy` 类（绿）；**不对故事库未上版卡片高亮**。

### 7.3 `dragleave`

- 移除 drag-over / 预览色 / synergy；影响区恢复默认一句「拖入报道可预览该版位影响」。

### 7.4 `drop`

- 读取 story id，若该版位**原有稿且新稿存在**：设置 `lastReplaceAction`，调用 `showReplaceToast(prev, new)`。
- `placeStory` 更新状态；`clearDropPreview()`。

### 7.5 覆盖 Toast 与撤销

- 文案：`已替换：{旧标题} -> {新标题}` + 按钮「撤销」。
- 撤销：恢复 `state.placed[slotId] = prevStory`，关闭 toast，重绘版位与实时统计。
- **2 秒**后自动清除 `lastReplaceAction` 并隐藏 toast（若未撤销）。

## 9. 结算本期输出

点击「结算本期」后右侧显示：

1. **本期净利润**（大号，正绿负红，`fmtMoney` 格式为 `+$` 或 `-$` 整数）
2. **分项**：卖报、订阅、广告、总收入、纸张/工资/运营、总成本、实际销量、订阅（下期）
3. **关键乘数**一行，依次为：  
   质量、重复基础、定位连携加权、重复最终、多样性基础、信息型加权、多样性最终、版位、空版、负面、连携、均衡、偏科  

结算后：

- 订阅数更新（见 6.4）。
- `editorialProfile` 按 5.2 更新。
- 再次调用 `renderLiveStats()`。

## 11. 实现注意与扩展建议

1. **连携**与 **comboRaw**：前者为「某标签 2～3 条」的条数奖励（mLink）；后者为全标签重复度的连续函数（comboBase），二者并存。
2. **编辑定位**只影响 combo 加权、diversity 加权、广告匹配；偏科（mBias）与均衡（mBalance）由**本期三轴占比**决定，与定位独立但同屏展示。
3. 若需与《News Tower》正式数值对齐，建议将本章公式系数抽到配置表，保留本文件为「原型基线」。
