## 2. 游戏定位与一周闭环

- **体裁**：NEWS TOWER 式周刊节奏 + 苏丹式卡牌/线索驱动；玩家为**小众未解之谜周刊主编**。
- **标准周循环**（与 [全链条汇总 §2](./世界未解之谜周刊_全链条版设计文档汇总.md)、[`design/systems/full-chain-loop.md`](../../../design/systems/full-chain-loop.md) 一致）：

  1. 回合起始（简报、事件提示等）  
  2. 全球地图 → 区域 → 探索节点  
  3. 组队（1–3 人）→ **检定** → **线索**入库  
  4. 日数耗尽或主动结束 → **编辑部**  
  5. **线索 → 报道**（全链含**合成台**；补足故事库规则见 [汇总 §3.3](../docs/世界未解之谜周刊_全链条版设计文档汇总.md)）  
  6. **6 版位组版** → **结算**（需求、销量、订阅、广告、成本、利润）  
  7. 更新跨周状态 → **下一周**

流程图与衔接表见：[世界未解之谜周刊_全链条版设计文档汇总.md §2–§3](./世界未解之谜周刊_全链条版设计文档汇总.md)。

## 4. 时间与探索流程

- **每周 7 日**：执行节点消耗 `days`；不足则不可执行。  
- **结束本周**：自然耗尽或手动结束 → 编辑部；新周重置日数、清空周临时线索等（Demo 规则）。  
- **全球层**：区域字段、`sci` / `occult` / `pop` 筛选（高亮与可探索性，不删除节点）、**回收噪音**等 —— [探索部分设计文档 §5–§6](../../世界未解之谜周刊_探索部分设计文档.md)。  
- **节点**：`permanent` / `temp`（可含 `deadlineDay`）/ `hidden`（`unlock(macro)`）；链式 `chain` —— 同文档 §7。  
- **探索状态机**简图：同文档 §5。

系统级摘要另见 [`design/systems/exploration.md`](../../../design/systems/exploration.md)。

## 5. 职员、队伍与检定

### 5.1 派遣

- **1–3 名职员**；各 `need` 维度为所选职员该维度**数值之和** ≥ 需求。  
- **实现现状**：全链 HTML Demo 仍使用五维：**探索、生存、洞察、诡思、理性**（与节点 `need` 一致）。
- **设计主干（2026-05-06）**：后续大改方向改为**角色骰系统**。每个正式角色有一颗 6 面角色骰，骰面产出原 10 项个人属性中的某一项数值；任务需求按具体属性写，玩家投骰后选骰并累计属性点判断是否达标。旧“属性加总 -> 骰子数量”的路线作为历史/分支方案保留，不作为当前主干设计继续推进。
- 更完整的「个人属性」列表见 [`design/systems/attributes.md`](../../../design/systems/attributes.md)。当前决策是**保留原 10 项个人属性**，不把它们改名为新专精，而是在系统层增加功能域分类：
  - **外勤域**：探索、生存、胆识 —— 进入现场、承受危险、高风险行动。
  - **认知域**：洞察、理性、诡思（灵视） —— 发现矛盾、理解证据、解释科学/超自然异常。
  - **内容域**：笔力、想象、造势 —— 写作、报道包装、震惊体、传播与内容质量。
  - **关系域**：人脉（话术） —— 采访、线人、势力交涉、官方/民间渠道。
- **探索判定常见属性**：以外勤域 + 认知域为主；内容域和关系域不是探索主输入，但部分探索任务也会使用。
- **线索追查任务**：尤其是“派遣队员获得线索后，第二天出现新正式任务”的线索任务，与关系域关联度较高，常需要采访、线人、人脉或机构口径来把线索推进成正式任务。
- **内容域使用场景**：主要用于写作、报道、局前特殊事件、舆论预热与素材包装；但例如现场直播、突发采访、抢发截稿等探索节点也可调用内容域属性。

### 5.2 角色骰系统（设计主干）

> 本节为 2026-05-06 讨论后的主干方向；当前 HTML 原型尚未完全迁移。

- **骰面格式**：首版骰面暂为单一属性数值，如 `洞察2`、`人脉1`、`胆识3`；后续特殊角色可有复合属性或特殊效果面。
- **任务需求**：按具体属性写，如 `探索3 + 洞察2`；特殊关卡可配置属性替代折算，但不作为普通任务默认规则。
- **判定阶段**：每个任务只有一个阶段判定；探索结束后，报道合成在专门界面进行。
- **普通任务结果**：只看是否达标，达标成功，未达标失败。
- **黑骰任务结果**：使用 `大成功 / 成功 / 失败 / 大失败`；大失败给较重 debuff，影响当局后续。
- **人数上限**：普通任务最多 3 人；高危任务最多 5 人；临时人物卡算完整角色骰并纳入人数限制。
- **额外骰**：道具骰不纳入人数限制；道具骰、黑骰、支援骰都计入总骰数。
- **升级**：每次升级必须改变骰面；可为数值提升、增加骰面或替换骰面。成长路线固定或半固定，玩家在分支中选择。
- **插槽**：特殊插槽只来自道具、卡牌或事件，不作为普遍升级项。
- **风险面**：普通任务没有风险面；风险面主要由黑骰失败 debuff 赋予个别角色，强力角色技能也可自带风险面作平衡。
- **选骰**：普通判定需要选骰，但必须提供便捷按钮直接跳过全部选骰 / 自动选择；个别特殊骰面、高危任务或 debuff 可以禁止重投。
- **预览**：不要求精准概率，仅给 `低 / 中 / 高` 风险提示。
- **UI**：角色卡只显示属性专精；点击角色进入详情页显示完整 6 面；升级页显示完整 6 面的新旧变化；任务配置页突出本任务相关面。

### 5.3 任务类型边界（2026-05-06）

| 类型 | 进入方式 | 判定机制 | 结束 / 消失规则 | UI 必须表达 |
|------|----------|----------|----------------|------------|
| **白色调查** | 常驻或可重复任务 | 普通达标判定 | 不因失败消失，可重复追踪 | 标明“可反复调查 / 累积题材信息” |
| **红色截稿** | 地图临时任务或限时机会 | 普通达标判定；可为高难但机制简单 | **失败、成功或超过截止日期后消失** | 标明截止日、一次性窗口、成功/失败/过期都会关闭 |
| **线索追查** | 由线索、读者来信、线人等触发 | 普通达标判定，常用关系域属性 | 成功后通常在次日生成正式任务；失败可延迟、降级或关闭线索 | 标明“线索 -> 正式任务”的转换关系 |
| **深度调查任务链** | 连续任务起点或线索追查成功后出现 | 每环一个独立任务、一次判定 | 当前环完成后生成下一环；链条最终完结 | 必须在地图、任务名、任务卡和进度 UI 上可视化连续关系 |
| **高危任务** | 多数直接出现在地图上，也可由任务链生成 | 难度较高但机制简单；**没有黑骰** | 可带 debuff 或更重失败后果，但不使用黑骰特殊机制 | 标明“高危 / 高难 / 失败后果重”，但不混同黑骰任务 |
| **黑骰任务** | 通常是深度调查最终环；也可由个别事件、主线或临时任务触发 | 使用黑骰关卡机制与四档结果 | 大失败通常给当局后续 debuff；具体黑骰能力由关卡定义 | 标明黑骰机制、特殊规则、可能 debuff |

### 5.4 深度调查任务链 UI 规则

- **地图位置**：同一深度调查链应在地图上形成可读的连续路径。后续阶段尽量在前一阶段附近生成，或用连线 / 轨迹表现“从这里继续查”。
- **任务名称**：名称必须带阶段感，例如 `330 末班车 · 乘客口述`、`330 末班车 · 无名站牌`、`330 末班车 · 最后三秒录像`。避免每环像毫无关系的新任务。
- **已完成任务环**：完成后的旧任务点不应完全消失，而应变成灰色 / 空心 / 打勾状态，保留在地图上作为“已完成任务环”。
- **当前任务环**：当前可执行任务应高亮，并显示 `第 2/4 环`、`继续深挖`、`真相风险上升` 等摘要。
- **未解锁后续环**：可选择显示为虚线占位、问号节点或“不明后续”，但不要提前暴露全部真相名称。
- **侧栏任务卡**：必须展示链条进度、上一环结果、当前环风险、下一环预期（如“成功后解锁下一环”）。
- **结算反馈**：当前环结算后，奖励/结果弹窗必须明确说明下一环是否出现、出现在哪里、旧环如何归档。
- **完结状态**：链条完结后，整条链在地图上应呈现完整闭环或完成标记，便于玩家回看自己完成了哪条深度调查。

### 5.5 历史分支：split + 二项分布

与 [骰子判定玩法_最终版.md](../../骰子判定玩法_最终版.md)、[汇总 §3.2](./世界未解之谜周刊_全链条版设计文档汇总.md) 一致：

- **调查池**：探索 + 洞察 + 诡思 → \(n_A\)，阈值 \(k_A\) = 对应需求之和。  
- **现场池**：生存 + 理性 → \(n_B\)，\(k_B\)。  
- **难度 → 单骰成功率 p**：简单 0.60 / 普通 0.50 / 困难 1/3；可叠 Δp。  
- **对手 `enemyAttr`**：对已投骰子**随机无放回作废**后再比 \(X \ge k\)（与苏丹 Wiki「减骰」分布等价）。  
- **结果档**：双过 → 大成功；一侧过 → 小成功；双不过 → 失败（实现可含大失败等扩展）。  
- **表现**：骰子动画 vs 纯数值，**逻辑相同**。

该路线当前作为旧 HTML 原型 / 历史分支保留。后续若进入角色骰系统大改，应同步替换 `event-check-resolution.md` 等正式分册。

### 7.2 全链原型中的配方命名（与分册「类型 1–4」对应关系为概念级）

|---------|------|----------------|----------------------|
| `r1` | 抢先快讯 | 现象 + 情报 + 可选工具 | 情报消耗；现象、认知不消耗 |
| `r2` | 深度报道 | 现象 + 认知 + 可选工具 | 情报类按公开规则消耗 |
| `r3` | 个人专栏 | 双认知 + 可选工具 | 同上 |
| `r4` | 爆炸性新闻 | 现象 + 认知 + 爆料卡 + 可选工具 | 同上 |
| `internal` | 内审定调 | 现象 + 可选主笔 + 可选心理干预 | 不消耗现象；产出认知牌 |

### 7.3 内审、主笔、SAN、狂性、公开取向（全链已落地的规则摘要）

- **认知牌**主要由**内审**生成；探索进合成时**不默认赠送**认知牌（与早期「线索自带认知」的 Demo 叙述不同，以 §13 代码为准）。  
- **主笔**：绑定 `staffInternal`（SAN、`nextInternalOkSession`）；内审**失败**锁定至**下一次进入合成阶段**；**心理干预包**可解除疲劳。  
- **异常题材**：`occult` 域或现象 `mysteryBias` 高于阈值时，尝试可概率扣 SAN；**成功**增加**全局** `macro.狂性`（宏观属性，非个人绑定）。  
- **公开取向** `publicStance`：由 r2/r3/r4 所带认知加权得 `sci` / `occult` / `pop`；**r1 快讯不写入取向追踪**。  
- **取向矛盾**：与 `lastPublicStanceByTopic` 冲突时，结算需求乘 **stance clash**（预览用 `peekStanceClashDemandMult` 不写入状态）。  
- **同题材多篇**：`topicSynthOrder` 与 `effectiveStoryBase` 疲劳系数 —— 见代码常量与 §13。
