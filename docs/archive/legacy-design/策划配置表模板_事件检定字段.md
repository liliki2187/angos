# 策划配置表模板（Excel 字段版）——事件检定系统

> 用途：给策划在 Excel 里配置“事件选项检定”，程序按表读入并驱动检定/展示。  
> 覆盖：`single / combined / split` 三种检定类型、对手对抗、难度修正 Δp、结果分档（可选）。

## 1. 表结构建议

建议拆成 3 张表（也可合并为 1 张总表，按字段是否为空判断）：

- **Sheet A：`EventCheck`**（事件检定主表，定义一次检定的类型、属性、阈值、对抗等）
- **Sheet B：`CheckBand`**（可选，单次检定按命中数分档：失败/小成功/大成功）
- **Sheet C：`TextKey`**（可选，文案 Key 与本地化内容，不影响检定逻辑）

下面给出每张表的字段清单（字段名即 Excel 列名）。

---

## 2. Sheet A：`EventCheck`（事件检定主表）

### 2.1 必填字段（所有检定类型都必填）

| 字段名 | 类型 | 示例 | 说明 |
|---|---|---:|---|
| `event_id` | string | `EVT_001` | 事件 ID（或剧情节点 ID） |
| `option_id` | string | `OPT_A` | 事件选项 ID（同一事件下唯一） |
| `check_id` | string | `CHK_001_A` | 检定 ID（全表唯一） |
| `check_type` | enum | `single` | `single / combined / split` |
| `difficulty_id` | enum | `normal` | `easy / normal / hard`（映射到 p_base） |
| `delta_p` | number | `-0.05` | 难度修正 Δp，最终 `p=clamp(p_base+delta_p,0,1)` |
| `show_mode_default` | enum | `numeric` | 默认表现模式：`numeric / dice`（玩家可切换） |
| `show_probability` | bool | `TRUE` | 是否显示理论成功率（不显示则可用风险文案代替） |
| `has_opponent` | bool | `TRUE` | 是否启用对手对抗扣减 |
| `enemy_attr_source` | enum | `input` | 对手属性来源：`none / input / npc_attr / fixed` |
| `enemy_attr_fixed` | int | `0` | 当 `enemy_attr_source=fixed` 时使用 |

> **备注**：原型中对手属性来自输入框（等价于 `enemy_attr_source=input`）。

### 2.2 属性来源字段（按 check_type 填）

#### A) `single` 必填

| 字段名 | 类型 | 示例 | 说明 |
|---|---|---:|---|
| `attr_a_id` | enum | `wisdom` | 属性 A 的标识（智慧/社交/战斗…） |
| `attr_a_source` | enum | `player` | 属性来源：`player / npc / fixed` |
| `attr_a_fixed` | int | 6 | 当 source=fixed 时使用 |
| `k` | int | 3 | 成功阈值：命中数 ≥ k 为成功 |

#### B) `combined` 必填

| 字段名 | 类型 | 示例 | 说明 |
|---|---|---:|---|
| `attr_a_id` | enum | `wisdom` | 属性 A |
| `attr_b_id` | enum | `social` | 属性 B |
| `attr_a_source` | enum | `player` | 同上 |
| `attr_b_source` | enum | `player` | 同上 |
| `attr_a_fixed` | int | 6 | 可选：source=fixed 时用 |
| `attr_b_fixed` | int | 4 | 可选：source=fixed 时用 |
| `combine_formula` | enum | `SUM` | 合并方式（原型为 `SUM`：n=A+B）；可扩展 `WEIGHT_SUM` |
| `attr_a_weight` | number | 1.0 | 仅当 `WEIGHT_SUM` 用 |
| `attr_b_weight` | number | 1.0 | 仅当 `WEIGHT_SUM` 用 |
| `k` | int | 6 | 成功阈值（对合并后的总命中数） |

#### C) `split` 必填

| 字段名 | 类型 | 示例 | 说明 |
|---|---|---:|---|
| `attr_a_id` | enum | `conceal` | 属性 A |
| `attr_b_id` | enum | `social` | 属性 B |
| `attr_a_source` | enum | `player` | 同上 |
| `attr_b_source` | enum | `player` | 同上 |
| `attr_a_fixed` | int | 6 | 可选：source=fixed 时用 |
| `attr_b_fixed` | int | 4 | 可选：source=fixed 时用 |
| `k_a` | int | 3 | A 子检定阈值：XA ≥ kA |
| `k_b` | int | 2 | B 子检定阈值：XB ≥ kB |
| `split_combine_logic` | enum | `BANDS` | 组合逻辑：`ALL / ANY / BANDS`（原型为 BANDS） |
| `opponent_split_policy` | enum | `PROPORTIONAL` | 对手扣减分配策略：`PROPORTIONAL / A_FIRST / B_FIRST / CUSTOM`（原型为 PROPORTIONAL） |

## 3. Sheet B：`CheckBand`（可选：分档结果表）

> 用途：把命中数映射到“失败/小成功/大成功”等档位。  
> 适用：`single / combined`（split 一般用组合逻辑直接出档位，也可扩展专用 band 表）。

| 字段名 | 类型 | 示例 | 说明 |
|---|---|---:|---|
| `check_id` | string | `CHK_001_A` | 对应 `EventCheck.check_id` |
| `band_id` | string | `BAND_1` | 分档 ID |
| `min_success` | int | 0 | 命中数下限（包含） |
| `max_success` | int | 2 | 命中数上限（包含），建议允许 `-1` 表示无上限 |
| `result_tier` | enum | `fail` | `fail / minor_success / major_success` |
| `result_text_key` | string | `TXT_EVT_001_FAIL` | 文案 key（可选） |
| `reward_id` | string | `RWD_001` | 奖励/效果 ID（可选） |
| `next_node` | string | `EVT_001_FAIL_BRANCH` | 分支节点（可选） |

**判定规则**：

- 命中数 `X` 落入 `[min_success, max_success]` 的 band 即命中该档。  
- 若多个 band 重叠，按 `band_id` 或行序优先，**建议策划侧禁止重叠**。

---

## 4. Sheet C：`TextKey`（可选：文案 Key 表）

> 仅用于本地化/文案管理，不影响检定计算。

| 字段名 | 类型 | 示例 | 说明 |
|---|---|---:|---|
| `text_key` | string | `TXT_EVT_001_FAIL` | 文案 key |
| `zh_cn` | string | `谈判失败，对方转而要价。` | 简体中文 |
| `en_us` | string | `Negotiation failed...` | 英文（可选） |

## 5. 枚举与校验规范（程序需要做的校验）

### 5.1 check_type

- `single / combined / split`

### 5.2 difficulty_id → p_base

- `easy` → 0.60  
- `normal` → 0.50  
- `hard` → 0.3333333333

### 5.3 数值约束（建议）

- `attr_*`：≥ 0（原型限制 1–30；正式可按游戏设计扩展）
- `enemy_attr`：≥ 0
- `k, k_a, k_b`：≥ 0
- `delta_p`：建议范围 [-0.3, +0.3]（最终仍会 clamp 到 [0,1]）

### 5.4 运行时 clamp 规则（必须）

- `p = clamp(p_base + delta_p, 0, 1)`
- 单/综合：`n_eff = max(0, n_self - enemy_attr)`，且 `k = clamp(k, 0, n_eff)`
- split：按 `opponent_split_policy` 得到 `enemyA/enemyB`，并：
  - `nA_eff = max(0, nA - enemyA)`，`kA = clamp(kA, 0, nA_eff)`
  - `nB_eff = max(0, nB - enemyB)`，`kB = clamp(kB, 0, nB_eff)`
