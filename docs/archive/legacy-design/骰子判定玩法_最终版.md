# 骰子判定玩法（最终版）——可并入总玩法文档

> **用途**：供剧情、系统、程序、UI 统一引用；与网页原型 `index.html` 行为一致。  
> **定位**：事件选项触发的一次性随机检定，决定分支/奖励档位。

---

## 1. 一句话定义

每次检定 = **投若干颗“成功/失败”骰子**（伯努利试验），统计**命中数** \(X\)；与阈值 \(k\) 比较判定成败。难度决定**单骰成功率** \(p\)；属性决定**骰子数量** \(n\)（或拆成多组）。可选**对手属性**从总骰子中**抵消**若干颗，再结算。

---

## 2. 核心公式（数学模型）

- 单骰：成功概率 \(p\)，失败 \(1-p\)。
- \(n\) 次独立投掷：命中数 \(X \sim \mathrm{Binomial}(n, p)\)。
- **通过条件（单段检定）**：\(X \ge k\)（\(k\) 为需求成功次数）。

**难度 → 基础 \(p\)（与《苏丹的游戏》公开设定一致）**

| 难度 | \(p\) |
|------|--------|
| 简单 | 0.60 |
| 普通 | 0.50 |
| 困难 | \(1/3 \approx 0.333\) |

**事件修正**：`Δp`（可正可负），最终  
`p = clamp(p_base + Δp, 0, 1)`。

---

## 3. 检定类型（三种）

### 3.1 单属性（single）

- **骰子数**：\(n =\) 属性 A 值。
- **对手**：若对手属性为 `enemyAttr`，则**有效骰子数**  
  \(n_{\mathrm{eff}} = \max(0, n - \mathrm{enemyAttr})\)。
- **判定**：\(X \sim \mathrm{Binomial}(n_{\mathrm{eff}}, p)\)，成功当 \(X \ge k\)。

### 3.2 综合多属性（combined）

- **骰子数**：\(n = \mathrm{attrA} + \mathrm{attrB}\)（两属性合并为一池骰子）。
- **对手**：\(n_{\mathrm{eff}} = \max(0, n - \mathrm{enemyAttr})\)。
- **判定**：\(X \sim \mathrm{Binomial}(n_{\mathrm{eff}}, p)\)，成功当 \(X \ge k\)。

### 3.3 分别多属性（split）

- **两组独立检定**：
  - A 池：\(n_A = \mathrm{attrA}\)，阈值 \(k_A\)。
  - B 池：\(n_B = \mathrm{attrB}\)，阈值 \(k_B\)。
- **对手分摊**：总扣减 `enemyAttr` 按 A:B 骰子数比例拆分：
  - `enemyA = floor(enemyAttr * attrA / (attrA + attrB))`
  - `enemyB = enemyAttr - enemyA`
- **有效骰子数**：  
  \(n_{A,\mathrm{eff}} = \max(0, n_A - \mathrm{enemyA})\)，  
  \(n_{B,\mathrm{eff}} = \max(0, n_B - \mathrm{enemyB})\)。
- **判定**：
  - \(X_A \sim \mathrm{Binomial}(n_{A,\mathrm{eff}}, p)\)，A 通过 \(\Leftrightarrow X_A \ge k_A\)。
  - \(X_B \sim \mathrm{Binomial}(n_{B,\mathrm{eff}}, p)\)，B 通过 \(\Leftrightarrow X_B \ge k_B\)。
- **结果档位（当前原型/文档约定）**：
  - A、B **都通过** → **大成功**
  - **仅一侧通过** → **小成功**
  - **都未通过** → **失败**

（若总玩法需要“必须两道都过才算成功”，可改为 `ALL` 逻辑，需在配置表中单列。）

---

## 4. 对手对抗：规则与表现（必须一致）

### 4.1 规则语义

- Wiki 表述：**总骰数减去对手属性后再判定** → 等价于用 **\(n_{\mathrm{eff}}\)** 做二项分布。

### 4.2 表现（骰子模式，二段式）

为让玩家感知“对手存在”，**不允许**用“优先把成功翻成失败”等会**改变概率分布**的做法。

**采用做法（与实现一致）**：

1. **玩家回合**：先对**完整** \(n\)（或 A/B 各自完整池）投出每颗骰子的成功/失败。
2. **对手回合**：从已投出的骰子中**随机无放回**选取 `enemyAttr`（或 split 下各池对应数量）颗，标记为**作废**（显示为 `—`），**不计入命中**。
3. **结算**：只对未作废骰子统计命中数，再与 \(k\) / \(k_A,k_B\) 比较。

> 在独立同分布伯努利假设下，该流程与“直接投 \(n_{\mathrm{eff}}\) 颗”在分布上等价，同时满足可展示性。

### 4.3 纯数值模式

- 不播放骰子动画；只显示**理论成功率**与**本次结算结果**（成功/失败或大成功/小成功/失败）。
- 随机种子与骰子模式可共用同一套后端逻辑，仅前端表现不同。

---

## 5. 表现模式切换（产品要求）

| 模式 | 玩家看到的内容 |
|------|----------------|
| **纯数值** | 成功率（%）、风险文案、结算结果、可选历史记录 |
| **骰子** | 同上 + 骰子格子（✓/×）+ 对手作废（`—`）动画 |

**一键切换**：不改变随机逻辑，只改变展示层。

---

## 6. 理论成功率（给 UI / 策划工具）

- **single / combined**：  
  \(P_{\mathrm{pass}} = P(X \ge k)\)，\(X \sim \mathrm{Binomial}(n_{\mathrm{eff}}, p)\)。

- **split（若 UI 展示“至少一侧通过”）**：  
  设 \(P_A = P(X_A \ge k_A)\)，\(P_B = P(X_B \ge k_B)\)，则  
  \(P_{\ge 1} = 1 - (1-P_A)(1-P_B)\)。  
  若需展示大成功/小成功/失败三档，应分别计算 \(P_A P_B\)、\(P_A(1-P_B)+(1-P_A)P_B\)、\((1-P_A)(1-P_B)\)。

## 7. 与总玩法的合并方式（建议）

在总玩法文档中增加一节 **「事件检定」**，引用本文；并约定：

1. **属性来源**：来自角色养成（或固定剧情加成）；对手属性来自 NPC/关卡配置。
2. **事件配置**：使用 `策划配置表模板_事件检定字段.md` 中的 `EventCheck` 等表。
3. **分支**：检定结果驱动 `result_tier` / `next_node`；split 优先用「大成功 / 小成功 / 失败」三档叙事。
4. **可选扩展**：`CheckBand` 表把命中数映射为多档奖励（不改动核心二项模型）。
