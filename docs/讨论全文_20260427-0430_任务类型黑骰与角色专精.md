## 2）A1：白色调查 / 红色截稿（先做规则闭环）

### 目标
- 白色：同题材可反复做，信息逐步累计。
- 红色：一次机会，失败即关闭，不再给重试入口。

### 数据层（最小改动）
- 在任务对象加：
  - `checkType: 'white' | 'red'`（默认 `white`）
  - `missionInstanceId`（红色任务唯一实例）
  - `topicKey`（复用已有线索主题键）
- 在状态里加：
  - `state.redMissionRetiredIds: Set<string>`
  - `state.topicRevealLog: Record<string, RevealEntry[]>`

### 结算规则
- 白色：
  - 失败不封死任务，只记录一次揭示进度（`topicRevealLog[topicKey].push(...)`）。
  - 成功也可继续做（是否加冷却由你拍板，建议 1 天冷却）。
- 红色：
  - 若 tier 属于失败档（先按“失败/大失败”），立即 `retire`。
  - `retire` 后从可选列表隐藏，并记入 `redMissionRetiredIds`，避免周内重刷。
  - 红色成功后是否也结束：我建议“是”，保持“一次截稿”的明确感。

### UI 提示
- 卡片标签：`白色调查` / `红色截稿`
- 红色副文案：`失败后将永久关闭`
- 白色副文案：`可反复调查，揭示逐步累积`

## 4）B1：再追一次（同回合二次判定）+ 灾厄累积

### 目标
- 给玩家一个“搏一下”的手动决策按钮。
- 第二次判定更危险，且把代价并入灾厄轨，不引入 Chaos Bag。

### 触发条件
- 仅当满足其一才显示按钮：
  - `mission.allowPush === true`
  - 参与角色带 `allowPush` 特质
  - 任务 tag 含 `pushable`
- 每次都需要玩家明确点击，不自动推进。

### 资源消耗（先用最小实现）
- 新增 `state.pushBudgetWeekly`（每周 2~3，建议先 2）
- 点击“再追一次”消耗 1 点；为 0 时按钮灰掉

### 数值惩罚（保证“二追更糟”）
- 任务上有 `pushDepth`（首次结算后若再追变 1）
- 第二次判定时给 `runSplitCheck` 输入惩罚：
  - 方案首选：`enemyAttr + 1`
  - 可选：主线阈值 `kA/kB + 1`
- 同时 `state.calamityMeter += 2`

### 灾厄轨（替代袋子）
- `state.calamityMeter`（0~10）
- 到 6：触发一次周负面事件（例如公信下跌）
- 到 10：强惩罚（如狂性上升）并回落一部分（例如减到 5）

### 流程
1. 第一次结算完成  
2. 显示“再追一次？”（若有资格且预算>0）  
3. 玩家点按钮 -> 扣预算 -> 带惩罚重跑一次  
4. 默认最多再追 1 次（先不做三连追）

## 5）B2：深度调查（跨回合累积）+ 真相风险

### 目标
- 和 B1 区分：B1 是“当场再掷一次”，B2 是“多天持续深挖”。
- 深度越高，真相收益越高，但精神/宏观代价显著提高。

### 任务标记
- `investigationMode: 'none' | 'deep'`（默认 `none`）
- 仅 `deep` 任务出现“继续深挖”入口

### 状态记录
- `state.deepInvestigation[missionKey] = { depth, lastWeek }`

### 行为
- 完成一次任务后，玩家可选“继续深挖”（不是立刻再掷）
- 选择后只增加 `depth`，并消耗下一次行动机会
- 下次再做同任务时，`runSplitCheck` 读取 `depth` 给修正

### 数值建议（可调）
- 收益：深度越高，线索质量下限提升（例如每 2 层+1 档）
- 代价：每次深挖 `macro.狂性 += 3 + depth`
- 真实性反噬：深度>=2 后，成功时有概率触发“普通解释”，削弱炒作收益

### 与 B1 的关系
- 建议首版互斥：`deep` 任务不显示“再追一次？”
- 原因：避免一个任务既连掷又叠深度，节奏太快且不易读

---

## 6）B3：高危任务五档判定（仅 high 风险）

### 目标
- 不重写骰子系统，只在结算层把高危任务细分成五档叙事结果。

### 启用条件
- `riskTier: 'normal' | 'high'`
- 仅 `high` 走五档映射；普通任务保持现状

### 五档（建议）
- `crit_success`：大成功
- `success_cost`：成功但反噬
- `partial`：部分成功
- `fail_clue`：失败但有线索
- `crit_fail`：大失败

### 映射策略（首版最稳）
- 保持 `runSplitCheck` 原返回不变（大成功/小成功/失败/大失败）
- 在 `runMission` 内按 `riskTier === 'high'` 二次映射：
  - 大成功 -> `crit_success`
  - 小成功 -> `success_cost` 或 `partial`（按争议条件分流）
  - 失败 -> `fail_clue`
  - 大失败 -> `crit_fail`

### 奖励意向
- `crit_success`：高收益，无代价
- `success_cost`：高收益+宏观负面
- `partial`：中收益
- `fail_clue`：低收益但保留推进感
- `crit_fail`：强负面或空收获

### 与 A1 的交互
- 红色+高危时，是否把 `fail_clue` 也算“失败关闭”要单独拍板  
- 我建议首版不算（只把硬失败关闭），减少挫败

## 总体层级

这 4 条不是并列“字段功能”，而是 4 个不同层级：

- **A1**：任务生命周期层，决定任务是“可反复调查”还是“一次截稿”。
- **B1**：结算后按钮层，某些角色/任务允许玩家赌一次“再追”。
- **B2**：高难任务类型层，某些任务本身围绕“越查越深、何时停手”设计。
- **B3**：高危结果层，只给高危任务更细的五档后果。

第一轮验证应集中在首个可玩区域，比如北美禁区，不把核心体验藏到后面区域。

## 2 / A1：白色调查与红色截稿

这条的重点不是“任务加颜色”，而是让玩家一眼知道：这件事是长期追踪，还是错过就没了。

**白色调查**适合都市怪谈、连续报道、可反复推敲的题材。它的体验是“我每次都更接近一点”，所以失败不应让任务消失，而是留下某种低价值信息、矛盾证词或新的疑点。白色任务的奖励不一定每次都大，但必须让玩家看到同一 `topicKey` 的信息在累积。

具体流程可以是：

1. 玩家在区域里看到白色任务，标签写“白色调查：可反复追踪”。
2. 第一次派遣后，无论成功失败，都给一条“已掌握信息”。
3. 成功给更高质量线索；失败给低质量疑点，但任务仍保留。
4. 再次派遣同一题材时，界面提示“已有调查基础”，让玩家知道这不是重开一次，而是在继续同一条线。
5. 当信息累积到一定程度，可以解锁更深报道、合成加成或新的派生任务。

**红色截稿**适合临时事件、突发电话、限时目击、周五必须拍板的任务。它的体验是“这次机会有重量”。失败后不是简单扣分，而是入口真的消失，玩家能感到错过窗口。

具体流程可以是：

1. 玩家看到红色任务，标签写“红色截稿：失败后关闭”。
2. 派遣前明确提示“这是一次性机会”。
3. 成功后完成报道或获得高价值线索，该任务关闭。
4. 失败后任务从该区域/事件池移除，不再出现。
5. 关闭时给一条冷淡文案，例如“目击者改口，线索窗口关闭。”

A1 的关键体验差异是：白色让玩家愿意“以后再来”，红色让玩家在派遣前犹豫“这次要不要上强人”。

## 4 / B1：再追一次？按钮与灾厄链

B1 是**结算后的冒险按钮**，不是通用重投。它只应该出现在特定角色、特定任务或特定异常情境里。

玩家第一次结算后，系统停住，不自动继续。面板上出现：

- `收手`
- `再追一次？消耗 1 行动点，解锁更糟黑骰`

这里的“更糟黑骰”必须被玩家看见，而不是只写在结果里。比如第二次判定前，骰池区域出现 1 颗新的黑骰，或某颗黑骰翻面显示“灾厄 / 反噬 / 官方介入”。玩家要清楚知道：我不是免费重来，而是在把局面推向更危险。

建议流程：

1. 第一次任务正常判定并结算。
2. 若任务或角色允许，出现“再追一次？”按钮。
3. 玩家点击后，消耗行动点。
4. 判定界面展示新增黑骰或更糟黑骰面。
5. 第二次结果覆盖或补充第一次结果，但必然带风险计数。
6. 多次使用此类行为，累积到灾厄事件链。

灾厄链不使用 Chaos Bag，因为你已经明确撤回。它应该走现有宏观系统：公信、诡名、声望、守序、狂性，以及周事件权重。

示例：

- 第一次再追：只是狂性 +1，面板提示“编辑部开始记住你的激进判断”。
- 第二次再追：公信 -2，出现“地方台开始质疑报道源”。
- 第三次再追：触发灾厄事件，“匿名举报 / 官方辟谣 / 员工失眠”。

B1 的核心不是“再掷一次”，而是玩家亲手按下一个危险按钮，并看到骰池真的变脏。

## 5 / B2：深度调查与真相风险

B2 是高难任务类型，不是 B1 的连按版。它服务的体验是：玩家越接近真相，越可能毁掉报道的戏剧性，甚至伤害员工和宏观局势。

它应该是一个任务本身的结构，比如“废弃电台连续异常”“罗斯威尔档案复核”“地下实验室回访”。这类任务不是一次派遣结束，而是分多层：

- 第 1 层：拿到传闻，适合煽动报道。
- 第 2 层：找到矛盾证据，线索更实。
- 第 3 层：接近真相，奖励变高，但狂性或 SAN 风险显著增加。
- 第 4 层：可能发现普通解释，导致“煞有介事”档位下降。

玩家每次完成后，不是立刻重掷，而是看到一个明确选择：

- `停止调查，带着当前材料回去写`
- `继续深挖，下次调查风险提高`

这里的张力是“停手”。停得早，材料有噱头但不够真；查得深，可能更真实，也可能把怪谈查成普通事故。

具体机制可以这样设计：

1. 深度调查任务有 `调查深度`，界面显示当前层级。
2. 每推进一层，线索真实性或合成质量提高。
3. 同时狂性、员工压力、官方注意、普通解释概率提高。
4. 若查出普通解释，不是任务失败，而是报道方向被迫改变：公信可能提高，诡名/煞有介事收益下降。
5. 玩家可以随时停手，把当前深度的材料带回报道链。

B2 和 B1 的区别必须在体验上非常清楚：

- B1 是“这次结算后，我要不要赌第二把？”
- B2 是“这个事件我已经查到第 2 层了，我还要不要继续往下挖？”

## 6 / B3：高危任务五档结果

B3 只属于高危任务，不替换普通任务。它的作用是让高风险事件不再只有成功/失败，而是每档都有叙事后果。

五档建议保留你采纳的结构：

1. **大成功**：拿到强线索，且几乎无反噬。
2. **成功但反噬**：报道材料很好，但代价同步出现，比如公信下降、狂性上升、员工受创。
3. **部分成功**：拿到一半收益，可以推进但不漂亮。
4. **失败但有线索**：任务失败，但留下下一次可用信息或新 `topicKey`。
5. **大失败**：无线索或线索被污染，并触发明显负面后果。

这里最重要的是“失败但有线索”。它让高危任务的失败不等于白跑，而是变成怪谈调查里的“糟糕但有用”。例如：

- “照片全糊了，但背景里出现了不该有的路牌。”
- “目击者拒绝采访，但反复提到同一个旧地名。”
- “设备烧毁，不过录音里留下 3 秒异常频率。”

B3 应该和 A1/B1/B2组合使用，但不要互相吞掉：

- 红色 + 高危：失败可能关闭任务，但“失败但有线索”可以留下一个低价值 `topicKey`，让玩家不至于完全断线。
- 深度调查 + 高危：越深越容易进入“成功但反噬”或“大失败”，但也更可能拿到大成功。
- 再追一次 + 高危：第二次判定更容易从“部分成功”滑向“成功但反噬”或“大失败”。

- 黑骰再追：听到空白段里有主编小时候的声音。
- 结果：获得超强素材，但员工 SAN / 狂性 / 灾厄链上升。

这样高危任务就不会只是“奖励更大”，而是：

**你真的可能知道得太多。**

## 可以发散成几个系统方向

**线索不再只按真假分类，而按可解释性分类**

比如：

- `硬证据`：照片、记录、样本
- `软证词`：目击、传闻、梦境
- `矛盾点`：官方口径冲突、时间线错位
- `异常残留`：噪声、空白、污染、低频
- `危险真相`：高危任务才产出，强力但反噬大

**编辑台增加“解释姿态”**

同一素材可以选择：

- 掩盖：降低风险，讨好政府
- 科普：提升公信，降低诡名
- 煞有介事：提升诡名和销量
- 震惊体：爆流量，但容易反噬
- 深挖真相：可能解锁高价值报道，也可能掉 SAN

**势力任务改成“解释委托”**

例如：

- 政府委托：把 UFO 写成飞行事故。
- 神秘社群委托：不要删掉“无法解释”的部分。
- 科学机构委托：必须附带实证解释。
- 商业方委托：标题必须足够吓人。
- 内部记者委托：别把真相包装成垃圾流量。

**报道结果可以不是“真/假”，而是“解释后果”**

比如：

- 读者相信了，但政府不爽。
- 政府满意了，但核心读者觉得你怂。
- 科学界认可了，但销量一般。
- 爆款了，但下周辟谣风险上升。
- 你真的说中了，于是某个势力开始盯上你。

## 我会把它概括成一句设计原则

**《未解之谜周刊》的调查负责制造可信的暧昧，编辑负责把暧昧变成可销售的真相。**

这句话很适合作为后续探索、合成、势力、深度任务的共同准绳。

适用叙事：

- 档案缺页；
- 采访对象拒绝确认。

玩家感知：我调查池本来够了，被黑骰压掉一颗。

---

### `抵扣现场`
效果：抵扣 1 个现场池成功。

适用叙事：

- 路线封锁；
- 设备故障；
- 天气恶化；
- 现场被清理。

---

### `吞没任意`
效果：抵扣当前最关键的 1 个成功。  
优先抵扣能让玩家从达标变成不达标的那一池。

适用高危任务。  
这个比普通抵扣更狠，但要少用。

这类不是直接抵扣成功，而是让己方骰变差。

叙事：

- 线索被污染；
- 样本来源不明；
- 照片有剪辑痕迹。

适合未解之谜小报，因为它不是让你失败，而是让素材变脏。

- 关键证人否认；
- 编辑部发现证据链不完整。

这是很适合黑色幽默的面：你不是没料，而是料开始臭了。

### `遮蔽`
效果：本次获得的线索 tier -1，但仍保留线索。

叙事：

- 录音缺失；

**判定本体要简单，黑骰机制要有大颗粒玩法创意。叙事不要塞进每个小骰面，而是由关卡主题决定黑骰的能力表达。**

## 我之前哪些地方也犯了
我之前在几个地方都有类似问题。

**1. 黑骰设计里把每个骰面都写成小剧情**

我把“抵扣调查”解释成证词翻供，把“抵扣现场”解释成设备坏、天气坏。这会让骰面变成文案标签，而不是玩法单位。

问题是玩家不会因为“证词翻供”这个名字觉得好玩。玩家觉得好玩，是因为它在机制上做了有意思的事，比如“偷走你一颗骰子”“复制你的成功”“让你刚锁定的骰子变成黑骰”。

**2. 高危五档也有类似倾向**

我之前容易把五档写成很多叙事文案：照片糊了、录音空白、新闻官记住你。  
这些可以保留，但它们不能替代五档本身的玩法意义。

五档真正应该解决的是：

- 成功但反噬在玩法上怎么不同？
- 失败但有线索如何改变后续选择？
- 大失败是否会关闭任务、污染素材、触发势力介入？

文案只是表达，不是机制。

**3. 深度调查里也容易把“每层发现什么”写太满**

如果每一层都只是“发现 A 文案，于是可写 B 文案”，玩家体验还是读故事。  
更好的方式是每层让规则状态变化：

- 新增一个报道方向槽；
- 解锁一个势力委托；
- 增加黑骰数量；
- 改变黑骰类型；
- 让某些素材从安全变成污染素材。

故事点服务这些变化，而不是替代这些变化。

## 为什么不能这样
因为判定阶段本来就应该短、清晰、有手感。

如果在判定中塞太多小叙事，会有几个问题：

**第一，玩家记不住。**  
每个骰面都有一段叙事解释，最后玩家只会觉得系统在播复杂结果。

**第二，机制不可预判。**  
玩家需要知道“鬼关卡会偷骰”“时间关卡会改时机”“政府关卡会封锁结果”。如果每个面都是小故事，玩家很难形成策略。

**第三，玩法不够创意。**  
“证词翻供所以抵扣 1 点”本质还是扣数值。  
“鬼偷走你的成功骰，下一轮变成黑骰”才是玩法。

**第四，叙事层级混乱。**  
小叙事应该放在任务、关卡、报道、后果里。判定中只保留最必要的机制表达，否则会抢走大叙事的空间。

玩法感受：玩家骰池变大了，但不一定是好事。

#### 2. `替座`
效果：随机替换 1 颗己方骰为黑骰。  
被替换的己方骰本轮消失。

玩法感受：车上座位没变，但乘客换了。

这个比“加入”更狠，因为它减少玩家真实骰。

#### 3. `露相`
效果：所有混入骰在结算前翻面。每个露相黑骰抵消一个同池成功，或制造一次失败标记。

玩法感受：刚才看起来正常的骰子，现在露出黑色。

#### 4. `多一站`
效果：如果本轮已有混入骰，额外延迟一次结算：玩家必须再投一小轮“下车判定”。失败则多留 1 个黑骰到下一阶段。

玩法感受：你以为到站了，但车又开过了一站。

### 首版可控设计
为了不复杂，首版只做两个技能：

- `混入`：加入 1 颗伪装黑骰。
- `替座`：替换 1 颗己方骰为黑骰。

结算依然简单：

- 成功数够：任务成功。
- 成功数不够：任务失败。
- 但混入骰会让你的骰池“不可信”。

### 330 阶段安排
**阶段 1：乘客口述**  
无黑骰，只埋“多出来的三个人”。

**阶段 2：废弃总站**  
出现 1 颗 `混入` 黑骰。  
玩家第一次感到：骰池里有“不属于自己”的东西。

**阶段 3：无名站牌**  
出现 `混入 + 替座`。  
玩家开始担心自己的骰子被替换。

**最终阶段：最后三秒录像**  
黑骰机制升级：

- 混入骰可能保留到下一轮；
- 若混入骰投出负面，触发“车上多了一个人”；
- 若玩家选择再追一次，额外加入一颗无法预览的黑骰。

一句话记忆点：

**330 公交车的黑骰不是攻击你，而是假装是你的人。**

## 二、球状闪电故事方向
如果球状闪电最终梗是：

**球状闪电不是普通自然现象，而像地球神经系统的一次局部放电。**

那这个事件可以这样设计。

### 世界观感觉
球状闪电不是“怪物”，也不是“外星科技”，而是一种短暂出现的地球神经信号。

它像：

- 地壳、电离层、电网、人体神经之间的一次错误连接；
- 地球在暴雨中“眨了一下眼”；
- 人类观测它时，反而成为它信号链的一部分。

这很适合黑色幽默：

编辑部本来想写“厨房火球震惊居民”，最后发现自己像是在采访地球的神经末梢。

## 球状闪电黑骰机制
### 叙事核心
不是“成功过多会爆炸”这么简单，而是：

**你观测它，它也通过你的骰子回传。**

所以机制应该表达“反馈”和“连接”。

### 机制名：`回传`
黑骰不一定直接扣你，而是复制 / 读取 / 改写你的骰面。

### 黑骰技能面
#### 1. `感应`
效果：黑骰复制一颗己方成功骰的点数 / 成功状态，生成一颗黑骰成功。

玩法感受：你越成功，它越能读懂你。

这比普通“复制”更贴合生命感：球状闪电像在学习观测者。

#### 2. `接地`
效果：选择一颗己方骰，将它变成“导体骰”。  
如果它成功，本次成功照算，但结算后触发反噬；如果失败，则黑骰获得 1 点成功。

玩法感受：这颗骰变成连接点，不管成败都危险。

#### 3. `神经脉冲`
效果：黑骰按固定节律影响骰池，例如每第 3 颗骰必须重投，或每第 3 颗成功会转为黑骰信号。

玩法感受：它不是随机捣乱，而是在按某种节律放电。

这个很适合“地球神经元”。

#### 4. `同步`
效果：如果调查池和现场池出现相同数量的成功，黑骰触发额外效果：生成一张危险素材，或增加狂性。

玩法感受：两个系统同步时，异常被接通。

### 首版可控设计
首版建议只做两个：

- `感应`：复制你的成功为黑骰成功。
- `接地`：把一颗己方骰变成导体骰，成功也带反噬。

这样比“过载”更贴合球状闪电像生命 / 神经信号。

### 球状闪电阶段安排
**阶段 1：厨房火球**  
无黑骰。只发现火球穿墙、磁化餐具、环形焦痕。

**阶段 2：气象站异常记录**  
出现 `接地`。  
玩家发现：观测设备不是中立工具，它会成为异常通路。

**阶段 3：废弃变电所**  
出现 `感应 + 接地`。  
玩家越成功，黑骰越能复制成功，像异常在学习你的调查。

**最终阶段：复现实验**  
出现 `神经脉冲` 或强化 `同步`。  
如果玩家的两个骰池形成某种同步，触发危险真相素材：

- 火球内部有心跳式周期；
- 设备记录到类似神经放电的节律；
- 队员在强光中短暂“听见地面在说话”；
- 实验曲线与城市电网、雷暴和人体脑电出现同频。

一句话记忆点：

**球状闪电的黑骰不是烧掉你的成功，而是把你的成功接入它的回路。**

### 330 公交车

机制关键词：

- 混入；
- 替换；

## 四、黑骰设计原则修正
现在黑骰应该按这个方式设计：



- 330：鬼混进人里 → 黑骰混进己方骰池。

- 政府封锁：解释权被控制 → 黑骰禁用报道方向或盖住素材。

这才是比较强的黑骰关卡机制。

# 一、混入：6 颗变 7 颗

## 机制描述
玩家本来有 6 颗白骰。  
投出后，黑骰混入，变成 7 颗骰：

```text
白 白 白 白 白 白 + 伪装黑骰
```

但玩家不一定知道哪颗是黑骰，或者只知道其中一颗“可疑”。

然后玩家需要从 7 颗里选出用于判定的骰子，比如选 4 颗。

## 玩家策略
混入的策略核心是：

**多给你一个选择，但其中可能有鬼。**

玩家会思考：

- 这颗高点数骰是不是太好了，像诱饵？
- 我要不要选安全的中等骰，放弃最高点？
- 如果我需要凑成功数，要不要冒险选可疑骰？
- 我能不能通过角色技能 / 道具识别它？
- 我是否宁愿少拿一个成功，也不碰那颗疑似黑骰？

这很适合“鬼混在人群里”的感觉。

## 混入的好处
它不是单纯削弱玩家，反而表面上给了玩家更多骰子。

玩家会感觉：

> 系统多给我一颗骰，好像是好事，但我不敢用。

这比“扣你一颗成功”有趣得多。

## 混入的风险设计
如果玩家选中了混入黑骰，可以有几种后果：

### 轻版
黑骰不计成功，并抵消 1 个成功。

### 中版
黑骰变成对手成功，影响最终成败。

### 重版
黑骰跟随进入下一阶段，称为“上车”。

例如：

```text
你选中了混入乘客。下一阶段开局额外混入 1 颗黑骰。
```

这会让玩家非常在意选择。

## 混入适合的阶段
适合 330 的中前段。

比如：

- 废弃总站：6 变 7，选 4。
- 无名站牌：6 变 8，选 4。
- 最终录像：6 变 7，但其中一颗必定伪装成高值。

混入的关键词是：

**诱惑。**

# 二、替换：6 颗变 5 白 + 1 黑

## 机制描述
玩家本来有 6 颗白骰。  
黑骰替换其中 1 颗：

```text
白 白 白 白 白 黑
```

总骰数不变，但玩家明确知道池里有黑骰。

然后玩家仍需从 6 颗里选用于判定的骰子，比如选 4 颗。

## 玩家策略
替换的策略核心是：

**你的资源被污染了，但污染是明牌的。**

玩家会思考：

- 我能不能完全避开这颗黑骰？
- 如果黑骰点数很高，我要不要冒险用它？
- 我现在剩下的白骰够不够完成判定？
- 是否要用技能净化 / 驱逐 / 重投黑骰？
- 如果黑骰占了关键位置，我要不要降低目标，争取部分成功？

替换比混入更直接、更压迫。

## 替换的好处
玩家能明确感到“我少了一颗真正属于自己的骰”。

如果原本 6 颗里有 4 颗好骰，现在变成：

```text
好 好 中 低 低 黑
```

玩家马上会觉得不舒服。

## 替换的风险设计
因为替换是明牌，黑骰本身可以设计得更有诱惑：

### 黑骰有高点
黑骰可能显示很高点数，但选它会触发负面。

例如：

```text
黑骰点数 6，可计为成功，但若选中，狂性 +2。
```

这样玩家会纠结：要不要坐上那辆车？

### 黑骰会污染相邻骰
如果选中黑骰，它旁边一颗白骰也变成黑骰。

这适合表现“鬼混入人群会传染”。

### 黑骰不选也有压力
如果黑骰未被选中，但留在池里，可能触发：

```text
未下车：下一阶段保留 1 颗黑骰。
```

这会让玩家不只是绕开它，还要考虑如何处理它。

## 替换适合的阶段
适合 330 的后段。

比如：

- 无名站牌：替换 1 颗。
- 最后三秒录像：替换 2 颗，且至少有 1 颗高点诱饵。
- 再追一次：替换后还额外混入 1 颗未知黑骰。

替换的关键词是：

**污染。**

# 四、要让策略成立，需要一个“选骰”步骤

如果玩家只是自动结算，混入和替换会弱很多。

- 低点但安全的选择；
- 黑骰识别技能价值；

这样它就是 330 的专属玩法，而不是全局复杂化。

## [assistant] #96

我倾向的最优解是：**一人一颗专属骰作为核心，但角色属性不再转换成“更多骰子”，而是转换成“这颗骰子的质量、面组、可控性和任务适配”。**

也就是保留 Slice and Dice 的强归属感，同时避免骰子数量爆炸。

## 两种纯方案的问题

### 方案 A：属性越高，骰子越多
优点是直观：角色多、属性高，就更强。现在原型也更接近这个方向。

问题很明显：

- 骰子一多，单颗骰子没有存在感。
- 角色归属感弱，玩家不知道“这颗成功是谁带来的”。
- 特殊骰面很容易被淹没。
- 黑骰也只能做抵扣，很难做“混入、替换、接地”这种有记忆点的机制。
- 玩家策略会变成堆数值，而不是理解角色。

这和你想要的黑骰、角色技能骰面、特殊关卡机制不太兼容。

### 方案 B：每个角色固定一颗专属骰
优点很强：

- 每个角色有存在感。
- 特殊骰面容易设计。
- 黑骰可以点名影响某个角色骰。
- 玩家能记住“这次是某某的骰面救了我”。

问题是：

- 属性成长怎么体现？
- 角色越多是不是就越强？
- 如果每人一骰，弱角色和强角色差距可能不明显。
- 如果只看成功/失败，数值深度会变薄。

所以纯一人一骰也不能直接套，需要把“属性”换一种表达方式。

## 我建议的最优解：一人一骰，属性改造骰面

每个派遣角色只带 **1 颗角色骰**。  
角色强弱不体现在多投几颗，而体现在这颗骰的结构。

例如一个角色骰有 6 面：


我先把《苏丹的游戏》的骰子机制抽象清楚：**它的属性并不是改变骰面，而是改变骰子数量。骰子本身很简单，属性越高，能投的骰子越多。**

这点很关键，因为它和我们想做的“每角色一颗专属骰”正好是两个方向。

## 苏丹牌式判定核心
公开攻略里描述的核心结构大致是：

```text
事件需要某种属性
→ 角色 / 物品提供属性值
→ 属性值转化为骰子数量
→ 每颗骰子按固定概率成功
→ 成功数越多，结果越好
```

也就是说：

```text
骰子数量 = 当前可用属性值 ± 物品 / 对抗 / 事件修正
```

骰面本身不复杂。通常可以理解为每颗骰只有：

```text
成功 / 失败
```

区别在于每颗骰成功率由难度决定，例如：

- 简单：约 0.6
- 普通：约 0.5
- 困难：约 1/3

所以《苏丹的游戏》的直觉是：

```text
属性 2 = 投 2 颗骰
属性 6 = 投 6 颗骰
属性 10 = 投 10 颗骰
```

玩家不用理解骰面变化，只要看到骰子变多，就知道变强了。

## 单属性检定
假设事件需要“智慧”。

角色智慧为 4，则投 4 颗骰。

```text
智慧 4
→ 4 颗骰
→ 每颗骰独立判定成功 / 失败
→ 成功数量决定结果
```

如果事件要求至少 2 个成功：

- 智慧 2：只能投 2 颗，压力很大。
- 智慧 5：投 5 颗，明显更稳。
- 智慧 8：投 8 颗，直觉上就很强。

这里的成长反馈非常直接：**属性数字 = 桌上骰子数量。**

## 两属性：综合检定
如果一个事件需要两种属性，例如：

```text
智慧 + 社交
```

综合检定就是把两个属性加起来，形成一个骰池。

例如角色：

```text
智慧 3
社交 2
```

那么：

```text
综合骰数 = 智慧 3 + 社交 2 = 5 颗骰
```

如果后来社交成长到 5：

```text
智慧 3 + 社交 5 = 8 颗骰
```

玩家马上能感到：这个角色更会说服人了，因为骰子从 5 颗变成 8 颗。

综合检定的特点是：

**两个属性可以互相补。**

智慧高、社交低也能凑；社交高、智慧低也能凑。只要总骰数够，成功率就上去。

## 两属性：分别检定
另一种是分别检定。比如一个事件要求：

```text
先战斗，再智慧
```

这不是把两个属性加起来，而是分成两个判定：

```text
战斗检定：用战斗值投骰
智慧检定：用智慧值投骰
```

角色：

```text
战斗 5
智慧 1
```

会出现：

```text
战斗阶段很稳
智慧阶段很虚
```

这类检定的特点是：

**不能只靠一个高属性补全部。**

它会逼玩家找更全面的人，或者用物品 / 盟友补短板。

## 对抗如何影响骰子数量
如果事件有对手，对手属性会减少你的骰子数量。

抽象成：

```text
有效骰子数 = 我方属性骰数 - 对手属性 / 难度减扣
```

例如：

```text
我方战斗 6
对手战斗 2
→ 有效骰子约 4 颗
```

这也很直观：

- 我方属性越高，能顶住对抗。
- 对手越强，你的骰子越少。
- 不是改骰面，而是直接把骰子拿走。

## 物品如何影响
物品大致有两类作用：

**1. 增加属性**

例如情报 +2 智慧：

```text
智慧 4 + 情报 2 = 6 颗骰
```

**2. 直接跳过或改变检定**

某些合适物品可以让事件直接成功，或改变事件路线。

但就骰子机制本身来说，最核心仍然是：

```text
加属性 = 加骰子
```

## 为什么它很直观
因为它让成长反馈可视化。

假设每颗骰成功率是 1/3，需要至少 2 个成功：

| 骰子数量 | 至少 2 成功概率 |
|---|---|
| 2 颗 | 很低 |
| 5 颗 | 明显有机会 |
| 8 颗 | 很稳 |

玩家不一定算概率，但会凭直觉知道：

```text
2 颗骰不稳
5 颗骰可以试
8 颗骰很有把握
```

这就是苏丹牌式机制的强项。

## 这种机制的本质
它其实是：

```text
属性值 = 尝试次数
```

不是：

```text
属性值 = 骰面质量
```

所以它的成长反馈非常线性：

```text
属性 +1
→ 多 1 颗骰
→ 多一次成功机会
```

这也是它适合“渐进式属性成长”的原因。

## 它和每角色一颗骰的冲突
如果我们改成 Slice and Dice 式：

```text
每个角色固定 1 颗骰
```

那属性就不能再直接变成骰子数量了。

比如：

```text
探索 3
探索 8
```

如果两者都只投 1 颗角色骰，玩家会问：

```text
那探索 8 到底强在哪里？
```

苏丹牌不会有这个问题，因为探索 8 就是 8 颗骰。

所以我们如果采用“一人一骰”，必须额外解决：

- 属性成长怎么可视化？
- 属性 +1 是否有反馈？
- 高属性如何比低属性直观地强？
- 两属性综合检定如何表达？
- 分别检定如何表达？

## 关键结论
《苏丹的游戏》的骰子机制不是复杂骰面机制，而是**固定骰面 + 属性决定骰子数量**。

它的优点是：

- 成长反馈极直观；
- 多属性综合很自然；
- 对抗和物品修正都容易理解；
- 适合渐进式数值成长。

它的问题是：

- 骰子会越来越多；
- 单颗骰子没有归属；
- 角色特殊技能难塞进骰面；
- 不适合我们想做的黑骰混入 / 替换 / 角色专属骰面。

所以我们后面要解决的不是“苏丹牌怎么做骰面”，而是：

**如果不让属性继续增加骰子数量，怎么把苏丹牌那种直观成长感转译到单颗角色骰上。**

## [assistant] #111

如果确定走 **角色专精 + 骰面数值**，这不是小改骰子公式，而是一次中等到偏大的系统改造。核心影响是：从“属性加总 → 生成骰子数量”，改成“角色骰面 → 产出专精点数 / 特殊效果”。

## 重大系统修改

### 1. 角色数据结构重做
现有角色如果是：

```text
探索 / 生存 / 洞察 / 诡思 / 理性
```

后续要改成：

```text
专精：科学 / 神秘 / 采访 / 现场 / 潜入 / 编辑 ...
角色骰面：[科学1, 采访1, 空, 风险, 科学2, 特殊面]
成长轨：升级后哪些面提升或替换
```

要新增：

- 角色专精标签；
- 角色骰面列表；
- 骰面数值；
- 骰面特殊效果；
- 成长规则；
- 临时骰面 / 插槽来源。

### 2. 任务需求重做
任务不能再写：

```text
需要 探索4 + 洞察3
```

而要写：

```text
需要 科学3 + 现场2
或
需要 采访2 + 神秘2
```

所有探索节点、线索调查、深度任务、高危任务都要重新映射需求。

这会影响：

- 普通探索；
- 红色截稿；
- 深度调查；
- 黑骰关卡；
- 势力任务；
- 报道方向解锁。

### 3. 判定流程重做
当前如果是“属性加总后算骰池”，要改成：

```text
选角色
→ 每个角色投 1 颗专属骰
→ 骰面产出专精点数 / 特殊效果
→ 汇总结果
→ 对照任务需求
→ 黑骰介入
→ 结算成功 / 失败 / 高危档位
```

这里要处理：

- 多个角色骰面产出不同专精；
- 一个骰面可能产出多个标签；
- 特殊面何时触发；
- 黑骰如何混入 / 替换 / 感应；
- 数值如何计入成功。

### 4. UI 大改
这是最大风险之一。

玩家必须能看懂：

- 每个角色骰有哪些面；
- 当前任务需要哪些专精；
- 派这个角色可能贡献什么；
- 升级后骰面会怎么变；
- 黑骰会影响哪颗角色骰；
- 临时插槽来自哪里、持续多久。

需要新增或改造：

- 角色卡骰面预览；
- 任务需求显示；
- 派遣前概率 / 贡献预览；
- 投骰动画；
- 骰面结果解释；
- 升级界面；
- 插槽 / 临时骰面界面。

### 5. 角色成长系统重做
原来成长可能是：

```text
探索 +1
理性 +1
```

现在要变成：

```text
科学1 → 科学2
空面 → 采访1
风险面 → 神秘1+风险
解锁特殊面：校准
```

要决定：

- 升级是固定路线，还是玩家选择；
- 每个角色有几级；
- 每次升级改变几个面；
- 高等级如何避免数值爆炸；
- 插槽是否独立于常规升级。

### 6. 临时人卡 / 道具重做
之前临时人卡可能是临时属性加成。  
现在可以变成：

- 临时支援骰；
- 给某个角色贴一个临时骰面；
- 替换一个空面；
- 给一次黑骰抵抗；
- 临时增加一个专精点数。

道具也要从“属性 +1”改成“骰面 / 面值 / 标签修正”。

### 7. 黑骰系统需要重新对齐
这是好处，也是工作量。

黑骰不再只是抵扣属性池，而是直接操作角色骰：

- 混入：加入伪装黑骰；
- 替换：替换某个角色骰；
- 感应：复制某个专精结果；
- 接地：让某个角色骰变成导体；
- 封锁：盖住一个骰面或禁用一个专精结果。

黑骰会更有趣，但实现和 UI 都更复杂。

### 8. 内容与平衡要重算
之前任务难度可以用属性阈值控制。  
现在要重新平衡：

- 任务需要多少专精点；
- 每个角色平均贡献多少；
- 几名角色能稳定完成普通任务；
- 高危任务成功率；
- 特殊面价值；
- 黑骰强度；
- 插槽 / 道具是否破坏平衡。

## 潜在问题

### 1. 玩家不如“属性越高骰子越多”那么一眼懂
苏丹牌的优势是：

```text
数值高 = 骰子多
```

非常直观。

现在变成：

```text
科学2 面、采访1 面、风险成功、特殊面
```

信息密度高很多。  
如果 UI 不够清楚，玩家会觉得复杂。

### 2. 角色成长反馈可能不连续
如果不是每次升级都明显改骰面，玩家会觉得没成长。  
如果每次升级都改骰面，又会造成学习负担。

需要控制成长节奏：

- 小成长可加资源或熟练度；
- 大成长才改骰面；
- UI 明确提示“下一次骰面变化”。

### 3. 角色数量不能太多
每个角色都要有骰面、成长、特色，设计成本高。  
如果角色太多，会失控。

所以要接受：

- 角色少；
- 每个角色更精；
- 剧情主要靠事件和现象推进。

这与你刚才的判断一致。

### 4. 任务设计工作量增加
每个任务都要考虑：

- 需要哪些专精；
- 哪些角色适配；
- 是否允许替代路线；
- 特殊面会不会过强；
- 黑骰如何针对本关主题。

这比写“探索 4 / 洞察 3”复杂。

### 5. 平衡更难
骰面特殊效果会产生组合爆炸。

例如：

- 某角色能复制科学2；
- 道具再加科学+1；
- 黑骰又被抵消；
- 任务需求瞬间被打穿。

需要早期严格限制：

- 每次派遣人数；
- 特殊面数量；
- 插槽数量；
- 道具叠加；
- 临时支援持续时间。

### 6. 旧系统迁移成本高
现有原型、文档、任务都围绕旧属性。

需要迁移：

- 角色表；
- 任务需求；
- 概率计算；
- UI 展示；
- 结算逻辑；
- GDD 文档；
- 已做的白/红任务、高危五档、深度任务。

不是不能做，但要分阶段。

### 7. 可能削弱经营模拟感
苏丹牌的“属性数值成长”很像经营模拟。  
如果全变成骰面，可能更像战棋 / 骰构筑。

要保留经营感，可以让：

- 训练提升专精等级；
- 采访经验解锁采访面；
- 道具给临时面；
- 员工状态污染骰面；
- 薪酬 / 士气影响风险面。

### 8. 概率预览更难做
以前可以用二项分布估算。  
现在每个角色骰面不同，还可能有特殊面。

要么：

- 用枚举计算所有骰面组合；
- 用模拟估算；
- 或只显示“预计贡献”和“风险提示”。

这会增加实现复杂度。

---

## 迁移建议

不要一次全改。

### 第一阶段：做一条垂直切片
只选 3-4 个角色、2 个任务：

- 330 公交车；
- 球状闪电。

做出：

- 每人一颗专精骰；
- 任务需要专精点；
- 黑骰混入 / 接地；
- 配置页能看懂骰面。

### 第二阶段：替换首区任务需求
把北美禁区带已有任务改为专精需求。

### 第三阶段：做角色成长
先做固定升级：

```text
Lv1 → Lv2：某个骰面数值 +1
Lv2 → Lv3：替换一个空面
```

不要一开始做自由插槽。

### 第四阶段：道具 / 卡牌插槽
等核心跑通，再让特殊道具临时改骰面。

---

## 我的判断

这个方向是值得做的，但前提是接受三件事：

- 角色数量少而精；
- UI 必须展示骰面，而不是只展示属性；
- 先用 1-2 条任务链验证，不要全系统一次替换。

它的最大收益是：

**角色、黑骰、任务主题会真正连成一个玩法系统。**

最大风险是：

**复杂度和 UI 负担显著上升。**
