## 这轮做对了什么

- 先用 ADR 锁边界，再重构运行时，避免了 schema 迁移继续靠临时命名推进。
- 把正式真源从 `full_chain` 挪到 `weekly_run` 后，运行时、GDD 和测试三者的对齐明显更清楚。
- UI 没继续维持 demo 式混合大页面，而是拆成“固定壳层 + 阶段场景 + 复用组件”，后续维护成本下降。
- 最小测试保护没有被继续拖延；阶段流、结算结果和布局都已有 smoke test 兜底。

## 这轮做得不好的地方

- UI 重构后才暴露出布局回归，说明阶段切片和布局验证没有提前绑定。
- `ExplorePhase` 的整页重建问题是在结构稳定后才被集中观察到，性能工作介入偏晚。
- 设计文档没有同步跟上 2026-04-09 的原型规则，形成了“运行时 / 原型已变，正式 GDD 还没跟上”的新漂移风险。
