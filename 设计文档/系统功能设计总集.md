## 2. 游戏定位与一周闭环

- **体裁**：NEWS TOWER 式周刊节奏 + 苏丹式卡牌/线索驱动；玩家为**小众未解之谜周刊主编**。
- **标准周循环**（与 [全链条汇总 §2](../docs/世界未解之谜周刊_全链条版设计文档汇总.md)、[`design/systems/full-chain-loop.md`](../design/systems/full-chain-loop.md) 一致）：

  1. 回合起始（简报、事件提示等）  
  2. 全球地图 → 区域 → 探索节点  
  3. 组队（1–3 人）→ **检定** → **线索**入库  
  4. 日数耗尽或主动结束 → **编辑部**  
  5. **线索 → 报道**（全链含**合成台**；补足故事库规则见 [汇总 §3.3](../docs/世界未解之谜周刊_全链条版设计文档汇总.md)）  
  6. **6 版位组版** → **结算**（需求、销量、订阅、广告、成本、利润）  
  7. 更新跨周状态 → **下一周**

流程图与衔接表见：[世界未解之谜周刊_全链条版设计文档汇总.md §2–§3](../docs/世界未解之谜周刊_全链条版设计文档汇总.md)。

## 3. 宏观属性（周刊五维）

| 属性 | 范围 | 概要 |
|------|------|------|
| **公信** | 0–100 | 可信度、官方/科学合作；与诡名对冲；区域解锁条件之一 |
| **诡名** | 0–100 | 神秘圈影响力；隐藏节点与密教向；与公信对冲 |
| **声望** | 0–100 | 品牌与解锁；探索/发刊反馈 |
| **守序** | 0–100 | 对秩序与规则；与狂性对冲 |
| **狂性** | 0–100 | 靠近真相/超自然污染/灵视；**原则上不自动回落**（仅特殊抑制） |

## 4. 时间与探索流程

- **每周 7 日**：执行节点消耗 `days`；不足则不可执行。  
- **结束本周**：自然耗尽或手动结束 → 编辑部；新周重置日数、清空周临时线索等（Demo 规则）。  
- **全球层**：区域字段、`sci` / `occult` / `pop` 筛选（高亮与可探索性，不删除节点）、**回收噪音**等 —— [探索部分设计文档 §5–§6](../docs/世界未解之谜周刊_探索部分设计文档.md)。  
- **节点**：`permanent` / `temp`（可含 `deadlineDay`）/ `hidden`（`unlock(macro)`）；链式 `chain` —— 同文档 §7。  
- **探索状态机**简图：同文档 §5。

系统级摘要另见 [`design/systems/exploration.md`](../design/systems/exploration.md)。

## 5. 职员、队伍与检定

### 5.1 派遣

- **1–3 名职员**；各 `need` 维度为所选职员该维度**数值之和** ≥ 需求。  
- 全链 Demo 使用五维：**探索、生存、洞察、诡思、理性**（与节点 `need` 一致）。  
- 更完整的「个人属性」列表（想象、笔力、胆识等）见 [`design/systems/attributes.md`](../design/systems/attributes.md) — **与当前全链检定输入未必一一对应**。

### 5.2 全链标准：split + 二项分布

与 [骰子判定玩法_最终版.md](../docs/骰子判定玩法_最终版.md)、[汇总 §3.2](../docs/世界未解之谜周刊_全链条版设计文档汇总.md) 一致：

- **调查池**：探索 + 洞察 + 诡思 → \(n_A\)，阈值 \(k_A\) = 对应需求之和。  
- **现场池**：生存 + 理性 → \(n_B\)，\(k_B\)。  
- **难度 → 单骰成功率 p**：简单 0.60 / 普通 0.50 / 困难 1/3；可叠 Δp。  
- **对手 `enemyAttr`**：对已投骰子**随机无放回作废**后再比 \(X \ge k\)（与苏丹 Wiki「减骰」分布等价）。  
- **结果档**：双过 → 大成功；一侧过 → 小成功；双不过 → 失败（实现可含大失败等扩展）。  
- **表现**：骰子动画 vs 纯数值，**逻辑相同**。

### 5.3 探索分册中的 d6 历史说明

[世界未解之谜周刊_探索部分设计文档 §9](../docs/世界未解之谜周刊_探索部分设计文档.md) 仍保留 **d6 单机 Demo** 公式；**全链条实现以 split 二项为准**，不得与 §5.2 混用为同一口径。

## 7. 故事合成与认知体系

### 7.1 设计规格（分册）

- 卡牌类型：现象、情报、认知、工具、报道 —— 字段、认知等级（1–10）、四维属性、四类配方思路、成功率/失败/污染 —— 全文见 [故事合成与报道编写系统设计文档.md](../docs/故事合成与报道编写系统设计文档.md)。  
- 与报刊接口：四维 → `baseValue`/标签/负面 —— 同文档 §7。  
- 头脑风暴存档（线索·认知·报道，待拍板项）：[`design/gdd/brainstorm-synthesis-cognition-2026-04-06.md`](../design/gdd/brainstorm-synthesis-cognition-2026-04-06.md)。

### 7.2 全链原型中的配方命名（与分册「类型 1–4」对应关系为概念级）

| 配方 ID | 名称 | 必需素材概要 | 消耗约定（全链基线） |
|---------|------|----------------|----------------------|
| `r1` | 抢先快讯 | 现象 + 情报 + 可选工具 | 情报消耗；现象、认知不消耗 |
| `r2` | 深度报道 | 现象 + 认知 + 可选工具 | 情报类按公开规则消耗 |
| `r3` | 个人专栏 | 双认知 + 可选工具 | 同上 |
| `r4` | 爆炸性新闻 | 现象 + 认知 + 爆料卡 + 可选工具 | 同上 |
| `internal` | 内审定调 | 现象 + 可选主笔 + 可选心理干预 | 不消耗现象；产出认知牌 |

### 7.3 内审、主笔、SAN、狂性、公开取向（全链已落地的规则摘要）

- **认知牌**主要由**内审**生成；探索进合成时**不默认赠送**认知牌（与早期「线索自带认知」的 Demo 叙述不同，以 §13 代码为准）。  
- **主笔**：绑定 `staffInternal`（SAN、`nextInternalOkSession`）；内审**失败**锁定至**下一次进入合成阶段**；**心理干预包**可解除疲劳。  
- **异常题材**：`occult` 域或现象 `mysteryBias` 高于阈值时，尝试可概率扣 SAN；**成功**增加**全局** `macro.狂性`（宏观属性，非个人绑定）。  
- **公开取向** `publicStance`：由 r2/r3/r4 所带认知加权得 `sci` / `occult` / `pop`；**r1 快讯不写入取向追踪**。  
- **取向矛盾**：与 `lastPublicStanceByTopic` 冲突时，结算需求乘 **stance clash**（预览用 `peekStanceClashDemandMult` 不写入状态）。  
- **同题材多篇**：`topicSynthOrder` 与 `effectiveStoryBase` 疲劳系数 —— 见代码常量与 §13。

### 7.4 实验页

- [`design/htmls/synth-workbench-lab.html`](../design/htmls/synth-workbench-lab.html)：`labMacro.狂性`、简化主笔名单、与全链内审规则对齐的试验场。
